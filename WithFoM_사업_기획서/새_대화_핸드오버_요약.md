## 1. 프로젝트 개요

### 프로젝트명
WithFoM (위드폼) - 파트너 서비스 사업 기획서

### 서비스 개념
- 카페/음식점 등 파트너 매장에 컵홀더 광고 캠페인 제공
- 파트너는 캠페인 참여 → 컵홀더 소진 → 리워드/등급 혜택 획득
- 소비자는 QR 스캔 → 리워드 적립/사용

### 주요 폴더 구조
```
WithFoM_사업_기획서/
├── 7. 파트너/
│   ├── 7-1. 라이프 사이클/ (가입~탈퇴 전체 흐름)
│   ├── 7-2. 화면 아키텍처 및 인터랙션/ (UI/UX 상세 명세)
│   ├── 7-3. 주체 별 시스템 설계 및 연동 명세/ (API, 연동)
│   └── 7-5. 권한 및 제한/
├── 3. 서비스 아키텍처/
│   └── 3-7. 파트 별 레퍼런스 북/ (핵심 정책/기준 참조용)
└── 참고자료/ (초기 기획 자료, 새 결정사항보다 하위 우선순위)
```

## 2. 사용자 작업 스타일 이해

### 기획서 작성 방식
1. **하이브리드 구조 선호**: UI 섹션 중심 + 속성별 파일 분리
2. **중복 내용 최소화**: 같은 정보가 여러 곳에 반복되지 않도록 구조화
3. **실행 우선**: 질문보다 먼저 작업 → 결과 보고 → 피드백 반영
4. **디테일 중시**: 번호 체계, 들여쓰기, 용어 일관성 철저히 준수

### 피드백 스타일
- 직접적이고 구체적 ("이거 왜 이래?", "삭제해", "A로 해")
- 문제점 지적 시 즉시 수정 기대
- 여러 수정 사항을 한 번에 나열하는 경우 많음

### 의사결정 방식
- 옵션 제시 → 선택 → 선택한 것만 적용
- 선택 후 추가 수정 요청 빈번 (점진적 개선)
- 참고자료보다 **새로 결정한 내용이 항상 우선**

## 3. 기술적 작업 시 주의사항

### PowerShell 한글 인코딩 문제
- Windows PowerShell에서 한글 폴더/파일명 처리 시 인코딩 오류 발생
- **해결 방법**: `Write` 도구로 파일 생성 (폴더 자동 생성됨)
- `Rename-Item`, `Move-Item`, `git mv` 등은 한글 경로에서 실패

### 폴더/파일 변경 작업
- 폴더명 변경: 새 폴더에 파일 생성 → 기존 파일 삭제
- 파일명 변경: 새 파일 생성 → 기존 파일 삭제
- 대량 작업 시 미리 목록 보고 후 진행

## 6. 새 대화에서 작업 시 참고

### 작업 전 확인 사항
1. `.cursor/rules/` 폴더의 룰 파일들이 자동 적용됨
2. 번호 체계, 들여쓰기 규칙은 `00-wf-router-always.mdc` 참조
3. 작업 프로토콜은 `40-wf-work-protocol.mdc` 참조

### 주의할 점
1. **참고자료 폴더**의 내용은 초기 기획이므로, 본문 파일과 다를 경우 **본문이 정답**
2. 용어: 보드→섹션, 박스→카드 (단, 체크박스는 그대로)
3. 등급 순서: 혜택•기준 (혜택이 먼저)
4. 사이드바에 "홈" 메뉴 없음 (로고 클릭으로 홈 진입)
