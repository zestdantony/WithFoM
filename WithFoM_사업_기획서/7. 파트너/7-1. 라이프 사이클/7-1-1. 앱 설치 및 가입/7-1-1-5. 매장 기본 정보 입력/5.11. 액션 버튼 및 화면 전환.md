**5.11.** 액션 버튼 및 화면 전환


**5.11.1.** 매장 기본 정보 입력 화면 구성

1: 화면 진입
　A. 트리거: 아이디/패스워드 설정 완료 후 [다음] 버튼 클릭
　B. 이전 단계: 7-1-1-4. 아이디(닉네임) 패스워드 설정
　C. 진행 상태: 5/9 단계

2: 입력 필드 구성 (순서)
　A. 사업자등록번호 (5.1)
　B. 사업자명 (5.2) - 사업자 인증 후 자동 표시
　C. 매장명 (5.3)
　D. 대표자명 (5.4) - 본인 인증 정보 자동 기입
　E. 이메일 (5.5)
　F. 연락처 (5.6) - 본인 인증 정보 자동 기입
　G. 매장 주소 (5.7)
　H. 핀 설정 주소 (5.8)
　I. 영업 요일/시간 설정 (5.9)
　J. 정산 계좌 입력 (5.10)


**5.11.2.** 사업자등록번호 입력 및 인증 연동

1: 사업자등록번호 입력
　A. 입력 형식: 숫자 10자리 (하이픈 자동 포맷팅)
　B. 실시간 형식 검사: 10자리 입력 시 형식 유효성 확인

2: 사업자 인증 단계 연동
　A. 입력한 번호가 7-1-1-6. 사업자 인증 화면에 자동 반영
　B. 사업자 인증 완료 시 사업자명 자동 표시 (5.2)


**5.11.3.** 자동 기입 필드 동작

1: 본인 인증 정보 자동 기입
　A. 대표자명: 본인 인증된 이름 자동 기입 (수정 불가)
　B. 연락처: 본인 인증된 휴대폰 번호 자동 기입 (수정 가능)

2: 자동 기입 필드 표시
　A. 배경색: 연한 회색 (자동 기입 표시)
　B. 수정 불가 필드: 잠금 아이콘 표시
　C. 수정 가능 필드: 편집 아이콘 표시


**5.11.4.** 주소 검색 버튼 동작

1: [주소 검색] 버튼 클릭 (매장 주소)
　A. 동작: 주소 검색 모달/화면 표시
　B. 검색 방식: 도로명 주소 검색 (카카오/다음 주소 API)
　C. 선택 완료: 주소 필드에 자동 입력

2: [핀 설정] 버튼 클릭
　A. 동작: 지도 화면 표시
　B. 기본 위치: 매장 주소 기반 위치
　C. 핀 드래그: 정확한 매장 위치 설정
　D. [확인] 버튼: 핀 위치 저장


**5.11.5.** 영업 시간 설정 동작

1: 영업 요일 선택
　A. 월~일 체크박스 형태
　B. [매일] 버튼: 전체 요일 일괄 선택
　C. [평일만] 버튼: 월~금 선택

2: 영업 시간 입력
　A. 시작 시간 / 종료 시간 선택
　B. 시간 선택: 드롭다운 또는 타임피커
　C. 브레이크 타임 설정 (선택)


**5.11.6.** 정산 계좌 입력 동작

1: 은행 선택
　A. 드롭다운에서 은행 선택
　B. 주요 은행 상단 표시

2: 계좌번호 입력
　A. 숫자만 입력 가능
　B. 은행별 계좌번호 형식 검사

3: 예금주 확인
　A. 예금주명 입력 또는 자동 조회
　B. 대표자명과 일치 여부 확인


**5.11.7.** [다음] 버튼 동작

1: 버튼 활성화 조건
　A. 모든 필수 항목 입력 완료
　B. 사업자등록번호 형식 유효
　C. 매장 주소 입력 완료
　D. 영업 시간 설정 완료
　E. 정산 계좌 정보 입력 완료

2: 버튼 클릭 시 동작
　A. 입력 정보 임시 저장
　B. 다음 단계로 화면 전환: 7-1-1-6. 사업자 인증
　C. 전환 애니메이션: 슬라이드 (좌 → 우)

3: 버튼 비활성화 시
　A. 버튼 색상: 회색 (비활성화 상태)
　B. 탭 시: 미입력 항목 안내 토스트 메시지


**5.11.8.** 뒤로가기 동작

1: 매장 기본 정보 입력 화면에서 뒤로가기
　A. 확인 팝업: "입력한 정보가 저장되지 않습니다. 뒤로 가시겠습니까?"
　B. [취소] 버튼: 팝업 닫힘, 현재 화면 유지
　C. [확인] 버튼: 7-1-1-4. 아이디/패스워드 설정 화면으로 이동

2: 입력 정보 처리
　A. 뒤로가기 시: 입력 정보 초기화 (자동 기입 제외)
　B. 자동 기입 정보: 유지됨


**5.11.9.** 다음 단계로의 화면 전환

1: 매장 기본 정보 입력 → 사업자 인증
　A. 트리거: [다음] 버튼 클릭
　B. 이동 경로: 7-1-1-6. 사업자 인증 화면
　C. 전달 데이터: 사업자등록번호 (인증 화면에 자동 반영)

2: 진행 상태 표시 업데이트
　A. 현재 단계: 5/9 → 6/9
　B. 다음 단계 예고: "다음: 사업자 인증"


**5.11.10.** 모바일 환경 고려사항

1: 긴 폼 스크롤
　A. 화면 세로 스크롤 가능
　B. [다음] 버튼: 화면 하단 고정 (floating)

2: 키보드 표시 시
　A. 현재 입력 필드가 보이도록 자동 스크롤
　B. 키보드 위에 [다음] 버튼 고정

3: 주소 검색 모달
　A. 풀스크린 또는 대형 바텀 시트
　B. 검색 결과 스크롤 가능


• 참조: 7-1-1-6. 사업자 인증
