**4.4.** 액션 버튼 및 화면 전환


**4.4.1.** 아이디/패스워드 설정 화면 구성

1: 화면 진입
　A. 트리거: 본인 인증 완료 후 [다음] 버튼 클릭
　B. 이전 단계: 7-1-1-3. 본인 인증
　C. 진행 상태: 4/9 단계

2: 입력 필드 구성
　A. 아이디 입력 필드
　B. 아이디 중복 확인 버튼
　C. 비밀번호 입력 필드
　D. 비밀번호 확인 입력 필드
　E. 비밀번호 강도 표시


**4.4.2.** 아이디 입력 및 중복 확인 동작

1: 아이디 입력
　A. 실시간 유효성 검사: 영문, 숫자 조합 확인
　B. 특수문자 입력 시: 오류 메시지 표시
　C. 최소 길이 미달 시: 안내 메시지 표시

2: [중복 확인] 버튼 클릭
　A. 버튼 활성화 조건: 아이디 형식 유효 시
　B. 동작: 서버에 아이디 중복 여부 조회

3: 중복 확인 결과 표시

| 결과 | 메시지 | UI 변화 |
|---|---|---|
| 사용 가능 | "사용 가능한 아이디입니다" | 초록색 체크 아이콘 표시 |
| 중복 | "이미 사용 중인 아이디입니다" | 빨간색 X 아이콘 표시 |
| 형식 오류 | "영문, 숫자 조합으로 입력해주세요" | 빨간색 경고 아이콘 표시 |


**4.4.3.** 비밀번호 입력 동작

1: 비밀번호 입력
　A. 실시간 유효성 검사: 최소 8자, 영문/숫자/특수문자 조합
　B. 비밀번호 강도 표시: 약함/보통/강함

2: 비밀번호 강도 표시

| 강도 | 조건 | UI 표시 |
|---|---|---|
| 약함 | 8자 미만 또는 1가지 종류만 사용 | 빨간색 바 (1/3) |
| 보통 | 8자 이상, 2가지 종류 조합 | 노란색 바 (2/3) |
| 강함 | 8자 이상, 3가지 이상 종류 조합 | 초록색 바 (3/3) |

3: 비밀번호 보기/숨기기 토글
　A. 눈 아이콘 탭: 비밀번호 표시/숨김 전환
　B. 기본 상태: 숨김 (●●●●●●●●)


**4.4.4.** 비밀번호 확인 입력 동작

1: 비밀번호 확인 입력
　A. 실시간 일치 여부 검사: 비밀번호 필드와 비교

2: 일치 여부 표시

| 상태 | 메시지 | UI 표시 |
|---|---|---|
| 일치 | "비밀번호가 일치합니다" | 초록색 체크 아이콘 |
| 불일치 | "비밀번호가 일치하지 않습니다" | 빨간색 X 아이콘 |
| 미입력 | - | 표시 없음 |


**4.4.5.** [다음] 버튼 동작

1: 버튼 활성화 조건
　A. 아이디 중복 확인 완료 (사용 가능)
　B. 비밀번호 강도 "강함" 이상
　C. 비밀번호 확인 일치

2: 버튼 클릭 시 동작
　A. 아이디/비밀번호 정보 서버 전송
　B. 다음 단계로 화면 전환: 7-1-1-5. 매장 기본 정보 입력
　C. 전환 애니메이션: 슬라이드 (좌 → 우)

3: 버튼 비활성화 시
　A. 버튼 색상: 회색 (비활성화 상태)
　B. 탭 시: 미완료 항목 안내 토스트 메시지


**4.4.6.** 뒤로가기 동작

1: 아이디/패스워드 설정 화면에서 뒤로가기
　A. 확인 팝업: "입력한 정보가 저장되지 않습니다. 뒤로 가시겠습니까?"
　B. [취소] 버튼: 팝업 닫힘, 현재 화면 유지
　C. [확인] 버튼: 7-1-1-3. 본인 인증 완료 화면으로 이동

2: 입력 정보 처리
　A. 뒤로가기 시: 입력 정보 초기화
　B. 본인 인증 정보: 유지됨


**4.4.7.** 다음 단계로의 화면 전환

1: 아이디/패스워드 설정 → 매장 기본 정보 입력
　A. 트리거: [다음] 버튼 클릭
　B. 이동 경로: 7-1-1-5. 매장 기본 정보 입력 화면
　C. 전달 데이터: 아이디 (비밀번호는 서버에만 전송)

2: 진행 상태 표시 업데이트
　A. 현재 단계: 4/9 → 5/9
　B. 다음 단계 예고: "다음: 매장 기본 정보 입력"


**4.4.8.** 키보드 인터랙션

1: 아이디 입력 필드
　A. 키보드 타입: 기본 (영문/숫자)
　B. Return 키: 다음 필드로 이동

2: 비밀번호 입력 필드
　A. 키보드 타입: 보안 키보드
　B. Return 키: 다음 필드로 이동

3: 비밀번호 확인 필드
　A. 키보드 타입: 보안 키보드
　B. Return 키: 키보드 숨김, [다음] 버튼 포커스


**4.4.9.** 모바일 환경 고려사항

1: 키보드 표시 시
　A. 화면 스크롤: 현재 입력 필드가 보이도록 자동 스크롤
　B. [다음] 버튼: 키보드 위에 고정 표시

2: 입력 필드 포커스
　A. 탭 시: 해당 필드로 포커스 이동
　B. 키보드 자동 표시


• 참조: 7-1-1-5. 매장 기본 정보 입력
