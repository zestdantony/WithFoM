　　**6.3.** 상권 입력  

↳ 매장 상세 정보 설정 단계 중 세 번째 화면  

**진행 단계 표시:** 8단계 중 3단계  

　A. 상단 영역  
　　a. 제목: "상권 정보 입력"  
　　b. 설명 텍스트: "매장 주변 상권 유형을 선택해 주세요 (최소 2개, 최대 4개)"  
　　c. 진행 단계 표시: "8단계 중 3단계" (우측 상단)  
　B. 메인 선택 영역  
　　a. 상권 유형 목록  
　　aa. 체크박스 형태로 표시  
　　bb. 상권 유형 종류 (대분류별)  
　	• 주거지역  
　		- 아파트 단지  
　		- 주택가  
　		- 원룸·오피스텔 밀집  
　	• 오피스  
　		- 업무지구  
　		- 공단·산업단지  
　		- 관공서·공공기관  
　		- 병원·의료기관  
　	• 상가 밀집  
　		- 번화가  
　		- 상권 중심가  
　		- 전통시장  
　		- 대형마트·백화점  
　		- 쇼핑몰  
　	• 역세권  
　		- 지하철역  
　		- 버스터미널  
　		- 고속버스터미널  
　		- 기차역  
　	• 관광지  
　		- 관광 명소  
　		- 테마파크  
　		- 해변·해수욕장  
　		- 산·등산로  
　	• 학교  
　		- 초등학교  
　		- 중학교  
　		- 고등학교  
　		- 대학교  
　		- 학교 선택 시 여러 학교 종류를 선택 가능 (체크박스 형태)  
　	• 학원가  
　		- 학원 밀집 지역  
　		- 입시 학원가  
　		- 어학원가  
　	• 시장  
　		- 전통시장  
　		- 재래시장  
　		- 야시장  
　		- 농수산물시장  
　b. 선택 개수 표시  
　　aa. 위치: 상권 유형 목록 하단  
　　bb. 표시 형식: "2개 선택됨 / 최소 2개 필요"  
　　cc. 실시간 업데이트: 선택/해제 시 즉시 반영  
　C. 하단 영역  
　　a. "다음 단계" 버튼  
　　aa. 버튼 상태  
　	• 비활성화 시: 회색 (#CCCCCC), 클릭 불가  
　	• 활성화 시: 파란색 (#0066FF), 클릭 가능  
　	• 조건: 최소 2개 선택 시 활성화  
　bb. 최대 선택 제한  
　	• 4개 선택 시 나머지 체크박스 비활성화 (회색 처리)  
　	• "최대 4개까지 선택 가능합니다" 안내 메시지 표시  
　cc. 버튼 클릭 시  
　	• 다음 화면(고객 층 입력)으로 이동  
　	• 선택한 정보는 자동 저장됨  
　D. 동작 흐름  
　　a. 화면 진입  
　　aa. 상권 유형 목록 표시 (체크박스 형태, 대분류별로 그룹화)  
　　bb. 모든 체크박스 미선택 상태  
　　cc. 선택 개수 표시: "0개 선택됨 / 최소 2개 필요"  
　　dd. "다음 단계" 버튼 비활성화 상태  
　　b. 상권 유형 선택  
　　aa. 체크박스 클릭으로 선택/해제  
　　bb. 학교 선택 시  
　	• 학교가 선택 1개로 카운트됨  
　	• 학교 종류는 여러 개 선택 가능 (초등학교, 중학교, 고등학교, 대학교 중 복수 선택)  
　	• 학교 종류는 체크박스 형태로 표시되어 각각 선택 가능  
　cc. 선택 개수 실시간 업데이트  
　	• 선택 시: "N개 선택됨 / 최소 2개 필요"  
　	• 최소 2개 선택 시 "다음 단계" 버튼 활성화  
　dd. 최대 4개 선택 시  
　	• 나머지 체크박스 비활성화 (회색 처리, 클릭 불가)  
　	• "최대 4개까지 선택 가능합니다" 안내 메시지 표시  
　c. 다음 단계 진행  
　　aa. 최소 2개 이상 선택 완료 시 "다음 단계" 버튼 클릭 가능  
　　bb. 버튼 클릭 시 다음 화면(고객 층 입력)으로 이동  
　　cc. 선택한 정보는 자동 저장됨  
　E. 입력 사항 활용 (파트너용 설명)  
　　a. 입력한 상권 정보는 매장에 맞는 캠페인 매칭에 활용되며, 광고주가 타깃 상권을 설정할 때 이를 바탕으로 적합한 캠페인이 매칭됩니다  
　　b. 동일 상권의 다른 매장과 벤치마크 비교를 통해 매장 운영에 도움이 되는 데이터를 제공받을 수 있습니다  
　F. 입력 사항 활용 (개발자용)  
　　a. 캠페인 매칭  
　　aa. 광고주 타깃 설정과 매칭하여 적합한 캠페인 추천  
　　bb. 동일 상권 우선 배정으로 매칭 확률 향상  
　　cc. 상권 유형 정보를 활용하여 매칭 알고리즘에 가중치 부여  
　b. 데이터 분석  
　　aa. 동일 상권(반경 500m) 평균 대비 지표 제공  
　　bb. 상권별 벤치마크 비교 데이터 생성  
　　cc. 매장 리포트 생성 시 상권 정보 활용  
　c. 매칭 알고리즘  
　　aa. 상권 유형 정보를 매칭 알고리즘의 주요 변수로 활용  
　　bb. 광고주 타깃 상권과 파트너 매장 상권의 일치도를 계산하여 매칭 점수 산출
