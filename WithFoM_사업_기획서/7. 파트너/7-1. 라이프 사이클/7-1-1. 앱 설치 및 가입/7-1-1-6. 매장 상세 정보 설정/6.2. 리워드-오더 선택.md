　　**6.2.** 리워드/오더 선택  


↳ 매장 상세 정보 설정 단계 중 두 번째 화면  

**진행 단계 표시:** 8단계 중 2단계  


　　　　　1: 상단 영역  
　　　　　　A. 제목: "서비스 타입 선택"  
　　　　　　B. 설명 텍스트: "매장에서 제공할 서비스 타입을 선택해 주세요 (1개 이상 선택)"  
　　　　　　C. 진행 단계 표시: "8단계 중 2단계" (우측 상단)  

　　　　　2: 메인 선택 영역  

　　　　　　A. 서비스 타입 카드 (2개)  
　　　　　　　　a. 카드 1: 리워드형  
　　　　　　　　　　aa. 카드 구성  
　　　　　　　　　　　　• 서비스 타입명: "리워드형" (큰 글씨, 상단 중앙)  
　　　　　　　　　　　　• 기능 상태: "리워드 기능 (☑️): 사용 가능"  
　　　　　　　　　　　　• 설명 텍스트: "QR 스캔 후 게임/리워드만 제공, 앱 주문 불가, 메뉴 오더/관리 미노출"  
　　　　　　　　　　　　• 체크박스: 좌측 상단에 체크박스 표시  

　　　　　　　　　　bb. 카드 선택 시  
　　　　　　　　　　　　• 체크박스 선택 표시 (체크마크)  
　　　　　　　　　　　　• 테두리 색상 변경: 회색 → 파란색 (#0066FF)  
　　　　　　　　　　　　• 배경색 약간 변경: 밝은 파란색 배경 (#F0F7FF)  

　　　　　　　　b. 카드 2: 오더형  
　　　　　　　　　　aa. 카드 구성  
　　　　　　　　　　　　• 서비스 타입명: "오더형" (큰 글씨, 상단 중앙)  
　　　　　　　　　　　　• 기능 상태: "오더 기능 (☑️): 사용 가능"  
　　　　　　　　　　　　• 설명 텍스트: "앱 주문·결제만 제공, 게임/리워드 없음"  
　　　　　　　　　　　　• 등급 산정 안내: "오더형 사용 시 파트너 등급 산정에 도움이 됩니다." (작은 글씨, 파란색, 별표 아이콘)  
　　　　　　　　　　　　• 체크박스: 좌측 상단에 체크박스 표시  

　　　　　　　　　　bb. 카드 선택 시  
　　　　　　　　　　　　• 체크박스 선택 표시 (체크마크)  
　　　　　　　　　　　　• 테두리 색상 변경: 회색 → 파란색 (#0066FF)  
　　　　　　　　　　　　• 배경색 약간 변경: 밝은 파란색 배경 (#F0F7FF)  

　　　　　　B. 선택 방식  
　　　　　　　　a. 각 카드에 체크박스 표시  
　　　　　　　　b. 체크박스 클릭으로 선택/해제 가능  
　　　　　　　　c. 하나만 선택 시: 각각의 기능만 적용  
　　　　　　　　　　aa. 리워드형만 선택: 리워드 기능만 활성화, 오더 기능 비활성화  
　　　　　　　　　　bb. 오더형만 선택: 오더 기능만 활성화, 리워드 기능 비활성화  

　　　　　　　　d. 둘 다 선택 시: 복합형으로 적용  
　　　　　　　　　　aa. 리워드와 오더 기능 모두 활성화  
　　　　　　　　　　bb. 쿠폰/포인트 결제 시 자동 적용  

　　　　　3: 하단 영역  

　　　　　　A. "다음 단계" 버튼  
　　　　　　　　a. 버튼 상태  
　　　　　　　　　　aa. 비활성화 시: 회색 (#CCCCCC), 클릭 불가  
　　　　　　　　　　bb. 활성화 시: 파란색 (#0066FF), 클릭 가능  
　　　　　　　　　　cc. 조건: 최소 1개 이상 선택 완료 시 활성화  

　　　　　　　　b. 버튼 클릭 시  
　　　　　　　　　　aa. 다음 화면(상권 입력)으로 이동  
　　　　　　　　　　bb. 선택한 정보는 자동 저장되며, 해당 기능들이 활성화됨  


　　　　　4: 동작 흐름  

　　　　　　A. 화면 진입  
　　　　　　　　a. 두 가지 서비스 타입 카드 표시 (리워드형, 오더형)  
　　　　　　　　b. 각 카드에 체크박스 표시 (미선택 상태)  
　　　　　　　　c. "다음 단계" 버튼 비활성화 상태  

　　　　　　B. 서비스 타입 선택  
　　　　　　　　a. 카드 내 체크박스 클릭으로 선택/해제  
　　　　　　　　b. 리워드형만 선택 시: 리워드 관련 메뉴 및 화면 표시  
　　　　　　　　c. 오더형만 선택 시: 오더 관련 메뉴 및 화면 표시  
　　　　　　　　　　aa. 등급 산정 안내 문구 강조 표시  

　　　　　　　　d. 둘 다 선택 시: 리워드와 오더 관련 메뉴 및 화면 모두 표시  
　　　　　　　　e. 최소 1개 이상 선택 시 "다음 단계" 버튼 활성화  

　　　　　　C. 다음 단계 진행  
　　　　　　　　a. "다음 단계" 버튼 클릭  
　　　　　　　　b. 선택한 정보 저장 및 기능 활성화  
　　　　　　　　c. 다음 화면(상권 입력)으로 이동  


　　　　　5: 입력 사항 활용 (파트너용 설명)  

　　　　　　A. 리워드형 선택 시  
　　　　　　　　a. 고객에게 리워드를 제공하여 고객 만족도를 높이고 재방문을 유도할 수 있습니다  
　　　　　　　　b. QR 스캔 후 게임을 통해 고객 참여를 높이고 매장 브랜딩 효과를 얻을 수 있습니다  

　　　　　　B. 오더형 선택 시  
　　　　　　　　a. 편리하고 간편한 결제 시스템으로 주문 처리를 효율화할 수 있습니다  
　　　　　　　　　　aa. 앱을 통한 주문·결제로 대기 시간을 줄이고 주문 오류를 최소화합니다  
　　　　　　　　　　bb. 무인 주문 시스템으로 인력 부담을 줄일 수 있습니다  

　　　　　　　　b. 파트너 등급 산정에 도움이 됩니다  
　　　　　　　　　　aa. 결제 기능 이용 여부가 등급 산정 기준 중 하나로 포함됩니다  
　　　　　　　　　　bb. 오더형 사용 시 등급 산정 점수에 가산점이 부여되어 더 높은 등급을 받을 수 있습니다  

　　　　　　C. 복합형 선택 시  
　　　　　　　　a. 리워드와 오더 기능을 모두 사용할 수 있어 더 다양한 서비스를 제공할 수 있습니다  
　　　　　　　　b. 고객에게 리워드를 제공하면서 동시에 편리한 주문·결제 서비스를 제공할 수 있습니다  
　　　　　　　　c. 쿠폰/포인트 결제 시 자동 적용되어 고객 편의성을 높일 수 있습니다  


　　　　　6: 입력 사항 활용 (개발자용)  

　　　　　　A. 서비스 기능 활성화  
　　　　　　　　a. 선택한 타입에 따라 리워드/오더 기능 활성화  
　　　　　　　　　　aa. 리워드형 선택 시: 리워드 관련 API 및 화면 활성화  
　　　　　　　　　　bb. 오더형 선택 시: 오더 관련 API 및 화면 활성화  
　　　　　　　　　　cc. 복합형 선택 시: 리워드와 오더 관련 API 및 화면 모두 활성화  

　　　　　　B. 캠페인 매칭  
　　　　　　　　a. 리워드형 선택 시: 리워드 캠페인 매칭 우선  
　　　　　　　　b. 오더형 선택 시: 오더 관련 캠페인 매칭 우선  
　　　　　　　　c. 복합형 선택 시: 리워드와 오더 캠페인 모두 매칭 가능  

　　　　　　C. 메뉴 관리  
　　　　　　　　a. 오더형/복합형 선택 시: 메뉴 등록 및 관리 기능 활성화  
　　　　　　　　b. 리워드형만 선택 시: 메뉴 관리 기능 비활성화  

　　　　　　D. 정산  
　　　　　　　　a. 서비스 타입에 따라 정산 항목 차등 적용  
　　　　　　　　b. 리워드 사용 내역, 오더 결제 내역 등 타입별 정산 데이터 수집  

　　　　　　E. 화면 구성  
　　　　　　　　a. 선택한 타입에 따라 표시되는 메뉴 및 화면 구성 변경  
　　　　　　　　　　aa. 리워드형 선택 시: 리워드 사용 내역 보드, 리워드 승인 기능 등 표시  
　　　　　　　　　　bb. 오더형 선택 시: 결제 내역 보드, 주문 승인 기능 등 표시  
　　　　　　　　　　cc. 복합형 선택 시: 리워드와 오더 관련 보드 및 기능 모두 표시  

　　　　　　F. 등급 산정  
　　　　　　　　a. 오더형 선택 및 사용 시 파트너 등급 산정에 도움이 됩니다  
　　　　　　　　　　aa. 결제 기능 이용 여부가 등급 산정 기준 중 하나로 포함됨  
　　　　　　　　　　bb. 오더형 사용 시 등급 산정 점수에 가산점 부여  

　　　　　　　　b. 등급 산정 기준  
　　　　　　　　　　aa. 일일 소진 보고율  
　　　　　　　　　　bb. 동일 캠페인 벤치마크 대비 QR 스캔율  
　　　　　　　　　　cc. 결제 기능 이용 여부 (오더형 사용 시 가산점)
