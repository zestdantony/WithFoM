　　**6.2.** 리워드/오더 선택  

↳ 매장 상세 정보 설정 단계 중 두 번째 화면  

**진행 단계 표시:** 8단계 중 2단계  

　A. 상단 영역  
　　a. 제목: "서비스 타입 선택"  
　　b. 설명 텍스트: "매장에서 제공할 서비스 타입을 선택해 주세요 (1개 이상 선택)"  
　　c. 진행 단계 표시: "8단계 중 2단계" (우측 상단)  
　B. 메인 선택 영역  
　　a. 서비스 타입 카드 (2개)  
　　aa. 카드 1: 리워드형  
　　• 카드 구성  
　	- 서비스 타입명: "리워드형" (큰 글씨, 상단 중앙)  
　	- 기능 상태: "리워드 기능 (☑️): 사용 가능"  
　	- 설명 텍스트: "QR 스캔 후 게임/리워드만 제공, 앱 주문 불가, 메뉴 오더/관리 미노출"  
　	- 체크박스: 좌측 상단에 체크박스 표시  
　• 카드 선택 시  
　	- 체크박스 선택 표시 (체크마크)  
　	- 테두리 색상 변경: 회색 → 파란색 (#0066FF)  
　	- 배경색 약간 변경: 밝은 파란색 배경 (#F0F7FF)  
　bb. 카드 2: 오더형  
　　• 카드 구성  
　	- 서비스 타입명: "오더형" (큰 글씨, 상단 중앙)  
　	- 기능 상태: "오더 기능 (☑️): 사용 가능"  
　	- 설명 텍스트: "앱 주문·결제만 제공, 게임/리워드 없음"  
　	- 등급 산정 안내: "오더형 사용 시 파트너 등급 산정에 도움이 됩니다." (작은 글씨, 파란색, 별표 아이콘)  
　	- 체크박스: 좌측 상단에 체크박스 표시  
　• 카드 선택 시  
　	- 체크박스 선택 표시 (체크마크)  
　	- 테두리 색상 변경: 회색 → 파란색 (#0066FF)  
　	- 배경색 약간 변경: 밝은 파란색 배경 (#F0F7FF)  
　b. 선택 방식  
　　aa. 각 카드에 체크박스 표시  
　　bb. 체크박스 클릭으로 선택/해제 가능  
　　cc. 하나만 선택 시: 각각의 기능만 적용  
　	• 리워드형만 선택: 리워드 기능만 활성화, 오더 기능 비활성화  
　	• 오더형만 선택: 오더 기능만 활성화, 리워드 기능 비활성화  
　dd. 둘 다 선택 시: 복합형으로 적용  
　	• 리워드와 오더 기능 모두 활성화  
　	• 쿠폰/포인트 결제 시 자동 적용  
　C. 하단 영역  
　　a. "다음 단계" 버튼  
　　aa. 버튼 상태  
　	• 비활성화 시: 회색 (#CCCCCC), 클릭 불가  
　	• 활성화 시: 파란색 (#0066FF), 클릭 가능  
　	• 조건: 최소 1개 이상 선택 완료 시 활성화  
　bb. 버튼 클릭 시  
　	• 다음 화면(상권 입력)으로 이동  
　	• 선택한 정보는 자동 저장되며, 해당 기능들이 활성화됨  
　D. 동작 흐름  
　　a. 화면 진입  
　　aa. 두 가지 서비스 타입 카드 표시 (리워드형, 오더형)  
　　bb. 각 카드에 체크박스 표시 (미선택 상태)  
　　cc. "다음 단계" 버튼 비활성화 상태  
　　b. 서비스 타입 선택  
　　aa. 카드 내 체크박스 클릭으로 선택/해제  
　　bb. 리워드형만 선택 시: 리워드 관련 메뉴 및 화면 표시  
　　cc. 오더형만 선택 시: 오더 관련 메뉴 및 화면 표시  
　	• 등급 산정 안내 문구 강조 표시  
　dd. 둘 다 선택 시: 리워드와 오더 관련 메뉴 및 화면 모두 표시  
　ee. 최소 1개 이상 선택 시 "다음 단계" 버튼 활성화  
　　c. 다음 단계 진행  
　　aa. "다음 단계" 버튼 클릭  
　　bb. 선택한 정보 저장 및 기능 활성화  
　　cc. 다음 화면(상권 입력)으로 이동  
　E. 입력 사항 활용 (파트너용 설명)  
　　a. 리워드형 선택 시  
　　aa. 고객에게 리워드를 제공하여 고객 만족도를 높이고 재방문을 유도할 수 있습니다  
　　bb. QR 스캔 후 게임을 통해 고객 참여를 높이고 매장 브랜딩 효과를 얻을 수 있습니다  
　b. 오더형 선택 시  
　　aa. 편리하고 간편한 결제 시스템으로 주문 처리를 효율화할 수 있습니다  
　	• 앱을 통한 주문·결제로 대기 시간을 줄이고 주문 오류를 최소화합니다  
　	• 무인 주문 시스템으로 인력 부담을 줄일 수 있습니다  
　bb. 파트너 등급 산정에 도움이 됩니다  
　	• 결제 기능 이용 여부가 등급 산정 기준 중 하나로 포함됩니다  
　	• 오더형 사용 시 등급 산정 점수에 가산점이 부여되어 더 높은 등급을 받을 수 있습니다  
　c. 복합형 선택 시  
　　aa. 리워드와 오더 기능을 모두 사용할 수 있어 더 다양한 서비스를 제공할 수 있습니다  
　　bb. 고객에게 리워드를 제공하면서 동시에 편리한 주문·결제 서비스를 제공할 수 있습니다  
　　cc. 쿠폰/포인트 결제 시 자동 적용되어 고객 편의성을 높일 수 있습니다  
　F. 입력 사항 활용 (개발자용)  
　　a. 서비스 기능 활성화  
　　aa. 선택한 타입에 따라 리워드/오더 기능 활성화  
　	• 리워드형 선택 시: 리워드 관련 API 및 화면 활성화  
　	• 오더형 선택 시: 오더 관련 API 및 화면 활성화  
　	• 복합형 선택 시: 리워드와 오더 관련 API 및 화면 모두 활성화  
　b. 캠페인 매칭  
　　aa. 리워드형 선택 시: 리워드 캠페인 매칭 우선  
　　bb. 오더형 선택 시: 오더 관련 캠페인 매칭 우선  
　　cc. 복합형 선택 시: 리워드와 오더 캠페인 모두 매칭 가능  
　c. 메뉴 관리  
　　aa. 오더형/복합형 선택 시: 메뉴 등록 및 관리 기능 활성화  
　　bb. 리워드형만 선택 시: 메뉴 관리 기능 비활성화  
　d. 정산  
　　aa. 서비스 타입에 따라 정산 항목 차등 적용  
　　bb. 리워드 사용 내역, 오더 결제 내역 등 타입별 정산 데이터 수집  
　e. 화면 구성  
　　aa. 선택한 타입에 따라 표시되는 메뉴 및 화면 구성 변경  
　	• 리워드형 선택 시: 리워드 사용 내역 보드, 리워드 승인 기능 등 표시  
　	• 오더형 선택 시: 결제 내역 보드, 주문 승인 기능 등 표시  
　	• 복합형 선택 시: 리워드와 오더 관련 보드 및 기능 모두 표시  
　f. 등급 산정  
　　aa. 오더형 선택 및 사용 시 파트너 등급 산정에 도움이 됩니다  
　	• 결제 기능 이용 여부가 등급 산정 기준 중 하나로 포함됨  
　	• 오더형 사용 시 등급 산정 점수에 가산점 부여  
　bb. 등급 산정 기준  
　	• 일일 소진 보고율  
　	• 동일 캠페인 벤치마크 대비 QR 스캔율  
　	• 결제 기능 이용 여부 (오더형 사용 시 가산점)
