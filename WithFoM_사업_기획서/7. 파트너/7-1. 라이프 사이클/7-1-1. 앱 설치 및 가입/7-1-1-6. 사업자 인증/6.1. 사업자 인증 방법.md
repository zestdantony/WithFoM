**6.1.** 사업자 인증 방법  

↳ 사업자등록번호로 사업자 진위 확인  
**진행 단계 표시:** 6/9 단계  


**6.1.1.** 화면 구성  

**6.1.1.1.** 사업자등록번호 입력/표시 필드  
　1: 이전 단계(7-1-1-5)에서 입력한 사업자등록번호 자동 표시  
　2: 필드 상태: 수정 가능 (필요 시 재입력)  
　3: 번호 수정 시 이전 인증 결과 초기화  

**6.1.1.2.** [인증하기] 버튼  
　1: 사업자등록번호 10자리 형식 유효 시 활성화  
　2: 형식 미달 시 비활성화 (회색)  

**6.1.1.3.** 인증 결과 표시 영역  
　1: 인증 전: 비어 있음  
　2: 인증 중: 로딩 스피너 + "사업자 정보를 확인 중입니다..." 메시지  
　3: 인증 성공: 사업자 정보 표시  
　4: 인증 실패: 오류 메시지 및 안내  

**6.1.1.4.** 수동 인증 요청 링크  
　1: 자동 인증 실패 시 하단에 "수동 인증 요청하기" 링크 표시  


**6.1.2.** 인증 절차  

**6.1.2.1.** 자동 인증 (기본)  
　1: 사업자등록번호 입력 확인 (이전 단계에서 자동 반영)  
　2: [인증하기] 버튼 클릭  
　3: 국세청 API 연동을 통한 사업자 진위 확인 (실시간 조회)  
　4: 인증 결과에 따라 화면 업데이트  

**6.1.2.2.** 수동 인증 (자동 인증 불가 시)  
　1: "수동 인증 요청하기" 링크 클릭  
　2: 수동 인증 안내 화면/팝업 표시  
　3: 사업자등록증 사본 업로드  
　　A. [사진 촬영] 버튼: 카메라 앱 실행  
　　B. [앨범에서 선택] 버튼: 갤러리 앱 실행  
　4: 업로드 완료 후 미리보기 표시  
　5: 검토 요청 제출  
　6: 영업일 기준 1~2일 내 검토 완료  


**6.1.3.** 인증 성공 시 표시 정보  

**6.1.3.1.** 상태 표시  
　1: 초록색 체크마크 아이콘  
　2: "사업자 인증이 완료되었습니다" 메시지  

**6.1.3.2.** 조회된 사업자 정보  

| 표시 항목 | 내용 |
|---|---|
| 사업자명 | 국세청 API에서 조회된 사업자명 |
| 대표자명 | 국세청 API에서 조회된 대표자명 |
| 사업장 소재지 | 국세청 API에서 조회된 주소 |

**6.1.3.3.** 정보 자동 반영  
　1: 사업자명 → 7-1-1-5. 매장 기본 정보 입력의 사업자명(5.2) 필드에 반영  
　2: 조회된 정보는 읽기 전용으로 표시  


**6.1.4.** 인증 실패 시 표시 정보  

**6.1.4.1.** 실패 유형별 메시지  

| 실패 유형 | 메시지 | 추가 동작 |
|---|---|---|
| 번호 오류 | "사업자등록번호를 확인해주세요" | 재입력 요청 |
| 휴업/폐업 | "휴업 또는 폐업 상태의 사업자입니다" | 고객센터 안내 |
| 조회 불가 | "사업자 정보를 조회할 수 없습니다" | 수동 인증 안내 |
| 네트워크 오류 | "네트워크 오류가 발생했습니다" | [다시 시도] 버튼 |


**6.1.5.** 다음 단계 진행 조건  

**6.1.5.1.** [다음] 버튼 활성화 조건  
　1: 사업자 인증 완료 (자동 인증 성공 또는 수동 인증 승인)  

**6.1.5.2.** 수동 인증 대기 중  
　1: [다음] 버튼 비활성화  
　2: "인증 검토가 완료되면 알림을 보내드립니다" 안내 메시지  
　3: 앱 종료 후에도 검토 완료 시 푸시 알림 발송  


**6.1.6.** 키보드 인터랙션  

**6.1.6.1.** 사업자등록번호 수정 시 키보드 타입: 숫자 전용  

**6.1.6.2.** Return 키: [인증하기] 버튼 포커스
