**7.9.** 액션 버튼 및 화면 전환


**7.9.1.** 매장 상세 정보 설정 화면 구성

1: 화면 진입
　A. 트리거: 사업자 인증 완료 후 [다음] 버튼 클릭
　B. 이전 단계: 7-1-1-6. 사업자 인증
　C. 진행 상태: 7/9 단계

2: 단계별 입력 구성 (8단계)
　A. 1단계: 컵홀더 유형 선택 (7.1)
　B. 2단계: 리워드/오더 선택 (7.2)
　C. 3단계: 상권 입력 (7.3)
　D. 4단계: 고객 층 입력 (7.4)
　E. 5단계: 고객 구성 (7.5)
　F. 6단계: 음료 3종 가격 (7.6)
　G. 7단계: 1일 평균 컵홀더 소진량 입력 (7.7)
　H. 8단계: 컵홀더 수용량 입력 (7.8)

3: 진행 표시
　A. 상단에 8단계 중 현재 단계 표시
　B. 진행 바 또는 스텝 인디케이터


**7.9.2.** 컵홀더 유형 선택 (1단계)

1: 유형 선택 버튼
　A. [단독] 버튼: 광고주 비주얼 100% + 컵홀더 무료 제공
　B. [콜라보] 버튼: 파트너 로고 노출 + 컵홀더 반값 지원
　C. 둘 다 선택 가능 (멀티 선택)

2: 컵홀더 종류 선택
　A. [컵홀더만] 버튼
　B. [에어 홀더 + 컵홀더] 버튼

3: 광고군 제외 설정
　A. [광고군 제외 설정] 확장 섹션 탭
　B. 광고군 목록 표시 (최대 3개 선택)
　C. 선택된 광고군 태그로 표시


**7.9.3.** 리워드/오더 선택 (2단계)

1: 서비스 타입 선택 버튼
　A. [리워드형] 버튼: QR 스캔 후 게임/리워드 제공
　B. [오더형] 버튼: 앱 주문·결제 제공
　C. [복합형] 버튼: 둘 다 선택

2: 선택에 따른 안내
　A. 리워드형: "QR 스캔 게임/리워드 기능이 활성화됩니다"
　B. 오더형: "앱 주문·결제 기능이 활성화됩니다"
　C. 복합형: "모든 기능이 활성화됩니다"


**7.9.4.** 상권/고객층/고객구성 입력 (3~5단계)

1: 상권 입력 (3단계)
　A. 상권 타입 선택: 드롭다운 또는 버튼 그룹
　B. 예: 오피스, 주거, 대학가, 역세권, 상업지구 등

2: 고객 층 입력 (4단계)
　A. 주요 고객층 선택: 멀티 선택 가능
　B. 예: 직장인, 학생, 주부, 가족 등

3: 고객 구성 (5단계)
　A. 성별 비율 슬라이더: 남성 / 여성
　B. 연령대 비율 입력 또는 선택


**7.9.5.** 음료 가격 및 컵홀더 정보 입력 (6~8단계)

1: 음료 3종 가격 (6단계)
　A. 대표 메뉴 3개 이름 및 가격 입력
　B. 가격 입력: 숫자 키패드

2: 1일 평균 컵홀더 소진량 (7단계)
　A. 숫자 입력 또는 슬라이더
　B. 예상 범위 안내: "일반적으로 50~200개"

3: 컵홀더 수용량 (8단계)
　A. 한 번에 보관 가능한 컵홀더 수량 입력
　B. 보관 장소 선택: 카운터, 창고 등


**7.9.6.** 단계 간 이동

1: [다음] 버튼
　A. 현재 단계 완료 시 활성화
　B. 클릭: 다음 단계로 이동
　C. 마지막 단계(8단계): "다음" → "완료"로 변경

2: [이전] 버튼 또는 뒤로가기
　A. 이전 단계로 이동
　B. 입력 정보: 유지됨

3: 스텝 인디케이터 탭
　A. 완료된 단계: 탭하여 바로 이동 가능
　B. 미완료 단계: 탭 불가 (순차 진행)


**7.9.7.** [완료] 버튼 동작 (8단계 완료 후)

1: 버튼 활성화 조건
　A. 8단계 모두 필수 항목 입력 완료

2: 버튼 클릭 시 동작
　A. 모든 상세 정보 서버 저장
　B. 다음 단계로 화면 전환: 7-1-1-8. 파트너 계약서 작성
　C. 전환 애니메이션: 슬라이드 (좌 → 우)


**7.9.8.** 뒤로가기 동작

1: 1단계에서 뒤로가기
　A. 확인 팝업: "입력한 정보가 저장되지 않습니다. 뒤로 가시겠습니까?"
　B. [확인] 버튼: 7-1-1-6. 사업자 인증 화면으로 이동

2: 2~8단계에서 뒤로가기
　A. 이전 단계로 이동
　B. 입력 정보: 유지됨


**7.9.9.** 다음 단계로의 화면 전환

1: 매장 상세 정보 설정 → 파트너 계약서 작성
　A. 트리거: 8단계 완료 후 [완료] 버튼 클릭
　B. 이동 경로: 7-1-1-8. 파트너 계약서 작성 화면
　C. 전달 데이터: 입력된 모든 상세 정보

2: 진행 상태 표시 업데이트
　A. 현재 단계: 7/9 → 8/9
　B. 다음 단계 예고: "다음: 파트너 계약서 작성"


**7.9.10.** 모바일 환경 고려사항

1: 단계별 화면 전환
　A. 각 단계를 별도 화면으로 구성
　B. 스와이프로 이전/다음 단계 이동 (선택적)

2: 입력 UI 최적화
　A. 버튼 그룹: 가로 스크롤 또는 그리드
　B. 슬라이더: 터치 드래그
　C. 숫자 입력: 숫자 키패드


• 참조: 7-1-1-8. 파트너 계약서 작성
