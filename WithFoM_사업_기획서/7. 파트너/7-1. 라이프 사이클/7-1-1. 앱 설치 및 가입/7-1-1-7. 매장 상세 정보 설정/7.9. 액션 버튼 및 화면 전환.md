**7.9.** 액션 버튼 및 화면 전환


**7.9.1.** 매장 상세 정보 설정 화면 구성

1: 화면 진입
　A. 트리거: 사업자 인증 완료 후 [다음] 버튼 클릭
　B. 이전 단계: 7-1-1-6. 사업자 인증
　C. 진행 상태: 7/9 단계

2: 단계별 입력 구성 (8단계)
　A. 1단계: 컵홀더 유형 선택 (7.1)
　B. 2단계: 리워드/오더 선택 (7.2)
　C. 3단계: 상권 입력 (7.3)
　D. 4단계: 고객 층 입력 (7.4)
　E. 5단계: 고객 구성 (7.5)
　F. 6단계: 음료 3종 가격 (7.6)
　G. 7단계: 1일 평균 컵홀더 소진량 입력 (7.7)
　H. 8단계: 컵홀더 수용량 입력 (7.8)

3: 진행 표시
　A. 상단에 "8단계 중 N단계" 텍스트 표시 (우측 상단)
　B. 스텝 인디케이터 (진행 바)


**7.9.2.** 컵홀더 유형 선택 (1단계)

1: 유형 선택 카드 (체크박스 기반, 멀티 선택)
　A. 카드 1 — [단독]: 전면 광고주 비주얼 100% + 광고주 QR (크게), 후면 광고주 비주얼 + 위드폼 로고/QR (작게)
　　a. 혜택 뱃지: "컵홀더 무료 제공" (초록색)
　B. 카드 2 — [콜라보]: 전면 광고주 비주얼 100% + 광고주 QR (크게), 후면 파트너 로고 60% + 위드폼 로고/QR 40% (작게)
　　a. 혜택 뱃지: "컵홀더 반값 지원" (주황색)
　　b. 안내 텍스트: "로고 업로드는 캠페인 매칭 시 진행됩니다" (회색)
　C. 선택 방식: 각 카드 좌측 상단 체크박스 클릭으로 선택/해제 (1개 이상 필수)
　D. 선택 시 UI: 체크마크 표시, 테두리 회색→파란색(#0066FF), 배경 밝은 파란색(#F0F7FF)

2: 컵홀더 종류 선택 (유형 1개 이상 선택 후 표시)
　A. 라디오 버튼 형태 양자택일
　　a. [컵홀더만 선택]
　　b. [에어 홀더 + 컵홀더 둘 다 선택]
　B. 한쪽 선택 시 다른 쪽 자동 해제

3: 광고군 제외 설정 (컵홀더 종류 선택 후 표시, 선택 사항)
　A. 제목: "광고군 제외 설정 (선택 사항)"
　B. 체크박스 형태, 대분류별 그룹화 (10개 대분류 + 세부 항목)
　　a. 공공/기관, 엔터/문화, 의료/건강, 교육, 금융, 기업/브랜딩, 패션·뷰티, 커머스/쇼핑, 음식/외식, 여행/레저
　C. 최대 3개 선택 가능 (선택 안 해도 다음 단계 진행 가능)
　D. 안내: "우선 순위에 따라 매칭 시 제외 대상으로 선정, 후순위로 밀리지만 정책상 100% 제외 되는 것은 아닙니다"

4: [다음 단계] 버튼
　A. 활성화 조건: 유형 1개 이상 선택 + 컵홀더 종류 선택 완료
　B. 비활성화: 회색(#CCCCCC), 클릭 불가
　C. 활성화: 파란색(#0066FF), 클릭 가능
　D. 클릭 시: 2단계(리워드/오더 선택)로 이동, 선택 정보 자동 저장


**7.9.3.** 리워드/오더 선택 (2단계)

1: 서비스 타입 카드 (체크박스 기반, 멀티 선택)
　A. 카드 1 — [리워드형]
　　a. 기능 상태: "리워드 기능 (☑️): 사용 가능"
　　b. 설명: "QR 스캔 후 게임/리워드만 제공, 앱 주문 불가, 메뉴 오더/관리 미노출"
　B. 카드 2 — [오더형]
　　a. 기능 상태: "오더 기능 (☑️): 사용 가능"
　　b. 설명: "앱 주문·결제만 제공, 게임/리워드 없음"
　　c. 등급 산정 안내: "오더형 사용 시 파트너 등급 산정에 도움이 됩니다." (파란색, 별표 아이콘)
　C. 선택 방식: 각 카드 좌측 상단 체크박스 클릭으로 선택/해제 (선택 사항)
　　a. 하나만 선택: 해당 기능만 활성화
　　b. 둘 다 선택: 복합형으로 자동 적용 (리워드 + 오더 모두 활성화, 쿠폰/포인트 결제 시 자동 적용)
　　c. 미선택: 리워드/오더 기능 모두 비활성화 상태로 진행
　D. 선택 시 UI: 체크마크 표시, 테두리 회색→파란색(#0066FF), 배경 밝은 파란색(#F0F7FF)

2: [다음 단계] 버튼
　A. 활성화 조건: 항상 활성화 (선택 여부와 무관)
　B. 클릭 시: 3단계(상권 입력)로 이동, 선택 정보 자동 저장 및 기능 활성화
　C. 미선택 시 클릭: 안내 문구 표시 후 3단계로 이동
　　a. 안내 문구: "리워드 기능은 리워드 설정 페이지에서, 오더 기능은 주문 관리 페이지에서 나중에 활성화할 수 있습니다."
　　b. 안내 문구 표시 후 자동 이동 (2초 후) 또는 즉시 이동 가능


**7.9.4.** 상권 입력 (3단계)

1: 상권 유형 목록 (체크박스 형태, 대분류별 그룹화)
　A. 8개 대분류 + 세부 항목
　　a. 주거지역 (아파트 단지, 주택가, 원룸·오피스텔 밀집)
　　b. 오피스 (업무지구, 공단·산업단지, 관공서·공공기관, 병원·의료기관)
　　c. 상가 밀집 (번화가, 상권 중심가, 전통시장, 대형마트·백화점, 쇼핑몰)
　　d. 역세권 (지하철역, 버스터미널, 고속버스터미널, 기차역)
　　e. 관광지 (관광 명소, 테마파크, 해변·해수욕장, 산·등산로)
　　f. 학교 (초등학교, 중학교, 고등학교, 대학교) — 학교 선택 시 여러 종류 복수 선택 가능, 학교 자체는 선택 1개로 카운트
　　g. 학원가 (학원 밀집 지역, 입시 학원가, 어학원가)
　　h. 시장 (전통시장, 재래시장, 야시장, 농수산물시장)
　B. 선택 개수 표시: "N개 선택됨 / 최소 2개 필요" (실시간 업데이트)

2: 선택 제한
　A. 최소 2개 선택 필수
　B. 최대 4개 선택 가능
　C. 4개 선택 시 나머지 체크박스 비활성화 (회색 처리) + "최대 4개까지 선택 가능합니다" 안내 메시지

3: [다음 단계] 버튼
　A. 활성화 조건: 최소 2개 선택 완료
　B. 클릭 시: 4단계(고객 층 입력)로 이동, 선택 정보 자동 저장


**7.9.5.** 고객 층 입력 (4단계)

1: 고객층 목록 (체크박스 형태, 대분류별 그룹화)
　A. 4개 대분류 + 세부 항목
　　a. 직장인 (오피스, 관공서·공공기관, 병원·의료기관, 대형마트·백화점·상가, 교대·야간 근무, 프리랜서, 공단·산업단지, 금융기관)
　　b. 학생 (초등학생, 중학생, 고등학생, 대학생, 취업 준비생, 고시·자격증 준비생, 어학연수생)
　　c. 학부모 (유아·초등 학부모, 중·고등 학부모, 입시·학원 동행 학부모, 어린이집·유치원 학부모)
　　d. 기타 (주부/가사 중심, 자영업자, 지역 주민, 여행객/관광객, 외국인, 시니어(60대 이상), 커플/데이트, 가족 단위 방문객, 퇴직자, 무직·구직자)
　B. 선택된 항목: 태그 형태로 표시 (각 태그에 X 삭제 버튼)
　C. 선택 개수 표시: "N개 선택됨 / 최소 2개 필요" (실시간 업데이트)

2: 선택 제한
　A. 최소 2개 선택 필수
　B. 최대 5개 선택 가능
　C. 5개 선택 시 추가 선택 불가 + "최대 5개까지 선택 가능합니다" 안내 메시지

3: [다음 단계] 버튼
　A. 활성화 조건: 최소 2개 선택 완료
　B. 클릭 시: 5단계(고객 구성)로 이동, 선택 정보 자동 저장


**7.9.6.** 고객 구성 (5단계)

1: 연령대별 성별 비율 슬라이더 (5개)
　A. 연령대: 10대, 20대, 30대, 40대, 50대+
　B. 각 연령대별 구성
　　a. 연령대명 표시
　　b. 성별 비율 슬라이더 (0%~100%): 왼쪽 = 남성, 오른쪽 = 여성
　　c. 슬라이더 이동 시 남/여 비율 자동 계산
　　d. 현재 비율 표시: "남성: N%, 여성: M%" (슬라이더 하단)
　C. 각 슬라이더 독립 조작, 초기값 50%:50%
　D. 모든 연령대 입력 완료 시 총 남/여 비율 자동 계산 표시

2: [다음 단계] 버튼
　A. 활성화 조건: 모든 연령대 남여 비율 입력 완료 + 총 남/여 비율 표시
　B. 클릭 시: 6단계(음료 3종 가격)로 이동, 입력 정보 자동 저장


**7.9.7.** 음료 3종 가격 (6단계)

1: 음료 1 — 아메리카노 (필수, 고정)
　A. 음료명: "아메리카노" (고정, 수정 불가, 회색 배경)
　B. 가격 입력창: 숫자 키패드, 단위 "원" (우측 표시), 필수 표시 빨간색 별표(*)
　C. 가격 검증: 최소 1,000원 ~ 최대 50,000원

2: 음료 2, 3 — 다른 음료 (필수, 순차 추가)
　A. "다른 음료 추가" 버튼 (아메리카노 입력 완료 후 표시)
　B. 버튼 클릭 시 입력창 생성
　　a. 카테고리 선택 드롭다운: 커피, 논커피, 에이드, 스무디, 티, 기타, 직접 입력
　　　　aa. "기타" 선택 시: 바로 음료명 입력 가능
　　　　bb. "직접 입력" 선택 시: 카테고리 없이 음료명만 입력 가능
　　b. 음료명 입력창 (텍스트 입력, 최대 20자, 특수문자 가능, 이모지 제외)
　　c. 가격 입력창 (숫자 입력, 1,000~50,000원)
　　d. 삭제 버튼 (우측 상단 X 버튼)
　C. 첫 번째 다른 음료 입력 완료 후 두 번째 "다른 음료 추가" 버튼 표시

3: [다음 단계] 버튼
　A. 활성화 조건: 3가지 음료 모두 입력 완료 (음료명 + 가격)
　B. 클릭 시: 7단계(1일 평균 컵홀더 소진량 입력)로 이동, 입력 정보 자동 저장


**7.9.8.** 1일 평균 컵홀더 소진량 입력 (7단계)

1: 숫자 입력창
　A. 입력창 위치: 화면 중앙
　B. 단위 표시: "개/일" (입력창 우측)
　C. 입력 방법: 숫자 키패드
　D. 가이드 텍스트: "예상 소진량을 입력해 주세요 (최소 10개, 최대 1000개, 10개 단위)"

2: 입력 검증
　A. 최소값: 10개
　B. 최대값: 1,000개
　C. 입력 단위: 10개 단위 (10, 20, 30, ... 1000)
　D. 검증 실패 시 경고 메시지
　　a. 숫자 외 입력: "숫자만 입력해 주세요"
　　b. 10개 미만: "최소 10개 이상 입력해 주세요"
　　c. 1,000개 초과: "최대 1000개까지 입력 가능합니다"
　　d. 10개 단위 아님: "10개 단위로 입력해 주세요"

3: 하단 안내
　A. "이 값은 컵홀더 발주량 산정에 활용됩니다"

4: [다음 단계] 버튼
　A. 활성화 조건: 입력 완료 및 검증 통과
　B. 클릭 시: 8단계(컵홀더 수용량 입력)로 이동, 입력 정보 자동 저장


**7.9.9.** 컵홀더 수용량 입력 (8단계)

1: 1단계 선택에 따른 입력창 분기
　A. 컵홀더만 선택한 경우
　　a. 입력창 1개: "컵홀더 수용량"
　　b. 숫자 입력창 (숫자만 입력 가능), 단위 "개"
　　c. 최소 1,000개 ~ 최대 10,000개
　B. 에어 홀더 + 컵홀더 둘 다 선택한 경우
　　a. 입력창 2개 동시 표시
　　　　aa. 입력창 1: "에어 홀더 수용량"
　　　　bb. 입력창 2: "컵홀더 수용량"
　　b. 각각 독립적 입력 (최소 1,000개 ~ 최대 10,000개)
　　c. 하나만 입력해도 다음 단계 진행 가능

2: 입력 검증
　A. 1,000개 미만: "최소 1,000개 이상 입력해 주세요"
　B. 10,000개 초과: "최대 10,000개까지 입력 가능합니다"
　C. 숫자 외 입력: "숫자만 입력해 주세요"

3: 하단 안내
　A. "입력한 수용량은 기본 발주량으로 활용됩니다"

4: [완료] 버튼 (마지막 단계이므로 "다음 단계" → "완료"로 변경)
　A. 활성화 조건: 수용량 입력 완료
　　a. 컵홀더만 선택: 컵홀더 수용량 입력 완료 시
　　b. 둘 다 선택: 두 입력창 모두 입력 완료 시 (하나만 입력해도 진행 가능)
　B. 클릭 시: 모든 상세 정보 서버 저장, 7-1-1-8. 파트너 계약서 작성 화면으로 이동


**7.9.10.** 단계 간 이동

1: [다음 단계] / [완료] 버튼 공통
　A. 비활성화: 회색(#CCCCCC), 텍스트 흰색(#FFFFFF), 클릭 불가
　B. 활성화: 파란색(#0066FF), 텍스트 흰색(#FFFFFF), 클릭 가능
　C. 위치: 화면 하단 고정, 전체 너비
　D. 1~7단계: "다음 단계" 텍스트
　E. 8단계: "완료" 텍스트

2: [이전] 버튼 또는 뒤로가기
　A. 이전 단계로 이동
　B. 입력 정보: 유지됨

3: 스텝 인디케이터 탭
　A. 완료된 단계: 탭하여 바로 이동 가능
　B. 미완료 단계: 탭 불가 (순차 진행)


**7.9.11.** [완료] 버튼 동작 (8단계 완료 후)

1: 버튼 활성화 조건
　A. 8단계 모두 필수 항목 입력 완료

2: 버튼 클릭 시 동작
　A. 모든 상세 정보 서버 저장
　B. 다음 단계로 화면 전환: 7-1-1-8. 파트너 계약서 작성
　C. 전환 애니메이션: 슬라이드 (좌 → 우)


**7.9.12.** 뒤로가기 동작

1: 1단계에서 뒤로가기
　A. 확인 팝업: "입력한 정보가 저장되지 않습니다. 뒤로 가시겠습니까?"
　B. [확인] 버튼: 7-1-1-6. 사업자 인증 화면으로 이동
　C. [취소] 버튼: 현재 화면 유지

2: 2~8단계에서 뒤로가기
　A. 이전 단계로 이동
　B. 입력 정보: 유지됨


**7.9.13.** 다음 단계로의 화면 전환

1: 매장 상세 정보 설정 → 파트너 계약서 작성
　A. 트리거: 8단계 완료 후 [완료] 버튼 클릭
　B. 이동 경로: 7-1-1-8. 파트너 계약서 작성 화면
　C. 전달 데이터: 입력된 모든 상세 정보

2: 진행 상태 표시 업데이트
　A. 현재 단계: 7/9 → 8/9
　B. 다음 단계 예고: "다음: 파트너 계약서 작성"


**7.9.14.** 모바일 환경 고려사항

1: 단계별 화면 전환
　A. 각 단계를 별도 화면으로 구성
　B. 스와이프로 이전/다음 단계 이동 (선택적)

2: 입력 UI 최적화
　A. 카드 선택: 체크박스 + 카드 터치 영역 전체 활성화
　B. 라디오 버튼: 터치 영역 확보
　C. 체크박스 목록: 대분류별 접기/펼치기 아코디언
　D. 슬라이더: 터치 드래그 + 좌우 끝 탭으로 미세 조정
　E. 숫자 입력: 숫자 키패드 팝업
　F. 드롭다운: 바텀시트 형태로 표시
