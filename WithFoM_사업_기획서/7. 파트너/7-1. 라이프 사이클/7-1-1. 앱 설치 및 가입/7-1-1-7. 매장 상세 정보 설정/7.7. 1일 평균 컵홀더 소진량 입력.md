**7.7.** 1일 평균 컵홀더 소진량 입력  


↳ 매장 상세 정보 설정 단계 중 일곱 번째 화면  

**진행 단계 표시:** 8단계 중 7단계  


**7.7.1.** 상단 영역  

**7.7.1.1.** 제목: "1일 평균 컵홀더 소진량 입력"  

**7.7.1.2.** 설명 텍스트: "하루 평균 소진되는 컵홀더 개수를 입력해 주세요"  

**7.7.1.3.** 진행 단계 표시: "8단계 중 7단계" (우측 상단)  


**7.7.2.** 메인 입력 영역  

**7.7.2.1.** 숫자 입력창  
　1: 입력창 위치: 화면 중앙  
　2: 단위 표시: "개/일" (입력창 우측)  
　3: 가이드 텍스트: "예상 소진량을 입력해 주세요 (최소 10개, 최대 1000개, 10개 단위)" (입력창 하단)  

　4: 입력 방법  
　　A. 숫자 키패드로 개수 입력  
　　B. 입력 단위: 개/일  
　　C. 최소값: 10개  
　　D. 최대값: 1000개  
　　E. 입력 단위: 최소 10개 단위로 입력 (10, 20, 30, 40, ... 1000)  

　5: 입력 검증  
　　A. 숫자가 아닌 값 입력 시 경고 메시지: "숫자만 입력해 주세요"  
　　B. 최소값 미만 입력 시 경고 메시지: "최소 10개 이상 입력해 주세요"  
　　C. 최대값 초과 입력 시 경고 메시지: "최대 1000개까지 입력 가능합니다"  
　　D. 10개 단위가 아닌 경우 경고 메시지: "10개 단위로 입력해 주세요"  


**7.7.3.** 하단 영역  

**7.7.3.1.** 입력 가이드: "이 값은 컵홀더 발주량 산정에 활용됩니다"  

**7.7.3.2.** "다음 단계" 버튼  
　1: 버튼 상태  
　　A. 비활성화 시: 회색 (#CCCCCC), 클릭 불가  
　　B. 활성화 시: 파란색 (#0066FF), 클릭 가능  
　　C. 조건: 입력 완료 시 활성화  

　2: 버튼 클릭 시  
　　A. 다음 화면(컵홀더 수용량 입력)으로 이동  
　　B. 입력한 정보는 자동 저장됨  


**7.7.4.** 동작 흐름  

**7.7.4.1.** 화면 진입  
　1: 숫자 입력창 표시  
　2: 입력창은 비어있음  
　3: "다음 단계" 버튼 비활성화 상태  

**7.7.4.2.** 소진량 입력  
　1: 입력창 클릭 시 숫자 키패드 팝업  
　2: 숫자 키패드로 소진량 입력  
　3: 입력값 검증 (최소값 10개, 최대값 1000개, 10개 단위)  
　　A. 검증 실패 시 경고 메시지 표시  
　　B. 검증 성공 시 입력값 표시  

　4: 입력 완료 시 "다음 단계" 버튼 활성화  

**7.7.4.3.** 다음 단계 진행  
　1: 입력 완료 후 "다음 단계" 버튼 클릭 가능  
　2: 버튼 클릭 시 다음 화면(컵홀더 수용량 입력)으로 이동  
　3: 입력한 정보는 자동 저장되며, 발주 시스템에 반영됨  


**7.7.5.** 입력 사항 활용 (파트너용 설명)  

**7.7.5.1.** 입력한 소진량 정보는 컵홀더 발주량 산정에 활용됩니다  

**7.7.5.2.** 소진량을 기준으로 적정 발주량을 자동 계산하여 발주 일정을 제안합니다  

**7.7.5.3.** 예상 소진량을 바탕으로 재고 부족 시점을 예측하여 발주 시점을 안내합니다  


**7.7.6.** 입력 사항 활용 (개발자용)  

**7.7.6.1.** 발주량 산정  
　1: 1일 평균 소진량을 기준으로 적정 발주량 계산  
　2: 소진량 × 캠페인 기간으로 총 필요량 산정  
　3: 발주 시스템에서 자동 발주량 계산 시 사용  

**7.7.6.2.** 재고 관리  
　1: 예상 소진량을 바탕으로 재고 부족 시점 예측  
　2: 재고 알림 기능에 활용  
　3: 자동 발주 기능의 트리거 조건으로 활용  

**7.7.6.3.** 캠페인 기간 산정  
　1: 캠페인 기간 및 수량 계획 수립  
　2: 캠페인 수량 ÷ 1일 평균 소진량으로 예상 캠페인 기간 계산  

**7.7.6.4.** 물류 계획  
　1: 물류팀이 배송 일정 및 수량 계획 수립  
　2: 배송 주기 및 배송량 결정에 활용  

**7.7.6.5.** 데이터 분석  
　1: 매장별 소진량 패턴 분석  
　2: 벤치마크 비교 데이터 생성  
　3: 매장 리포트 생성 시 소진량 정보 활용
