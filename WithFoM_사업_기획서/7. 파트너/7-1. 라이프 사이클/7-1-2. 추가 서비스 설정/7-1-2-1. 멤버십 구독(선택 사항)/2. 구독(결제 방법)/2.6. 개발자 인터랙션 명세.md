**2.6.** 개발자 인터랙션 명세  


**2.6.1.** 화면 초기 로딩  

　1: URL 파라미터 수신  
　　A. plan: 선택한 플랜 (plus | premium)  
　　B. price: 정가 (원)  
　　C. discounted_price: 할인가 (원, 프리미엄만)  
　　D. promo: 프로모션 코드  
　　E. source: 진입 경로 (onboarding)  

　2: API 호출  
　　A. 엔드포인트: GET /api/partner/membership/payment-methods  
　　B. 응답 데이터: 지원하는 결제 수단 목록  
　　C. 엔드포인트: GET /api/partner/membership/terms  
　　D. 응답 데이터: 필수 약관 목록 및 내용  


**2.6.2.** 결제 수단 선택 시  

　1: 상태 관리  
　　A. selectedPaymentMethod: "card" | "bank" | "simple"  
　　B. 입력 필드 표시/숨김 처리  

　2: 입력 필드 유효성 검사  
　　A. 실시간 검증 (onBlur 또는 onChange)  
　　B. 에러 메시지 즉시 표시  


**2.6.3.** 결제 완료 버튼 클릭  

　1: 사전 검증  
　　A. 결제 수단 선택 여부 확인  
　　B. 필수 입력 필드 검증  
　　C. 필수 약관 동의 확인  

　2: API 호출  
　　A. 엔드포인트: POST /api/partner/membership/subscribe  
　　B. 요청 데이터  

| 필드 | 타입 | 설명 |  
|---|---|---|  
| plan | string | 선택한 플랜 (plus | premium) |  
| payment_method | string | 결제 수단 (card | bank | simple) |  
| payment_info | object | 결제 정보 (카드/계좌/간편결제별 상이) |  
| terms_agreed | array | 동의한 약관 ID 배열 |  
| promo_code | string | 프로모션 코드 |  

　3: 결제 처리  
　　A. 결제 대행사 API 연동 (PG사)  
　　B. 결제 승인 처리  
　　C. 멤버십 구독 상태 업데이트  


**2.6.4.** 에러 처리  

　1: 네트워크 오류  
　　A. 에러 메시지: "네트워크 연결을 확인해주세요"  
　　B. 재시도 버튼 제공  

　2: 결제 실패  
　　A. PG사 에러 코드에 따른 메시지 표시  
　　B. 에러 로그 기록  

　3: 입력 검증 실패  
　　A. 필드별 에러 메시지 표시  
　　B. 첫 번째 에러 필드로 스크롤 이동  


**2.6.5.** 보안 고려사항  

　1: 결제 정보 암호화  
　　A. 전송 시 HTTPS 필수  
　　B. 카드 정보는 서버에 저장하지 않음 (토큰화 처리)  

　2: 입력 필드 보안  
　　A. 카드 번호, CVC, 비밀번호 마스킹 처리  
　　B. 자동 완성 비활성화 (보안 필드)  


**2.6.6.** 접근성 및 UX 고려사항  

　1: 키보드 네비게이션  
　　A. Tab 키로 입력 필드 간 이동  
　　B. Enter 키로 다음 단계 진행  

　2: 스크린 리더 지원  
　　A. 입력 필드에 적절한 ARIA 라벨 적용  
　　B. 에러 메시지에 aria-live 속성 적용  

　3: 로딩 상태 피드백  
　　A. 결제 처리 중: 버튼 비활성화 + 스피너 표시  
　　B. 중복 클릭 방지
