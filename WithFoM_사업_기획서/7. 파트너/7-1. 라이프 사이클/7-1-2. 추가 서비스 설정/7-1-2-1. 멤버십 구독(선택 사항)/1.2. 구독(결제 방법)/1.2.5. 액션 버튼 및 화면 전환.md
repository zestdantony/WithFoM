**1.2.5.** 액션 버튼 및 화면 전환  


**1.2.5.1.** 하단 액션 영역 구성  

　1: 결제 완료 버튼  
　　A. 버튼 스타일  
　　　　a. Primary 버튼, 전체 너비  
　　　　b. 텍스트: "구독 시작하기" 또는 "결제 완료"  
　　　　c. 위치: 화면 하단 고정 (Sticky Footer)  
　　B. 버튼 상태  
　　　　a. 활성화: 모든 필수 정보 입력 및 약관 동의 완료 시  
　　　　b. 비활성화: 필수 정보 미입력 또는 약관 미동의 시  
　　　　c. 로딩: 결제 처리 중 스피너 표시 및 버튼 비활성화  


**1.2.5.2.** 결제 성공 시 화면 전환  

　1: 결제 완료 API 응답 수신  
　　A. 결제 승인 완료 확인  
　　B. 멤버십 구독 상태 업데이트  

　2: 성공 화면 표시 (선택 사항)  
　　A. 성공 아이콘 및 메시지  
　　B. "멤버십 구독이 완료되었습니다"  

　3: 다음 가입 단계로 자동 이동  
　　A. 이동 경로: /partner/onboarding/reward-setup  
　　B. 이동 시간: 2~3초 후 자동 이동 또는 "다음 단계로" 버튼 클릭  


**1.2.5.3.** 결제 실패 시 처리  

　1: 에러 메시지 표시  
　　A. 위치: 결제 완료 버튼 상단 또는 토스트 메시지  
　　B. 메시지: "결제에 실패했습니다. 다시 시도해주세요."  

　2: 실패 원인 안내 (상세 에러인 경우)  
　　A. 잔액 부족: "계좌 잔액을 확인해주세요"  
　　B. 카드 한도 초과: "카드 한도를 확인해주세요"  
　　C. 카드 유효기간 만료: "카드 유효기간을 확인해주세요"  
　　D. 기타: "결제 정보를 확인하고 다시 시도해주세요"  

　3: 재시도 가능  
　　A. 결제 정보 수정 후 재시도  


**1.2.5.4.** 결제 취소 시 처리  

　1: 멤버십 플랜 선택 화면으로 복귀  
　　A. 뒤로가기 버튼 클릭 시  
　　B. 확인 다이얼로그 표시 후 이동  

　2: 취소 메시지 (선택 사항)  
　　A. 별도 메시지 없음 또는 "결제가 취소되었습니다" 토스트 메시지
