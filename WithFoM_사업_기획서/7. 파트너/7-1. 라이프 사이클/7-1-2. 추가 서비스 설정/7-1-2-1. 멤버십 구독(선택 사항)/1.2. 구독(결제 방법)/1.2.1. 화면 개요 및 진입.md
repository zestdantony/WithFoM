**1.2.1.** 화면 개요 및 진입  


**1.2.1.1.** 화면 목적  

　1: 멤버십 플랜 선택 후 결제 수단을 등록하고 구독을 완료하는 화면  

　2: 가입 직후 멤버십 구독 화면에서 플랜 선택 시 자동 진입  

　3: 결제 정보 입력 및 구독 완료 처리  


**1.2.1.2.** 화면 진입 경로  

　1: 멤버십 플랜 선택 화면에서 진입  
　　A. "플러스 시작하기" 또는 "프리미엄 시작하기" 버튼 클릭  
　　B. URL 경로  
　　　　a. 플러스: /partner/onboarding/membership/plus/payment  
　　　　b. 프리미엄: /partner/onboarding/membership/premium/payment  
　　C. 전달 파라미터: plan, price, promo, source (상세는 개발자 명세 참조)  


**1.2.1.3.** 전체 화면 레이아웃  

　1: 화면 구성 영역  

| 영역 | 위치 | 설명 |  
|---|---|---|  
| 헤더 영역 | 상단 | 화면 제목, 뒤로가기 버튼 |  
| 구독 정보 확인 영역 | 헤더 아래 | 매장명, 사업자등록번호, 대표자명 표시 (접기/펼치기 가능) |  
| 선택 플랜 요약 영역 | 구독 정보 아래 | 선택한 플랜 정보 및 가격 표시 |  
| 결제 수단 선택 영역 | 중앙 본문 | 결제 방법 선택 및 정보 입력 |  
| 약관 동의 영역 | 결제 수단 아래 | 필수 약관 동의 체크박스 |  
| 하단 액션 영역 | 하단 고정 | 결제 완료 버튼 |


**1.2.1.5.** 구독 정보 확인 영역 구성  

　1: 영역 목적  
　　A. 결제 주체(파트너) 정보를 명확히 표시하여 잘못된 계정 결제 방지  
　　B. 전자상거래법 및 정기결제 약관에 따른 결제 당사자 정보 명시  
　　C. 여러 매장 운영 시 결제 대상 매장 확인 용이  

　2: UI 구성 요소  
　　A. 접기/펼치기 버튼  
　　　　a. 기본 상태: 펼침 (첫 진입 시)  
　　　　b. 사용자 선택에 따라 접기 가능  
　　　　c. 아이콘: 화살표(▼/▲) 또는 +/- 아이콘  
　　B. 정보 표시 영역  
　　　　a. 박스 형태, 연한 배경색 (회색 또는 브랜드 컬러 10%)  
　　　　b. 라운드 코너 적용  

　3: 표시 정보  

| 항목 | 표시 내용 | 비고 |  
|---|---|---|  
| 매장명 | 가입 시 입력한 매장명 | 필수 표시 |  
| 사업자등록번호 | XXX-XX-XXXXX 형식 | 필수 표시 |  
| 대표자명 | 가입 시 입력한 대표자명 | 필수 표시 |  

　4: 정보 수정 안내  
　　A. 정보 수정 필요 시: "정보가 다른가요?" 텍스트 링크 제공  
　　B. 클릭 시: 이전 단계(매장 정보 수정)로 이동 또는 고객센터 안내  

　5: 접근성 고려사항  
　　A. 접기/펼치기 상태를 스크린 리더에 aria-expanded 속성으로 전달  
　　B. 정보 영역에 적절한 ARIA 라벨 적용  

　2: 반응형 레이아웃  
　　A. 모바일: 세로 스크롤, 하단 버튼 고정  
　　B. 태블릿/데스크톱: 중앙 정렬, 최대 너비 제한  


**1.2.1.4.** 헤더 영역 구성  

　1: 뒤로가기 버튼  
　　A. 위치: 좌측 상단  
　　B. 클릭 시: 멤버십 플랜 선택 화면으로 복귀  
　　C. 확인 다이얼로그: 입력 중인 정보가 있을 경우 확인 후 이동  

　2: 화면 제목  
　　A. 텍스트: "결제 정보 입력" 또는 "구독 결제"  
　　B. 스타일: 중앙 정렬, 볼드
