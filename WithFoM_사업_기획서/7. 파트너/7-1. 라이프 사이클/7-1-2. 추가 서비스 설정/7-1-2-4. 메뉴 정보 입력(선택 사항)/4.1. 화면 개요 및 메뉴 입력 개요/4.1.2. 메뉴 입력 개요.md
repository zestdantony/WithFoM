**4.1.2.** 메뉴 입력 개요  


**4.1.2.1.** 메뉴 입력 프로세스 개요  

　1: 전체 프로세스 흐름  
　　A. 1단계: 카테고리 생성 (예: 음료, 디저트, 기타)  
　　B. 2단계: 메뉴 추가 및 기본 정보 입력  
　　C. 3단계: 메뉴 가격 및 옵션 설정  
　　D. 4단계: 메뉴 이미지 업로드 (권장)  
　　E. 5단계: 판매 상태 및 노출 시간 설정  
　　F. 6단계: 리워드 사용 가능 여부 설정 (선택 사항)  
　　G. 7단계: 저장 및 완료  

　2: 필수 입력 항목  
　　A. 메뉴 카테고리 (필수)  
　　B. 메뉴명 (필수)  
　　C. 가격 (필수)  

　3: 선택 입력 항목  
　　A. 메뉴 설명 (선택)  
　　B. 메뉴 이미지 (권장)  
　　C. 영문 메뉴명 (선택)  
　　D. 옵션 (사이즈, 온도, 당도 등, 선택)  
　　E. 판매 상태 (판매중/일시중지/솔드아웃)  
　　F. 노출 시간대 설정 (선택)  
　　G. 인기 메뉴 표시 (선택)  
　　H. 알레르기 정보 (선택)  


**4.1.2.2.** 메뉴 입력 항목 상세  

| 항목 | 설명 | 필수 여부 | 입력 형식 |  
|---|---|---|---|  
| 메뉴 카테고리 | 음료, 디저트, 기타 등 | 필수 | 드롭다운 선택 또는 직접 입력 |  
| 메뉴명 | 메뉴 이름 | 필수 | 텍스트 입력 (최대 20자) |  
| 영문 메뉴명 | 메뉴 영문 이름 | 선택 | 텍스트 입력 (최대 20자) |  
| 가격 | 메뉴 가격 | 필수 | 숫자 입력 |  
| 할인 가격 | 할인된 가격 | 선택 | 숫자 입력 |  
| 메뉴 설명 | 메뉴 설명 | 선택 | 텍스트 입력 (최대 100자) |  
| 메뉴 이미지 | 메뉴 사진 | 권장 | 이미지 업로드 (1:1, 최대 5MB) |  
| 옵션 | 사이즈, 온도, 당도 등 | 선택 | 옵션 그룹 선택 또는 직접 생성 |  
| 판매 상태 | 판매중/일시중지/솔드아웃 | 선택 | 상태 선택 (기본값: 판매중) |  
| 노출 시간대 | 특정 시간대만 노출 | 선택 | 시간 범위 설정 |  
| 인기 메뉴 | 인기 뱃지 표시 | 선택 | 체크박스 |  
| 알레르기 정보 | 알레르기 유발 성분 | 선택 | 다중 선택 |  


**4.1.2.3.** 카테고리 관리 개요  

　1: 카테고리 생성  
　　A. 카테고리명 입력 (예: 음료, 디저트, 기타)  
　　B. 카테고리 저장  
　　C. 카테고리별로 메뉴를 분류하여 관리  

　2: 카테고리 수정 및 삭제  
　　A. 카테고리명 수정 가능  
　　B. 카테고리 삭제 시 삭제 옵션 선택  
　　　　a. 옵션 1: 카테고리만 삭제 (메뉴는 "미분류"로 이동)  
　　　　b. 옵션 2: 카테고리와 모든 메뉴 함께 삭제 (복구 불가)  
　　C. 카테고리 노출 여부 ON/OFF 설정 가능  

　3: 카테고리 정렬  
　　A. 위/아래 이동으로 노출 순서 조정 가능  
　　B. 드래그 앤 드롭으로 순서 변경 가능  


**4.1.2.4.** 메뉴 순서 조정  

　1: 카테고리 내 메뉴 순서 조정  
　　A. 드래그 앤 드롭으로 메뉴 순서 변경  
　　B. 위/아래 이동 버튼으로 순서 조정  
　　C. 순서는 소비자 앱 메뉴 노출 순서에 반영  

　2: 순서 저장  
　　A. 순서 변경 시 자동 저장  
　　B. 또는 "순서 저장" 버튼 클릭으로 저장  


**4.1.2.5.** 옵션 그룹 관리 개요  

　1: 옵션 그룹 종류  
　　A. 사이즈: Small, Medium, Large 등  
　　B. 온도: Hot, Ice 등  
　　C. 당도: 당도 조절 (0~100%)  
　　D. 추가 옵션: 샷 추가, 시럽 추가 등  

　2: 옵션 그룹 생성  
　　A. 옵션 그룹명 입력 (예: 사이즈 선택)  
　　B. 옵션 항목 및 가격 설정  
　　C. 여러 메뉴에 일괄 적용 가능  

　3: 옵션 유형  
　　A. 단일형: 옵션을 1개만 선택할 수 있는 형태  
　　　　• 예시: 온도 선택 (Hot 또는 Ice 중 1개만)  
　　　　• 예시: 사이즈 선택 (Small, Medium, Large 중 1개만)  
　　B. 복수형: 옵션을 1개 이상 선택할 수 있는 형태  
　　　　• 예시: 토핑 추가 (휘핑크림, 초코칩, 카라멜 소스 등 여러 개 선택 가능)  
　　　　• 각 옵션은 선택/미선택만 가능, 개수 지정 불가  
　　C. 수량형: 옵션을 선택하면서 개수까지 지정하는 형태  
　　　　• 예시: 샷 추가 - 에스프레소 샷 [2]개 (+500원/개) → +1,000원  
　　　　• 예시: 시럽 추가 - 바닐라 시럽 [3]펌프 (+200원/펌프) → +600원  
　　　　• 고객이 수량 조절 버튼(-, +)으로 개수 지정  


**4.1.2.6.** 메뉴 이미지 업로드 규칙  

　1: 이미지 비율  
　　A. 1:1 정사각형 비율 Crop Tool 강제 실행  
　　B. 가이드 박스 안에서 위치·확대/축소 조정 후 [자르기] 필수  

　2: 이미지 목적  
　　A. 소비자 앱 메뉴 리스트에서 모든 썸네일이 통일된 규격으로 노출  
　　B. 메뉴 시각적 식별 용이성 향상  

　3: 이미지 제한 사항  
　　A. 파일 크기: 최대 5MB  
　　B. 파일 형식: JPG, PNG, WebP  
　　C. 해상도: 최소 500x500px 권장  

　4: 이미지 자동 최적화  
　　A. 업로드 시 자동 리사이징 및 압축  
　　B. CDN 연동으로 빠른 로딩 지원  
　　C. 썸네일 자동 생성  


**4.1.2.7.** 판매 상태 관리  

　1: 판매 상태 유형  
　　A. 판매중: 소비자 앱에 정상 노출 및 주문 가능  
　　B. 일시중지: 시즌 종료, 재료 소진 등으로 일시적 판매 중단  
　　C. 솔드아웃: 당일 품절 상태 (다음날 자동 해제 옵션 제공)  

　2: 상태 변경 방법  
　　A. 토글 스위치: 판매중 ↔ 일시중지 빠른 전환  
　　B. 드롭다운: 판매중/일시중지/솔드아웃 상세 선택  
　　C. 일괄 변경: 여러 메뉴 선택 후 상태 일괄 변경  

　3: 솔드아웃 자동 해제  
　　A. "다음날 자동 판매중으로 변경" 옵션 (기본 ON)  
　　B. 매일 오전 6시에 자동 해제  
　　C. 수동 해제도 가능  


**4.1.2.8.** 노출 시간대 설정  

　1: 시간대별 노출 설정  
　　A. 항상 노출 (기본값)  
　　B. 특정 시간대만 노출  
　　　　• 예시: 모닝 메뉴 (07:00 ~ 10:00)  
　　　　• 예시: 런치 메뉴 (11:00 ~ 14:00)  
　　　　• 예시: 야간 메뉴 (18:00 ~ 22:00)  

　2: 요일별 노출 설정  
　　A. 매일 노출 (기본값)  
　　B. 특정 요일만 노출  
　　　　• 예시: 주말 한정 메뉴 (토, 일요일만)  
　　　　• 예시: 평일 한정 메뉴 (월~금요일만)  

　3: 노출 시간 프리셋  
　　A. 프리셋 선택: 모닝, 런치, 디너, 종일, 주말 등  
　　B. 커스텀 설정: 시작 시간, 종료 시간, 요일 직접 입력  

　4: 시간대 외 메뉴 표시  
　　A. 숨김: 해당 시간대 외에는 메뉴 자체 미노출  
　　B. 비활성화: 메뉴는 보이지만 "준비 시간 외" 표시, 주문 불가  


**4.1.2.9.** 리워드 연동 설정  

　1: 리워드 사용 가능 여부  
　　A. 쿠폰/포인트/스탬프 사용 가능 여부 체크  
　　B. 리워드 제외 메뉴 설정 가능  

　2: 연동 설정  
　　A. 7-2-8-3. 설정과 자동 연동  
　　B. 어느 쪽에서 설정해도 동기화  


**4.1.2.10.** 메뉴 복사 기능  

　1: 메뉴 복사  
　　A. 기존 메뉴 선택 후 "복사" 버튼 클릭  
　　B. 모든 정보 복사 (이름, 가격, 옵션, 이미지 등)  
　　C. 메뉴명 자동 변경: "[원본 메뉴명] (복사본)"  
　　D. 복사 후 수정하여 유사 메뉴 빠르게 등록  

　2: 복사 활용 예시  
　　A. 아메리카노 → 아메리카노 (디카페인) 등록  
　　B. 동일 메뉴의 사이즈별 가격만 다른 경우  
　　C. 옵션만 다른 유사 메뉴 등록  


**4.1.2.11.** 일괄 등록 기능 (엑셀 업로드)  

　1: 엑셀 템플릿 다운로드  
　　A. 표준 엑셀 템플릿 다운로드 제공  
　　B. 템플릿에 메뉴 정보 일괄 입력  

　2: 엑셀 업로드  
　　A. 작성된 엑셀 파일 업로드  
　　B. 데이터 검증 및 미리보기  
　　C. 오류 항목 표시 및 수정 안내  

　3: 일괄 저장  
　　A. 검증 완료 후 일괄 저장  
　　B. 저장 결과 리포트 표시  
　　C. 실패 항목 별도 안내
