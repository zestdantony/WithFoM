**1.6.5.** 저장 처리  


**1.6.5.1.** 저장 버튼 클릭  

　1: 사전 검증  
　　A. 리워드 기능 Off 선택 시: 즉시 저장 가능  
　　B. 리워드 기능 On 선택 시  
　　　　a. 최소 1개 이상의 리워드 정보 입력 확인 (쿠폰, 포인트, 스탬프 중 1개)  
　　　　b. 각 리워드의 필수 항목 입력 완료 확인  
　　　　c. 입력값 유효성 검사 통과 확인  

　2: API 호출  
　　A. 엔드포인트: POST /api/partner/onboarding/save-reward-info  
　　B. 요청 데이터  

| 필드 | 타입 | 설명 |  
|---|---|---|  
| reward_enabled | boolean | 리워드 기능 활성화 여부 |  
| rewards | array | 리워드 정보 배열 (쿠폰/포인트) |  
| stamp_card | object | 스탬프 카드 정보 (스탬프 입력 시) |  
| source | string | 진입 경로 (onboarding) |  

　3: 저장 처리  
　　A. 리워드 정보 저장  
　　B. 리워드 기능 활성화 상태 업데이트  
　　C. 게임 기능 자동 활성화 (리워드 활성화 시)  


**1.6.5.2.** 저장 성공 시 처리  

　1: 성공 응답 수신  
　　A. 저장된 리워드 정보 반환  
　　B. 파트너 리워드 설정 상태 업데이트  

　2: 화면 전환  
　　A. 다음 가입 단계로 리다이렉트: /partner/onboarding/next-step  
　　B. 성공 토스트 메시지: "리워드 정보가 저장되었습니다"  


**1.6.5.3.** 저장 실패 시 처리  

　1: 에러 응답 처리  
　　A. 에러 메시지 표시  
　　B. 필드별 에러가 있는 경우: 해당 필드에 에러 표시  

　2: 재시도  
　　A. 입력 정보 수정 후 재시도 가능  
　　B. 에러 로그 기록
