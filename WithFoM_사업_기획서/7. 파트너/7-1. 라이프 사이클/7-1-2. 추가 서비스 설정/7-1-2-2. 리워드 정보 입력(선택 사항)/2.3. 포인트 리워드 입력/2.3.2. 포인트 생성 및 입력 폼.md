**2.3.2.** 포인트 쿠폰 생성 및 입력 폼  


**1.3.2.1.** 포인트 쿠폰 생성 인터페이스  

　1: 기본 입력박스  
　　A. 기본적으로 1개 포인트 쿠폰 입력박스 노출  
　　B. 총 6개까지 생성 가능 (할인 쿠폰과 합산)  

　2: 포인트 쿠폰 추가 버튼  
　　A. 위치: 입력박스 영역 우측 상단  
　　B. 버튼 텍스트: "포인트 쿠폰 추가"  
　　C. 클릭 시: 새로운 포인트 쿠폰 입력박스 1개 추가  
　　D. 최대 개수 도달 시: 버튼 비활성화 또는 "최대 6개까지 생성 가능합니다" 메시지 표시  


**1.3.2.2.** 포인트 쿠폰 입력박스 공통 구성  

　1: 박스 헤더  
　　A. 좌측 상단: N번째 포인트 쿠폰 숫자 표시 (예: "포인트 쿠폰 1", "포인트 쿠폰 2")  
　　B. 우측 상단: 삭제 버튼 (X 아이콘)  

　2: 입력 항목  
　　A. 포인트 쿠폰명 (필수)  
　　　　a. 입력 형식: 텍스트 입력창  
　　　　b. 예시: "포인트 500점", "포인트 3000점", "포인트 1000점"  
　　　　c. 최대 글자 수: 50자  
　　　　d. 권장 네이밍: "포인트 [금액]점" 형식 (게임 당첨 화면에 표시됨)  
　　B. 포인트 금액 (필수)  
　　　　a. 입력 형식: 숫자 입력창, 단위 "점" (1점 = 1원으로 사용)  
　　　　b. 설명: "게임 당첨 시 제공되는 포인트 금액 (결제 시 1점=1원으로 차감)"  
　　　　c. 예시: 500, 1000, 3000, 5000  
　　　　d. 최소값: 100점, 최대값: 50,000점  
　　C. 총 발행 수량 (선택)  
　　　　a. 입력 형식: 숫자 입력창  
　　　　b. 설명: "게임에서 당첨 가능한 총 수량 (미입력 시 무제한 발행)"  
　　　　c. 최소값: 1  
　　D. 유효 기간 (필수)  
　　　　a. 입력 형식: 날짜 선택기 (시작일, 종료일)  
　　　　b. 또는 기간 선택: 30일, 60일, 90일, 사용자 지정  
　　　　c. 유효 기간 미설정 체크박스: 체크 시 무기한 유효  


**1.3.2.3.** 포인트 쿠폰 사용 조건 설정  

　1: 최소 결제금액 조건  
　　A. 최소 결제 금액 (선택)  
　　　　a. 입력 형식: 숫자 입력창, 단위 "원"  
　　　　b. 설명: "이 금액 이상 결제 시 포인트 쿠폰 사용 가능"  
　　　　c. 예시: 5,000원 이상 결제 시 사용 가능  

　2: 메뉴 조건 (선택)  
　　A. 카테고리 조건  
　　　　a. 형식: 체크박스로 카테고리 선택  
　　　　b. 설명: "선택한 카테고리 메뉴에만 포인트 사용 가능"  
　　B. 단일 메뉴 조건  
　　　　a. 형식: 메뉴 선택 드롭다운  
　　　　b. 설명: "특정 메뉴에만 포인트 사용 가능"  

　3: 채널 조건 (선택)  
　　A. 옵션: 매장 / 테이크아웃 / 오더  
　　B. 형식: 체크박스로 복수 선택 가능  
　　C. 설명: "선택한 채널에서만 포인트 사용 가능"  

　4: 시간/요일 조건 (선택)  
　　A. 시간대 조건  
　　　　a. 형식: 시간 선택 (시작 시간, 종료 시간)  
　　　　b. 예시: "09:00 ~ 18:00"  
　　B. 요일 조건  
　　　　a. 형식: 체크박스로 요일 선택  
　　　　b. 예시: "월, 화, 수, 목, 금"  


**1.3.2.4.** 포인트 쿠폰 사용 제한 설정  

　1: 할인 적용 메뉴 제외  
　　A. 체크박스: "할인 적용 메뉴에는 포인트 사용 불가"  
　　B. 기본값: 체크 (권장)  
　　C. 설명: "메뉴 관리에서 할인 가격이 설정된 메뉴에는 포인트 적용 불가"  
　　D. 체크 시: 할인 메뉴 선택 시 "할인 메뉴에는 포인트를 사용할 수 없습니다" 안내 표시  

　2: 1일 사용 제한  
　　A. 1일 포인트 사용 가능 총액 (선택)  
　　　　a. 입력 형식: 숫자 입력창, 단위 "원"  
　　　　b. 설명: "고객 1인당 1일 사용 가능한 총 포인트 금액"  
　　　　c. 예시: 5,000원  

　3: 1회 결제당 사용 제한  
　　A. 1회 결제 기준 최대 사용 비율 (선택)  
　　　　a. 입력 형식: 숫자 입력창, 단위 "%"  
　　　　b. 설명: "1회 결제 금액 대비 사용 가능한 포인트 비율"  
　　　　c. 예시: "결제 금액의 50%까지 포인트 사용 가능"  
　　B. 1회 결제당 최대 사용 금액 (선택)  
　　　　a. 입력 형식: 숫자 입력창, 단위 "원"  
　　　　b. 설명: "1회 결제 시 사용 가능한 최대 포인트 금액"  
　　　　c. 예시: 3,000원  


**1.3.2.5.** 포인트 쿠폰과 할인 쿠폰의 차이  

　1: 포인트 쿠폰  
　　A. 형태: 포인트 금액 단위 (예: 500점, 1000점)  
　　B. 사용 방식: 1점 = 1원으로 결제 금액에서 차감  
　　C. 제공 방식: 게임 당첨 보상  

　2: 할인 쿠폰  
　　A. 형태: 금액 할인 또는 비율 할인 (예: 500원 할인, 10% 할인)  
　　B. 사용 방식: 할인 금액만큼 결제 금액에서 차감  
　　C. 제공 방식: 게임 당첨 보상, 프로모션 등  


**1.3.2.6.** 포인트 적립 안내 (향후 확장)  

　1: 현재 기능 안내  
　　A. 안내 메시지: "현재는 게임 당첨 보상으로 포인트 쿠폰을 제공합니다"  
　　B. 포인트 쿠폰: 게임에서 당첨 시 고객에게 제공되는 쿠폰 형태의 포인트  
　　C. 사용 방식: 결제 시 할인 쿠폰처럼 사용 (1점 = 1원)  

　2: 향후 확장 기능 안내  
　　A. 안내 메시지: "결제 금액 기반 자동 적립 기능은 향후 추가 예정입니다"  
　　B. 자세한 내용: `10. 향후 확장 로드맵\10-2. 신규 기능 추가\5. 포인트 적립 시스템.md`  
　　C. 예상 출시: 런칭 +6개월 후
