**1.1.1.** 화면 개요 및 진입  


**1.1.1.1.** 화면 목적  

　1: 가입 완료 직후 파트너에게 메뉴 정보 입력 기능을 소개하고 메뉴 등록을 독려하는 화면  

　2: 파트너는 오더형 또는 복합형 서비스를 선택한 경우에만 메뉴 정보 입력 가능  

　3: 메뉴 정보 입력은 선택 사항이므로 "건너뛰기" 버튼을 통해 다음 단계로 이동 가능  

　4: 메뉴 정보는 나중에 언제든지 수정할 수 있음을 안내  


**1.1.1.2.** 화면 진입 경로  

　1: 가입 플로우 내 진입  
　　A. 가입 완료 직후 또는 리워드 정보 입력 건너뛴 직후 → 메뉴 정보 입력 화면 자동 진입  
　　B. URL 경로: /partner/onboarding/menu-setup  

　2: 화면 노출 조건  
　　A. 파트너 가입 완료 직후  
　　B. 서비스 타입이 오더형 또는 복합형인 경우  
　　C. 리워드형 서비스 타입에는 이 화면이 노출되지 않음  


**1.1.1.3.** 전체 화면 레이아웃  

　1: 화면 구성 영역  

| 영역 | 위치 | 설명 |  
|---|---|---|  
| 헤더 영역 | 상단 | 화면 제목, 뒤로가기 버튼 |  
| 메뉴 입력 안내 영역 | 헤더 아래 | 메뉴 정보 입력 소개 및 안내 |  
| 카테고리 관리 영역 | 중앙 상단 | 카테고리 추가/수정/삭제 |  
| 메뉴 입력 영역 | 중앙 본문 | 메뉴 추가 및 정보 입력 폼 |  
| 하단 액션 영역 | 하단 고정 | 저장 및 건너뛰기 버튼 |  

　2: 반응형 레이아웃  
　　A. 모바일: 세로 스크롤, 하단 버튼 고정  
　　B. 태블릿/데스크톱: 중앙 정렬, 최대 너비 제한, 카테고리와 메뉴 입력 영역 분할 표시  


**1.1.1.4.** 헤더 영역 구성  

　1: 뒤로가기 버튼  
　　A. 위치: 좌측 상단  
　　B. 클릭 시: 이전 가입 단계로 이동  

　2: 화면 제목  
　　A. 텍스트: "메뉴 정보 입력" 또는 "메뉴 등록"  
　　B. 스타일: 중앙 정렬, 볼드  

　3: 단계 인디케이터 (선택 사항)  
　　A. 현재 가입 진행 단계 표시  
　　B. 예시: "5/5 단계" 또는 프로그레스 바  


**1.1.1.5.** 메뉴 입력 안내 영역  

　1: 안내 문구  
　　A. 메인 문구: "고객이 앱을 통해 주문할 수 있도록 메뉴 정보를 등록해주세요"  
　　B. 보조 문구: "메뉴 정보는 나중에 언제든지 수정할 수 있습니다"  

　2: 주요 기능 안내  
　　A. 카테고리별 메뉴 분류 가능  
　　B. 메뉴 옵션(사이즈, 온도, 당도 등) 설정 가능  
　　C. 메뉴 이미지 업로드 가능  
　　D. 리워드 사용 가능 여부 설정 가능
