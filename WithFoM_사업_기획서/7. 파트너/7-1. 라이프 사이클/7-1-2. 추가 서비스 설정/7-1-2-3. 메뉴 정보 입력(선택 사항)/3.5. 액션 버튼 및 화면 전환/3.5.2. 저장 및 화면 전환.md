**3.5.2.** 저장 및 화면 전환  


**3.5.2.1.** 저장 처리 프로세스  

　1: 저장 버튼 클릭 시 동작  
　　A. 1단계: 입력된 모든 메뉴 정보 유효성 검사  
　　B. 2단계: 검증 통과 시 서버에 저장 요청  
　　C. 3단계: 저장 성공 시 다음 단계로 이동  
　　D. 4단계: 저장 실패 시 에러 메시지 표시  

　2: 유효성 검사 항목  
　　A. 카테고리: 최소 1개 이상의 카테고리 생성 여부  
　　B. 메뉴명: 필수 입력 여부, 최대 길이 확인  
　　C. 가격: 필수 입력 여부, 숫자 형식 확인  
　　D. 카테고리 선택: 메뉴별 카테고리 선택 여부  

　3: 저장 중 상태 표시  
　　A. 로딩 스피너: 저장 버튼에 로딩 스피너 표시  
　　B. 버튼 비활성화: 저장 중에는 버튼 비활성화 (중복 저장 방지)  
　　C. 진행 상태: "저장 중..." 텍스트 표시 (선택 사항)  


**3.5.2.2.** 저장 성공 처리  

　1: 저장 성공 메시지  
　　A. 성공 알림: "메뉴 정보가 저장되었습니다" 토스트 메시지  
　　B. 저장된 메뉴 개수 표시: "N개의 메뉴가 저장되었습니다" (선택 사항)  

　2: 저장 성공 후 화면 전환  
　　A. 다음 단계로 자동 이동: 가입 완료 화면 또는 최종 단계  
　　B. URL 경로: /partner/onboarding/complete 또는 /partner/dashboard  
　　C. 전환 애니메이션: 부드러운 화면 전환 효과  

　3: 저장된 데이터 확인  
　　A. 저장된 메뉴 정보는 메뉴 관리 화면에서 확인 가능  
　　B. 나중에 수정: 메뉴 관리 화면에서 언제든지 수정 가능  


**3.5.2.3.** 저장 실패 처리  

　1: 에러 메시지 표시  
　　A. 네트워크 에러: "네트워크 연결을 확인해주세요"  
　　B. 서버 에러: "저장 중 오류가 발생했습니다. 잠시 후 다시 시도해주세요"  
　　C. 유효성 검사 실패: "필수 항목을 확인해주세요" 및 구체적인 항목 표시  

　2: 에러 발생 시 동작  
　　A. 화면 전환 없음: 에러 발생 시 현재 화면에 유지  
　　B. 재시도 가능: 저장 버튼 다시 클릭하여 재시도  
　　C. 에러 항목 하이라이트: 유효성 검사 실패 항목 강조 표시  


**3.5.2.4.** 건너뛰기 처리  

　1: 건너뛰기 버튼 클릭 시 동작  
　　A. 확인 모달 표시 (선택 사항): "메뉴 정보 입력을 건너뛰시겠습니까?"  
　　B. 확인 클릭: 다음 단계로 이동  
　　C. 취소 클릭: 현재 화면에 유지  

　2: 건너뛰기 후 화면 전환  
　　A. 다음 단계로 자동 이동: 가입 완료 화면 또는 최종 단계  
　　B. URL 경로: /partner/onboarding/complete 또는 /partner/dashboard  
　　C. 메뉴 정보는 저장되지 않음  

　3: 건너뛰기 후 안내  
　　A. 안내 문구: "메뉴 정보는 나중에 메뉴 관리 화면에서 입력할 수 있습니다"  
　　B. 바로가기 링크: 메뉴 관리 화면으로 이동하는 링크 제공 (선택 사항)  


**3.5.2.5.** 뒤로가기 처리  

　1: 뒤로가기 버튼 클릭 시 동작  
　　A. 이전 가입 단계로 이동  
　　B. 입력된 메뉴 정보는 임시 저장 (선택 사항)  
　　C. 확인 모달 없이 즉시 이동  

　2: 뒤로가기 후 재진입  
　　A. 이전에 입력한 메뉴 정보 복원 (임시 저장된 경우)  
　　B. 입력 폼 상태 복원: 카테고리, 메뉴 목록 등  


**3.5.2.6.** 화면 전환 애니메이션  

　1: 전환 효과  
　　A. 저장 성공 후: 부드러운 슬라이드 전환  
　　B. 건너뛰기 후: 부드러운 페이드 전환  
　　C. 뒤로가기: 슬라이드 뒤로 전환  

　2: 전환 시간  
　　A. 기본 전환 시간: 300ms  
　　B. 사용자 경험: 자연스러운 전환 효과로 부드러운 느낌 제공
