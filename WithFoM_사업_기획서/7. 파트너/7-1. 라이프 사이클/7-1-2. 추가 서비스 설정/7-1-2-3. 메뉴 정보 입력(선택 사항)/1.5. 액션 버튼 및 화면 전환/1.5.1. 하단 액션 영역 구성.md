**1.5.1.** 하단 액션 영역 구성  


**1.5.1.1.** 하단 액션 영역 레이아웃  

　1: 영역 위치  
　　A. 화면 하단 고정 영역  
　　B. 모바일: 하단 고정, 스크롤 시에도 항상 표시  
　　C. 태블릿/데스크톱: 하단 고정 또는 화면 하단에 배치  

　2: 버튼 배치  
　　A. 좌측: "건너뛰기" 버튼 (선택 사항)  
　　B. 우측: "저장" 버튼 (주요 액션)  
　　C. 버튼 간격: 적절한 간격 유지  

　3: 반응형 레이아웃  
　　A. 모바일: 버튼 전체 너비, 세로 배치 또는 가로 배치  
　　B. 태블릿/데스크톱: 버튼 최대 너비 제한, 중앙 정렬  


**1.5.1.2.** 저장 버튼  

　1: 저장 버튼 스타일  
　　A. 버튼 텍스트: "저장" 또는 "메뉴 정보 저장"  
　　B. 버튼 색상: 주요 액션 색상 (예: 파란색, 초록색)  
　　C. 버튼 크기: 충분한 크기로 터치하기 쉽게  

　2: 저장 버튼 상태  
　　A. 활성화 상태: 입력된 정보가 유효할 때 활성화  
　　B. 비활성화 상태: 필수 항목 미입력 시 비활성화  
　　C. 로딩 상태: 저장 중일 때 로딩 스피너 표시  

　3: 저장 버튼 클릭 동작  
　　A. 클릭 시: 입력된 모든 메뉴 정보 검증  
　　B. 검증 통과: 메뉴 정보 저장 및 다음 단계로 이동  
　　C. 검증 실패: 에러 메시지 표시 및 수정 유도  


**1.5.1.3.** 건너뛰기 버튼  

　1: 건너뛰기 버튼 스타일  
　　A. 버튼 텍스트: "건너뛰기" 또는 "나중에 입력"  
　　B. 버튼 색상: 보조 액션 색상 (예: 회색, 투명)  
　　C. 버튼 스타일: 텍스트 버튼 또는 아웃라인 버튼  

　2: 건너뛰기 버튼 클릭 동작  
　　A. 클릭 시: 확인 모달 표시 (선택 사항)  
　　B. 확인 모달: "메뉴 정보 입력을 건너뛰시겠습니까? 나중에 언제든지 입력할 수 있습니다."  
　　C. 확인 클릭: 다음 단계로 이동 (메뉴 정보 없이 진행)  

　3: 건너뛰기 후 처리  
　　A. 메뉴 정보는 저장되지 않음  
　　B. 다음 단계로 이동 (가입 완료 또는 최종 단계)  
　　C. 나중에 메뉴 관리 화면에서 메뉴 정보 입력 가능  


**1.5.1.4.** 버튼 상태 관리  

　1: 저장 버튼 활성화 조건  
　　A. 최소 1개 이상의 카테고리 생성  
　　B. 최소 1개 이상의 메뉴 입력 (선택 사항, 권장)  
　　C. 입력된 메뉴의 필수 항목(카테고리, 메뉴명, 가격) 모두 입력  

　2: 저장 버튼 비활성화 조건  
　　A. 필수 항목 미입력 시 비활성화  
　　B. 유효성 검사 실패 시 비활성화  
　　C. 저장 중일 때 비활성화 (중복 저장 방지)  

　3: 버튼 툴팁 또는 안내 문구  
　　A. 비활성화 상태: "필수 항목을 입력해주세요" 툴팁 표시  
　　B. 활성화 상태: "저장 버튼을 클릭하여 메뉴 정보를 저장하세요" 안내 (선택 사항)  


**1.5.1.5.** 하단 액션 영역 스타일  

　1: 배경 및 테두리  
　　A. 배경: 흰색 또는 반투명 배경  
　　B. 상단 테두리: 구분선 표시 (선택 사항)  
　　C. 그림자: 상단 그림자 효과 (선택 사항)  

　2: 버튼 간격 및 정렬  
　　A. 버튼 간격: 적절한 간격 유지 (예: 16px)  
　　B. 버튼 정렬: 좌우 정렬 또는 중앙 정렬  
　　C. 버튼 크기: 동일한 높이, 너비는 내용에 맞게  


**1.5.1.6.** 모바일 최적화  

　1: 터치 친화적 디자인  
　　A. 버튼 최소 크기: 44x44px 이상 (터치 영역)  
　　B. 버튼 간격: 충분한 간격으로 오클릭 방지  
　　C. 하단 고정: 스크롤 시에도 항상 접근 가능  

　2: 키보드 처리  
　　A. 키보드 노출 시: 하단 액션 영역이 키보드 위로 이동  
　　B. 키보드 숨김 시: 하단 액션 영역 원래 위치로 복귀
