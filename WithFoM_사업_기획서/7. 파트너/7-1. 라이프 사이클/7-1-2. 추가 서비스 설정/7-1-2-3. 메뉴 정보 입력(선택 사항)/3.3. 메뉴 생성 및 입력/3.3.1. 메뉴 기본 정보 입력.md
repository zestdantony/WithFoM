**3.3.1.** 메뉴 기본 정보 입력  


**3.3.1.1.** 메뉴 추가 버튼  

　1: 메뉴 추가 버튼 위치  
　　A. 메뉴 입력 영역 상단 우측  
　　B. 버튼 텍스트: "메뉴 추가" 또는 "+ 메뉴 추가"  
　　C. 클릭 시: 메뉴 입력 폼 표시  

　2: 메뉴 추가 조건  
　　A. 최소 1개 이상의 카테고리가 생성되어 있어야 함  
　　B. 카테고리가 없을 경우: "먼저 카테고리를 생성해주세요" 안내 표시  


**3.3.1.2.** 메뉴 기본 정보 입력 폼  

　1: 메뉴명 입력 (필수)  
　　A. 입력 필드: 메뉴명 (예: 아메리카노)  
　　B. 필수 입력: 메뉴명은 필수 항목  
　　C. 최대 길이: 20자  
　　D. 중복 체크: 동일한 메뉴명이 이미 존재하는지 실시간 확인  
　　E. 입력 예시: "아메리카노", "카페라떼", "초콜릿 케이크" 등  

　2: 메뉴 설명 입력 (선택)  
　　A. 입력 필드: 메뉴 설명  
　　B. 선택 입력: 메뉴 설명은 선택 사항  
　　C. 최대 길이: 100자  
　　D. 입력 예시: "진한 에스프레소에 물을 더한 커피", "부드러운 우유와 에스프레소의 조화" 등  

　3: 카테고리 선택 (필수)  
　　A. 드롭다운 선택: 생성된 카테고리 목록에서 선택  
　　B. 필수 선택: 카테고리는 필수 항목  
　　C. 기본값: 첫 번째 카테고리 또는 "카테고리 선택" 플레이스홀더  


**3.3.1.3.** 메뉴명 국문/영문 입력  

　1: 국문 메뉴명  
　　A. 기본 입력 필드: 국문 메뉴명 (예: 아메리카노)  
　　B. 필수 입력: 국문 메뉴명은 필수 항목  

　2: 영문 메뉴명 (선택)  
　　A. 추가 입력 필드: 영문 메뉴명 (예: Americano)  
　　B. 선택 입력: 영문 메뉴명은 선택 사항  
　　C. 최대 길이: 20자  
　　D. 입력 시: 소비자 앱에서 영문명도 함께 표시 가능  


**3.3.1.4.** 판매 상태 설정  

　1: 판매 상태 (기본값: 판매중)  
　　A. 판매중: 소비자 앱에 정상 노출 및 주문 가능  
　　B. 일시중지: 시즌 종료, 재료 소진 등 일시적 판매 중단  
　　C. 솔드아웃: 당일 품절 상태  

　2: 상태 선택 방법  
　　A. 토글 스위치: 판매중 ↔ 일시중지 빠른 전환  
　　B. 드롭다운: 3가지 상태 중 선택  

　3: 솔드아웃 자동 해제 옵션  
　　A. "다음날 자동 판매중으로 변경" 체크박스 (기본 ON)  
　　B. 매일 오전 6시에 자동 해제  
　　C. 수동 해제도 가능  


**3.3.1.5.** 노출 시간대 설정  

　1: 노출 시간 유형  
　　A. 항상 노출 (기본값): 영업 시간 내 항상 노출  
　　B. 시간대 지정: 특정 시간대에만 노출  

　2: 시간대 설정 방법  
　　A. 프리셋 선택 (기본)  
　　　　• 모닝 (07:00 ~ 10:00)  
　　　　• 런치 (11:00 ~ 14:00)  
　　　　• 티타임 (14:00 ~ 17:00)  
　　　　• 디너 (17:00 ~ 21:00)  
　　B. 프리셋 선택 후 시간 수정  
　　　　• 프리셋 선택 시 해당 시간이 시작/종료 입력 필드에 자동 입력  
　　　　• 시작 시간: 시간 선택 또는 직접 입력 (00:00 ~ 23:59)  
　　　　• 종료 시간: 시간 선택 또는 직접 입력 (00:00 ~ 23:59)  
　　　　• 수정된 시간은 "커스텀"으로 표시  

　3: 요일별 노출 설정  
　　A. 매일 노출 (기본값)  
　　B. 요일 선택: 월, 화, 수, 목, 금, 토, 일 중 선택  
　　C. 프리셋: 평일만, 주말만, 매일  

　4: 시간대 외 메뉴 표시 방식  
　　A. 숨김: 해당 시간 외 메뉴 미노출  
　　B. 비활성화: 메뉴 노출되나 "준비 시간 외" 표시, 주문 불가  


**3.3.1.6.** 추가 정보 설정  

　1: 인기 메뉴 표시 (선택)  
　　A. "인기" 체크박스 선택 시 인기 뱃지 표시  
　　B. 소비자 앱에서 "인기" 뱃지와 함께 노출  
　　C. 인기 메뉴 필터링에 포함  
　　D. 최대 표시 개수: 무제한 (파트너 재량)  

　2: 알레르기 정보 (선택)  
　　A. 알레르기 유발 성분 다중 선택  
　　B. 선택 가능 항목:  
　　　　a. 우유  
　　　　b. 대두  
　　　　c. 밀  
　　　　d. 땅콩  
　　　　e. 견과류  
　　　　f. 계란  
　　　　g. 갑각류  
　　　　h. 조개류  
　　　　i. 복숭아  
　　　　j. 토마토  
　　　　k. 기타 (직접 입력)  
　　C. 소비자 앱에서 알레르기 정보 표시  
　　D. 알레르기 필터링 검색에 활용  


**3.3.1.7.** 메뉴 입력 폼 레이아웃  

　1: 입력 폼 구성  
　　A. 상단: 메뉴명 입력 필드 (필수)  
　　B. 중간: 메뉴 설명 입력 필드 (선택)  
　　C. 중간: 카테고리 선택 드롭다운 (필수)  
　　D. 하단: 판매 상태, 노출 시간대, 인기 표시, 알레르기 설정  

　2: 입력 필드 스타일  
　　A. 필수 항목: 레이블에 "*" 표시 또는 "필수" 텍스트  
　　B. 선택 항목: 레이블에 "(선택)" 표시  
　　C. 플레이스홀더: 입력 예시 또는 안내 문구 표시  


**3.3.1.8.** 메뉴 입력 유효성 검사  

　1: 실시간 유효성 검사  
　　A. 메뉴명 입력 시: 최대 길이 초과 시 경고 표시  
　　B. 메뉴명 중복 검사: 동일한 메뉴명이 이미 존재하면 경고 표시  
　　　　• 입력 중 실시간으로 중복 여부 확인  
　　　　• 중복 시: "이미 존재하는 메뉴명입니다" 에러 메시지  
　　　　• 입력 필드 하단에 빨간색 경고 표시  
　　C. 카테고리 선택 시: 미선택 시 경고 표시  
　　D. 저장 버튼 클릭 시: 필수 항목 미입력 시 경고 표시  

　2: 에러 메시지  
　　A. 메뉴명 미입력: "메뉴명을 입력해주세요"  
　　B. 메뉴명 중복: "이미 존재하는 메뉴명입니다"  
　　C. 카테고리 미선택: "카테고리를 선택해주세요"  
　　D. 메뉴명 길이 초과: "메뉴명은 20자 이하로 입력해주세요"  
　　E. 노출 시간 오류: "시작 시간은 종료 시간보다 이전이어야 합니다"  


**3.3.1.9.** 메뉴 입력 폼 저장  

　1: 저장 버튼  
　　A. 위치: 메뉴 입력 폼 하단  
　　B. 버튼 텍스트: "저장" 또는 "메뉴 추가"  
　　C. 클릭 시: 입력된 정보 검증 후 저장  

　2: 저장 후 처리  
　　A. 저장 성공: 메뉴 목록에 추가된 메뉴 표시  
　　B. 저장 실패: 에러 메시지 표시 및 재입력 유도  
　　C. 저장 후 입력 폼 초기화 또는 다음 메뉴 추가 모드로 전환  


**3.3.1.10.** 메뉴 복사 기능  

　1: 복사 버튼 위치  
　　A. 메뉴 목록에서 각 메뉴 우측에 "복사" 버튼  
　　B. 또는 메뉴 상세 화면에서 "메뉴 복사" 버튼  

　2: 복사 프로세스  
　　A. 복사 버튼 클릭  
　　B. 모든 정보 복사 (메뉴명, 설명, 가격, 옵션, 이미지 등)  
　　C. 메뉴명 자동 변경: "[원본 메뉴명] (복사본)"  
　　D. 새 메뉴로 입력 폼 표시 (수정 모드)  

　3: 복사 후 처리  
　　A. 복사된 메뉴 수정 후 저장  
　　B. 저장 시 새로운 메뉴로 등록  
　　C. 원본 메뉴는 그대로 유지  

　4: 복사 활용 사례  
　　A. 유사 메뉴 빠른 등록 (예: 아메리카노 → 디카페인 아메리카노)  
　　B. 옵션만 다른 메뉴 등록  
　　C. 가격만 다른 메뉴 등록 (예: 사이즈별 별도 메뉴)  


**3.3.1.11.** 메뉴 순서 조정  

　1: 드래그 앤 드롭 정렬  
　　A. 메뉴 목록에서 메뉴 항목을 드래그하여 순서 변경  
　　B. 드래그 핸들: 메뉴 항목 좌측에 드래그 아이콘 표시  
　　C. 드롭 시 순서 즉시 반영  

　2: 위/아래 이동 버튼  
　　A. 메뉴 항목 우측에 위/아래 이동 버튼  
　　B. 클릭 시 한 칸씩 이동  
　　C. 맨 위/맨 아래 이동 옵션 (선택 사항)  

　3: 순서 저장  
　　A. 자동 저장: 순서 변경 시 자동 저장  
　　B. 또는 "순서 저장" 버튼 클릭으로 명시적 저장  
　　C. 저장된 순서는 소비자 앱 메뉴 노출에 반영
