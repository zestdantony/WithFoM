**1.6.2.** 카테고리 관리  


**1.6.2.1.** 카테고리 생성 API  

　1: 엔드포인트: POST /api/partner/onboarding/categories  
　　A. 요청 데이터  
　　　　a. name: 카테고리명 (필수, 최대 20자)  
　　　　b. display_order: 노출 순서 (선택, 기본값: 마지막 순서)  
　　　　c. is_visible: 노출 여부 (선택, 기본값: true)  

　　B. 응답 데이터  
　　　　a. category_id: 생성된 카테고리 ID  
　　　　b. name: 카테고리명  
　　　　c. display_order: 노출 순서  
　　　　d. is_visible: 노출 여부  
　　　　e. menu_count: 해당 카테고리의 메뉴 개수 (0)  

　2: 성공 처리  
　　A. 카테고리 목록에 새 카테고리 추가  
　　B. 카테고리 입력 폼 초기화  
　　C. 성공 토스트 메시지 표시 (선택 사항)  

　3: 에러 처리  
　　A. 중복 카테고리명: "이미 존재하는 카테고리명입니다" 에러 메시지  
　　B. 네트워크 에러: "네트워크 연결을 확인해주세요" 에러 메시지  
　　C. 서버 에러: "카테고리 생성 중 오류가 발생했습니다" 에러 메시지  


**1.6.2.2.** 카테고리 수정 API  

　1: 엔드포인트: PUT /api/partner/onboarding/categories/{category_id}  
　　A. 요청 데이터  
　　　　a. name: 카테고리명 (선택, 수정할 경우만)  
　　　　b. display_order: 노출 순서 (선택)  
　　　　c. is_visible: 노출 여부 (선택)  

　　B. 응답 데이터  
　　　　a. category_id: 카테고리 ID  
　　　　b. 업데이트된 카테고리 정보  

　2: 성공 처리  
　　A. 카테고리 목록에서 해당 카테고리 정보 업데이트  
　　B. 수정 모드 종료  
　　C. 성공 토스트 메시지 표시 (선택 사항)  


**1.6.2.3.** 카테고리 삭제 API  

　1: 엔드포인트: DELETE /api/partner/onboarding/categories/{category_id}  
　　A. 요청 데이터: 없음 (URL 파라미터로 category_id 전달)  

　　B. 응답 데이터  
　　　　a. success: 삭제 성공 여부  
　　　　b. message: 삭제 결과 메시지  

　2: 성공 처리  
　　A. 카테고리 목록에서 해당 카테고리 제거  
　　B. 해당 카테고리의 메뉴는 "미분류" 카테고리로 자동 이동  
　　C. 성공 토스트 메시지 표시  

　3: 에러 처리  
　　A. 삭제 실패: "카테고리 삭제 중 오류가 발생했습니다" 에러 메시지  


**1.6.2.4.** 카테고리 순서 변경 API  

　1: 엔드포인트: PUT /api/partner/onboarding/categories/reorder  
　　A. 요청 데이터  
　　　　a. category_orders: 카테고리 ID와 순서 배열  
　　　　　　예: [{category_id: 1, display_order: 1}, {category_id: 2, display_order: 2}]  

　　B. 응답 데이터  
　　　　a. success: 순서 변경 성공 여부  

　2: 성공 처리  
　　A. 카테고리 목록 순서 업데이트  
　　B. UI에 변경된 순서 반영
