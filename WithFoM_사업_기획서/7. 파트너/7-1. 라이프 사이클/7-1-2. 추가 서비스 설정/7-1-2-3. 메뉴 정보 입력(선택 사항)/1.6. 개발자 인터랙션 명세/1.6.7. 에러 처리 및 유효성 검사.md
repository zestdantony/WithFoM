**1.6.7.** 에러 처리 및 유효성 검사  


**1.6.7.1.** 클라이언트 측 유효성 검사  

　1: 실시간 유효성 검사  
　　A. 메뉴명 입력 시: 최대 길이(50자) 초과 시 경고 표시  
　　B. 가격 입력 시: 숫자 형식 확인, 음수 입력 방지  
　　C. 카테고리 선택 시: 미선택 시 경고 표시  
　　D. 이미지 업로드 시: 파일 크기(5MB), 형식(JPG/PNG/WebP) 확인  

　2: 저장 버튼 클릭 시 검증  
　　A. 필수 항목 확인: 카테고리, 메뉴명, 가격  
　　B. 형식 검증: 숫자 형식, 문자열 길이 등  
　　C. 검증 실패 시: 에러 메시지 표시 및 해당 필드 하이라이트  

　3: 에러 메시지 표시  
　　A. 필드별 에러 메시지: 각 입력 필드 하단에 에러 메시지 표시  
　　B. 전체 에러 메시지: 상단에 요약 에러 메시지 표시 (선택 사항)  


**1.6.7.2.** 서버 측 유효성 검사  

　1: 서버 응답 에러 처리  
　　A. 400 Bad Request: "잘못된 요청입니다. 입력 정보를 확인해주세요"  
　　B. 401 Unauthorized: "인증이 필요합니다. 다시 로그인해주세요"  
　　C. 403 Forbidden: "권한이 없습니다"  
　　D. 404 Not Found: "요청한 리소스를 찾을 수 없습니다"  
　　E. 500 Internal Server Error: "서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요"  

　2: 비즈니스 로직 에러  
　　A. 중복 카테고리명: "이미 존재하는 카테고리명입니다"  
　　B. 존재하지 않는 카테고리: "선택한 카테고리가 존재하지 않습니다"  
　　C. 가격 범위 초과: "가격은 0원 이상이어야 합니다"  


**1.6.7.3.** 네트워크 에러 처리  

　1: 네트워크 연결 실패  
　　A. 오프라인 상태 감지: "네트워크 연결을 확인해주세요" 메시지  
　　B. 재시도 버튼: 네트워크 연결 후 재시도 가능  
　　C. 오프라인 모드: 입력한 데이터 임시 저장 (선택 사항)  

　2: 타임아웃 처리  
　　A. 요청 타임아웃: "요청 시간이 초과되었습니다. 다시 시도해주세요"  
　　B. 자동 재시도: 일정 횟수(예: 3회)까지 자동 재시도 (선택 사항)  


**1.6.7.4.** 이미지 업로드 에러 처리  

　1: 파일 크기 초과  
　　A. 에러 메시지: "파일 크기는 5MB 이하여야 합니다"  
　　B. 해결 방법 안내: "더 작은 크기의 이미지를 선택해주세요"  

　2: 파일 형식 오류  
　　A. 에러 메시지: "지원하는 이미지 형식은 JPG, PNG, WebP입니다"  
　　B. 해결 방법 안내: "지원하는 형식의 이미지를 선택해주세요"  

　3: 이미지 업로드 실패  
　　A. 네트워크 에러: "이미지 업로드 중 네트워크 오류가 발생했습니다"  
　　B. 서버 에러: "이미지 업로드 중 오류가 발생했습니다. 다시 시도해주세요"  
　　C. 재시도 버튼: 이미지 업로드 재시도 가능  


**1.6.7.5.** 에러 복구 및 재시도  

　1: 에러 발생 시 사용자 안내  
　　A. 명확한 에러 메시지 표시  
　　B. 해결 방법 안내 (가능한 경우)  
　　C. 재시도 버튼 제공 (가능한 경우)  

　2: 자동 복구 시도  
　　A. 네트워크 에러: 자동 재연결 시도 (선택 사항)  
　　B. 일시적 서버 에러: 일정 시간 후 자동 재시도 (선택 사항)  

　3: 에러 로그 기록  
　　A. 클라이언트 측 에러 로그: 개발자 디버깅을 위한 에러 로그 기록  
　　B. 서버 측 에러 로그: 서버 로그에 에러 정보 기록
