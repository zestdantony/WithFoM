**1.6.4.** 옵션 그룹 관리  


**1.6.4.1.** 옵션 그룹 생성 API  

　1: 엔드포인트: POST /api/partner/onboarding/option-groups  
　　A. 요청 데이터  
　　　　a. name: 옵션 그룹명 (필수, 최대 30자)  
　　　　b. option_type: 옵션 유형 (필수, single | multiple | quantity)  
　　　　c. options: 옵션 항목 배열 (필수, 최소 1개)  
　　　　　　각 옵션 항목:  
　　　　　　- name: 옵션 항목명 (필수)  
　　　　　　- additional_price: 추가 금액 (필수, 숫자)  
　　　　　　- display_order: 노출 순서 (선택)  

　　B. 응답 데이터  
　　　　a. option_group_id: 생성된 옵션 그룹 ID  
　　　　b. name: 옵션 그룹명  
　　　　c. option_type: 옵션 유형  
　　　　d. options: 옵션 항목 배열  

　2: 성공 처리  
　　A. 옵션 그룹 목록에 새 옵션 그룹 추가  
　　B. 옵션 그룹 입력 폼 초기화  
　　C. 성공 토스트 메시지 표시 (선택 사항)  

　3: 에러 처리  
　　A. 옵션 항목 미입력: "최소 1개 이상의 옵션 항목을 입력해주세요" 에러 메시지  
　　B. 네트워크 에러: "네트워크 연결을 확인해주세요" 에러 메시지  


**1.6.4.2.** 옵션 그룹 수정 API  

　1: 엔드포인트: PUT /api/partner/onboarding/option-groups/{option_group_id}  
　　A. 요청 데이터: 옵션 그룹 생성 API와 동일 (모든 필드 선택)  

　　B. 응답 데이터: 업데이트된 옵션 그룹 정보  

　2: 성공 처리  
　　A. 옵션 그룹 목록에서 해당 옵션 그룹 정보 업데이트  
　　B. 수정 모드 종료  
　　C. 해당 옵션 그룹을 사용하는 메뉴에 자동 반영 (선택 사항)  


**1.6.4.3.** 옵션 그룹 삭제 API  

　1: 엔드포인트: DELETE /api/partner/onboarding/option-groups/{option_group_id}  
　　A. 요청 데이터: 없음 (URL 파라미터로 option_group_id 전달)  

　　B. 응답 데이터  
　　　　a. success: 삭제 성공 여부  
　　　　b. message: 삭제 결과 메시지  
　　　　c. affected_menus: 해당 옵션 그룹을 사용하는 메뉴 ID 배열 (선택)  

　2: 성공 처리  
　　A. 옵션 그룹 목록에서 해당 옵션 그룹 제거  
　　B. 해당 옵션 그룹을 사용하는 메뉴에서 옵션 자동 제거  
　　C. 성공 토스트 메시지 표시  


**1.6.4.4.** 옵션 그룹 일괄 적용 API  

　1: 엔드포인트: POST /api/partner/onboarding/option-groups/apply  
　　A. 요청 데이터  
　　　　a. option_group_id: 옵션 그룹 ID (필수)  
　　　　b. menu_ids: 메뉴 ID 배열 (필수, 최소 1개)  

　　B. 응답 데이터  
　　　　a. success: 적용 성공 여부  
　　　　b. applied_count: 적용된 메뉴 개수  

　2: 성공 처리  
　　A. 선택한 모든 메뉴에 옵션 그룹 적용  
　　B. 메뉴 목록에서 옵션 그룹 적용 상태 업데이트  
　　C. 성공 토스트 메시지 표시
