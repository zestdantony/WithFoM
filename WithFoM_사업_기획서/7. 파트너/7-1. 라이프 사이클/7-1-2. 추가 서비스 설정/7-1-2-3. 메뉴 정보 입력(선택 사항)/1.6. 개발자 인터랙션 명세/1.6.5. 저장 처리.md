**1.6.5.** 저장 처리  


**1.6.5.1.** 전체 메뉴 정보 저장 API  

　1: 엔드포인트: POST /api/partner/onboarding/menus/save-all  
　　A. 요청 데이터  
　　　　a. categories: 카테고리 정보 배열 (선택, 생성/수정된 카테고리만)  
　　　　b. menus: 메뉴 정보 배열 (선택, 생성/수정된 메뉴만)  
　　　　c. option_groups: 옵션 그룹 정보 배열 (선택, 생성/수정된 옵션 그룹만)  

　　B. 응답 데이터  
　　　　a. success: 저장 성공 여부  
　　　　b. saved_categories: 저장된 카테고리 개수  
　　　　c. saved_menus: 저장된 메뉴 개수  
　　　　d. saved_option_groups: 저장된 옵션 그룹 개수  

　2: 저장 전 유효성 검사  
　　A. 클라이언트 측 검증: 필수 항목 확인, 형식 검증  
　　B. 서버 측 검증: 중복 체크, 데이터 무결성 검증  

　3: 성공 처리  
　　A. 저장 성공 토스트 메시지 표시  
　　B. 다음 단계로 화면 전환  
　　C. 저장된 데이터는 메뉴 관리 화면에서 확인 가능  


**1.6.5.2.** 저장 상태 관리  

　1: 저장 중 상태  
　　A. 저장 버튼 비활성화  
　　B. 로딩 스피너 표시  
　　C. "저장 중..." 텍스트 표시 (선택 사항)  

　2: 저장 완료 상태  
　　A. 저장 버튼 활성화 (다음 작업을 위해)  
　　B. 로딩 스피너 숨김  
　　C. 성공 메시지 표시  

　3: 저장 실패 상태  
　　A. 저장 버튼 활성화 (재시도 가능)  
　　B. 로딩 스피너 숨김  
　　C. 에러 메시지 표시  


**1.6.5.3.** 임시 저장 (자동 저장)  

　1: 임시 저장 기능 (선택 사항)  
　　A. 일정 시간(예: 30초)마다 자동으로 임시 저장  
　　B. 또는 입력 필드 변경 시 자동 저장  
　　C. 엔드포인트: POST /api/partner/onboarding/menus/auto-save  

　2: 임시 저장 데이터 복원  
　　A. 화면 재진입 시 임시 저장된 데이터 자동 복원  
　　B. 사용자 확인 후 복원 또는 자동 복원  

　3: 임시 저장 데이터 삭제  
　　A. 최종 저장 완료 시 임시 저장 데이터 삭제  
　　B. 또는 일정 기간(예: 7일) 후 자동 삭제
