**1.6.3.** 메뉴 생성 및 관리  


**1.6.3.1.** 메뉴 생성 API  

　1: 엔드포인트: POST /api/partner/onboarding/menus  
　　A. 요청 데이터  
　　　　a. name: 메뉴명 (필수, 최대 50자)  
　　　　b. name_en: 영문 메뉴명 (선택, 최대 50자)  
　　　　c. description: 메뉴 설명 (선택, 최대 200자)  
　　　　d. category_id: 카테고리 ID (필수)  
　　　　e. base_price: 기본 가격 (필수, 숫자)  
　　　　f. discount_price: 할인 가격 (선택, 숫자)  
　　　　g. image_url: 이미지 URL (선택, 이미지 업로드 후 반환된 URL)  
　　　　h. option_group_ids: 옵션 그룹 ID 배열 (선택)  
　　　　i. is_reward_available: 리워드 사용 가능 여부 (선택, 기본값: true)  
　　　　j. is_visible: 노출 여부 (선택, 기본값: true)  

　　B. 응답 데이터  
　　　　a. menu_id: 생성된 메뉴 ID  
　　　　b. name: 메뉴명  
　　　　c. category_id: 카테고리 ID  
　　　　d. base_price: 기본 가격  
　　　　e. image_url: 이미지 URL  
　　　　f. created_at: 생성 일시  

　2: 성공 처리  
　　A. 메뉴 목록에 새 메뉴 추가  
　　B. 메뉴 입력 폼 초기화  
　　C. 성공 토스트 메시지 표시 (선택 사항)  

　3: 에러 처리  
　　A. 필수 항목 미입력: "필수 항목을 입력해주세요" 에러 메시지  
　　B. 가격 형식 오류: "가격은 숫자로 입력해주세요" 에러 메시지  
　　C. 네트워크 에러: "네트워크 연결을 확인해주세요" 에러 메시지  


**1.6.3.2.** 메뉴 수정 API  

　1: 엔드포인트: PUT /api/partner/onboarding/menus/{menu_id}  
　　A. 요청 데이터: 메뉴 생성 API와 동일 (모든 필드 선택)  

　　B. 응답 데이터: 업데이트된 메뉴 정보  

　2: 성공 처리  
　　A. 메뉴 목록에서 해당 메뉴 정보 업데이트  
　　B. 수정 모드 종료  
　　C. 성공 토스트 메시지 표시  


**1.6.3.3.** 메뉴 삭제 API  

　1: 엔드포인트: DELETE /api/partner/onboarding/menus/{menu_id}  
　　A. 요청 데이터: 없음 (URL 파라미터로 menu_id 전달)  

　　B. 응답 데이터  
　　　　a. success: 삭제 성공 여부  
　　　　b. message: 삭제 결과 메시지  

　2: 성공 처리  
　　A. 메뉴 목록에서 해당 메뉴 제거  
　　B. 성공 토스트 메시지 표시  

　3: 에러 처리  
　　A. 삭제 실패: "메뉴 삭제 중 오류가 발생했습니다" 에러 메시지  


**1.6.3.4.** 메뉴 이미지 업로드 API  

　1: 엔드포인트: POST /api/partner/onboarding/menus/upload-image  
　　A. 요청 데이터 (multipart/form-data)  
　　　　a. image: 이미지 파일 (필수, 최대 5MB)  

　　B. 응답 데이터  
　　　　a. image_url: 업로드된 이미지 URL  
　　　　b. thumbnail_url: 썸네일 이미지 URL (선택)  

　2: 이미지 Crop 처리  
　　A. 클라이언트에서 1:1 비율로 Crop 후 업로드  
　　B. 또는 서버에서 Crop 처리 (서버 설정에 따라)  

　3: 성공 처리  
　　A. 이미지 미리보기 표시  
　　B. image_url을 메뉴 생성/수정 API에 포함하여 전송  

　4: 에러 처리  
　　A. 파일 크기 초과: "파일 크기는 5MB 이하여야 합니다" 에러 메시지  
　　B. 파일 형식 오류: "지원하는 이미지 형식은 JPG, PNG, WebP입니다" 에러 메시지  


**1.6.3.5.** 메뉴 일괄 저장 API  

　1: 엔드포인트: POST /api/partner/onboarding/menus/batch-save  
　　A. 요청 데이터  
　　　　a. menus: 메뉴 정보 배열  
　　　　　　각 메뉴 객체는 메뉴 생성 API의 요청 데이터 구조와 동일  

　　B. 응답 데이터  
　　　　a. success: 저장 성공 여부  
　　　　b. saved_count: 저장된 메뉴 개수  
　　　　c. failed_count: 저장 실패한 메뉴 개수 (있을 경우)  

　2: 성공 처리  
　　A. 모든 메뉴 정보 저장 완료  
　　B. 다음 단계로 화면 전환  
　　C. 성공 토스트 메시지 표시  

　3: 부분 실패 처리  
　　A. 일부 메뉴만 저장 성공: "N개 중 M개 메뉴가 저장되었습니다" 메시지  
　　B. 실패한 메뉴 목록 표시 (선택 사항)
