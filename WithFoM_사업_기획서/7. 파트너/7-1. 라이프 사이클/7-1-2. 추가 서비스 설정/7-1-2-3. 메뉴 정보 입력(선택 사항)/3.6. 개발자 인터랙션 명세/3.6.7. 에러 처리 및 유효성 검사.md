**3.6.7.** 에러 처리 및 유효성 검사  


**3.6.7.1.** 클라이언트 측 유효성 검사  

　1: 실시간 유효성 검사  
　　A. 메뉴명 입력 시: 최대 길이(20자) 초과 시 경고 표시  
　　B. 메뉴명 중복 검사: 동일한 메뉴명 존재 시 경고 표시 (debounce 300ms)  
　　C. 메뉴 설명 입력 시: 최대 길이(100자) 초과 시 경고 표시  
　　D. 가격 입력 시: 숫자 형식 확인, 음수 입력 방지  
　　E. 카테고리 선택 시: 미선택 시 경고 표시  
　　F. 이미지 업로드 시: 파일 크기(5MB), 형식(JPG/PNG/WebP) 확인  
　　G. 노출 시간 입력 시: 시작 시간 ≤ 종료 시간 확인  
　　H. 할인 가격 입력 시: 기본 가격보다 낮은지 확인  
　　I. 옵션 그룹명 입력 시: 최대 길이(20자) 초과 시 경고 표시  

　2: 저장 버튼 클릭 시 검증  
　　A. 필수 항목 확인: 카테고리, 메뉴명, 가격  
　　B. 형식 검증: 숫자 형식, 문자열 길이 등  
　　C. 중복 검증: 메뉴명 중복 여부 최종 확인  
　　D. 시간대 검증: 노출 시작 시간이 종료 시간보다 이전인지 확인  
　　E. 검증 실패 시: 에러 메시지 표시 및 해당 필드 하이라이트  

　3: 에러 메시지 표시  
　　A. 필드별 에러 메시지: 각 입력 필드 하단에 에러 메시지 표시  
　　B. 전체 에러 메시지: 상단에 요약 에러 메시지 표시  


**3.6.7.2.** 서버 측 유효성 검사  

　1: 서버 응답 에러 처리

　　C. 403 Forbidden: "권한이 없습니다"  
　　D. 404 Not Found: "요청한 리소스를 찾을 수 없습니다"  
　　E. 500 Internal Server Error: "서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요"  

　2: 비즈니스 로직 에러

　　B. 중복 메뉴명: "이미 존재하는 메뉴명입니다"  
　　C. 존재하지 않는 카테고리: "선택한 카테고리가 존재하지 않습니다"  
　　D. 가격 범위 초과: "가격은 0원 이상이어야 합니다"  
　　E. 옵션 그룹 미존재: "선택한 옵션 그룹이 존재하지 않습니다"  
　　F. 미분류 삭제 불가: "미분류 카테고리는 삭제할 수 없습니다"

　2: 파일 크기 초과

　　B. 해결 방법: "파일을 분할하여 업로드해주세요"  

　3: 행 수 초과  
　　A. 에러 메시지: "최대 500개 메뉴까지 일괄 등록 가능합니다"  
　　B. 해결 방법: "파일을 분할하여 업로드해주세요"  

　4: 데이터 검증 오류  
　　A. 필수 항목 누락  
　　　　a. "메뉴명이 입력되지 않았습니다 (N행)"  
　　　　b. "가격이 입력되지 않았습니다 (N행)"  
　　　　c. "카테고리가 입력되지 않았습니다 (N행)"  
　　B. 형식 오류  
　　　　a. "가격은 숫자만 입력 가능합니다 (N행)"  
　　　　b. "메뉴명은 20자 이내로 입력해주세요 (N행)"  
　　　　c. "메뉴 설명은 100자 이내로 입력해주세요 (N행)"  
　　C. 참조 오류  
　　　　a. "카테고리 '[이름]'이 존재하지 않습니다 (N행)"  
　　　　b. "옵션그룹 '[이름]'이 존재하지 않습니다 (N행)"  
　　D. 중복 오류  
　　　　a. "이미 존재하는 메뉴명입니다 (N행)"

　5: 부분 저장 시 처리
