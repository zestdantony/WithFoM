**3.6.1.** 화면 초기 로딩  


**3.6.1.1.** API 호출  

　1: 엔드포인트: GET /api/partner/onboarding/reward-info  
　　A. 응답 데이터 구조  

| 필드 | 타입 | 설명 |  
|---|---|---|  
| service_type | string | 서비스 타입 (reward / order / hybrid) |  
| reward_enabled | boolean | 현재 리워드 기능 활성화 여부 |  
| existing_coupons | array | 기존에 입력된 쿠폰 정보 배열 |  
| existing_point_coupons | array | 기존에 입력된 포인트 쿠폰 정보 배열 |  
| existing_stamp_card | object | 기존에 입력된 스탬프 카드 정보 (null 시 미설정) |  

　2: 로딩 상태 처리  
　　A. 리워드 정보 로딩 중: 스켈레톤 UI 또는 로딩 스피너 표시  
　　B. 로딩 완료 후: 리워드 기능 선택 UI 렌더링  


**3.6.1.2.** 기존 리워드 정보 표시  

　1: 기존 리워드가 있는 경우  
　　A. 리워드 기능 On 상태로 자동 설정  
　　B. 기존 쿠폰 목록 표시 (쿠폰 추가 카드 영역에 렌더링)  
　　C. 기존 포인트 쿠폰 목록 표시 (포인트 쿠폰 추가 카드 영역에 렌더링)  
　　D. 기존 스탬프 카드 표시 (스탬프 카드 영역에 렌더링)  
　　E. 모든 리워드 수정/삭제 가능  

　2: 기존 리워드가 없는 경우  
　　A. 리워드 기능 On/Off 선택 대기 상태  
　　B. 빈 상태 UI 표시: "리워드를 추가하면 고객에게 게임 당첨 보상으로 제공됩니다"  


**3.6.1.3.** 상태 초기화  

　1: 클라이언트 상태 초기화  
　　A. rewardEnabled: API 응답의 reward_enabled 값  
　　B. coupons: existing_coupons 배열 또는 빈 배열  
　　C. pointCoupons: existing_point_coupons 배열 또는 빈 배열  
　　D. stampCard: existing_stamp_card 객체 또는 null  

　2: 총 리워드 개수 계산  
　　A. totalRewardCount = coupons.length + pointCoupons.length  
　　B. 6개 도달 시: "쿠폰 추가", "포인트 쿠폰 추가" 버튼 비활성화  
　　C. 스탬프는 1종만 허용이므로 별도 카운트
