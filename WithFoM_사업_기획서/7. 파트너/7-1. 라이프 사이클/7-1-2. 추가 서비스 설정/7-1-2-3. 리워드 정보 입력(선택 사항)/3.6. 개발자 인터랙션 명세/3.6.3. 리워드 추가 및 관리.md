**3.6.3.** 리워드 추가 및 관리  


**3.6.3.1.** 쿠폰 추가  

　1: "쿠폰 추가" 버튼 클릭  
　　A. 상태 관리: rewards 배열에 새 쿠폰 객체 추가  
　　B. 쿠폰 객체 구조  
　　　　a. id: 임시 ID (UUID 또는 timestamp)  
　　　　b. type: "coupon"  
　　　　c. name: 쿠폰명 (예: "아메리카노 500원 할인", "10% 할인 쿠폰")  
　　　　d. discount_type: "amount" | "percent"  
　　　　e. discount_value: 할인 금액 또는 할인율  
　　　　f. total_quantity: 총 발행 수량 (선택, null 시 무제한)  
　　　　g. valid_period: 유효 기간 객체 { start_date, end_date, is_unlimited }  
　　　　h. conditions: 사용 조건 객체  
　　　　　　• min_amount: 최소 결제 금액 (선택)  
　　　　　　• menu_category: 카테고리 조건 (선택)  
　　　　　　• menu_id: 단일 메뉴 조건 (선택)  
　　　　　　• channel: 채널 조건 배열 (선택)  
　　　　　　• time_range: 시간대 조건 (선택)  
　　　　　　• day_of_week: 요일 조건 배열 (선택)  
　　　　i. limits: 사용 제한 객체  
　　　　　　• daily_total_amount: 1일 사용 가능 총액 (선택)  
　　　　　　• daily_count: 1일 사용 횟수 제한 (선택)  
　　　　　　• per_payment_max_amount: 1회 결제 시 최대 사용 금액 (선택)  
　　　　　　• per_payment_count: 1회 결제당 사용 개수 제한 (선택)  

　2: 최대 개수 확인  
　　A. 쿠폰·포인트 합산 개수 확인  
　　B. 6개 도달 시: 추가 버튼 비활성화  
　　C. 에러 메시지: "쿠폰·포인트는 최대 6개까지 생성 가능합니다"  


**3.6.3.2.** 포인트 쿠폰 추가  

　1: "포인트 쿠폰 추가" 버튼 클릭  
　　A. 상태 관리: rewards 배열에 새 포인트 쿠폰 객체 추가  
　　B. 포인트 쿠폰 객체 구조  
　　　　a. id: 임시 ID (UUID 또는 timestamp)  
　　　　b. type: "point_coupon"  
　　　　c. name: 포인트 쿠폰명 (예: "포인트 500점", "포인트 3000점")  
　　　　d. point_amount: 포인트 금액 (1점 = 1원으로 사용)  
　　　　e. total_quantity: 총 발행 수량 (선택, null 시 무제한)  
　　　　f. valid_period: 유효 기간 객체 { start_date, end_date, is_unlimited }  
　　　　g. conditions: 사용 조건 객체 (쿠폰과 동일 구조)  
　　　　h. limits: 사용 제한 객체  
　　　　　　• daily_total_amount: 1일 사용 가능 총액 (선택)  
　　　　　　• per_payment_max_ratio: 1회 결제 기준 최대 사용 비율 (선택)  
　　　　　　• per_payment_max_amount: 1회 결제당 최대 사용 금액 (선택)  

　2: 포인트 금액 검증  
　　A. 최소값: 100점  
　　B. 최대값: 50,000점  
　　C. 에러 메시지: "포인트는 100점 이상 50,000점 이하로 입력해주세요"  


**3.6.3.3.** 리워드 삭제  

　1: 삭제 버튼 클릭  
　　A. 확인 다이얼로그 표시: "정말 삭제하시겠습니까?"  
　　B. 확인 시: rewards 배열에서 해당 리워드 제거  
　　C. 취소 시: 아무 동작 없음  

　2: 삭제 후 상태 업데이트  
　　A. 리워드 목록 재렌더링  
　　B. 리워드 개수 재계산  
　　C. 추가 버튼 활성화 여부 재확인  


**3.6.3.4.** 리워드 입력 필드 유효성 검사  

　1: 실시간 검증  
　　A. 입력 필드 onBlur 또는 onChange 이벤트  
　　B. 필드별 유효성 검사 실행  
　　C. 에러 메시지 즉시 표시  

　2: 쿠폰 검증 항목  
　　A. 필수 필드: name (최대 50자), discount_type, discount_value, valid_period  
　　B. 할인 금액: 1원 이상  
　　C. 할인율: 1~100%  
　　D. 최소 결제 금액: 0원 이상 (선택)  
　　E. 날짜: 시작일 ≤ 종료일  

　3: 포인트 쿠폰 검증 항목  
　　A. 필수 필드: name (최대 50자), point_amount, valid_period  
　　B. 포인트 금액: 100점 이상 50,000점 이하  
　　C. 최소 결제 금액: 0원 이상 (선택)  
　　D. 날짜: 시작일 ≤ 종료일
