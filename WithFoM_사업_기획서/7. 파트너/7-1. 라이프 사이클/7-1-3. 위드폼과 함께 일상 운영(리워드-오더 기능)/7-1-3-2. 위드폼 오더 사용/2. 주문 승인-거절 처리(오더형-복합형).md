　　**2.2.** 주문 승인/거절 처리(오더형/복합형)  

　A. 승인  
　　　a. 주문 상태 플로우  
　　　　aa. 신규 상태  
　　　　　• 버튼: [승인] / [취소]  
　　　　　• 설명: 주문 접수 상태  
　　　　bb. 준비중 상태  
　　　　　• 버튼: [준비 완료]  
　　　　　• 설명: 포장/제조 진행 상태  
　　　　cc. 준비완료 상태  
　　　　　• 버튼: (추가 버튼 없음)  
　　　　　• 설명: 상태 표시만, 픽업 완료는 추후 고급 옵션  
　　　b. 승인 처리  
　　　　aa. [승인] 버튼 클릭 시 상태가 "준비중"으로 변경됩니다.  
　　　c. 주문 리스트 표시 항목  
　　　　aa. 주문 번호  
　　　　bb. 주문 채널(매장/테이크아웃/위드폼 오더)  
　　　　cc. 주문 시간+경과 시간(예: 10:02, +3분)  
　　　　dd. 메뉴 요약(메뉴명/옵션/수량)  
　　　　ee. 총 결제 금액  
　　　　ff. 결제 상태(선결제/후결제)  
　　　　gg. 리워드 사용 여부(쿠폰/포인트/스탬프 간단 표시)  
　　　　hh. 현재 상태(신규/준비중/준비완료/취소)  
　　　　ii. 상태 변경/취소 버튼  
　　　• 참조: WithFoM_Master_Planning_v3.10.md 7장 7-3절 7-3-4. 메뉴 오더 (카페 메뉴 주문)의 "주문 상태 플로우" 표 및 "주문 리스트 표시 항목" 항목  
　B. 거절  
　　　a. 주문 취소 처리  
　　　　aa. [취소] 버튼 클릭 시 상태가 "취소"로 변경됩니다.  
　　　　bb. 주문 취소 로직(결제 취소·로그 기록)이 실행됩니다.  
　　　b. 취소 상태  
　　　　aa. 취소된 주문은 상태만 표시됩니다.
