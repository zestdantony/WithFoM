**2.1.** 등급 산정 연동

## 파트너 동작

| 동작 | 내용 |
|---|---|
| 등급 확인 | 파트너 포털 > 파트너 등급 > 나의 등급에서 확인 |
| 등급 기준 확인 | 등급 혜택•기준 메뉴에서 산정 기준 확인 |
| 등급 변동 알림 수신 | 등급 변경 예정 및 확정 알림 수신 |
| 매칭 등급 확인 | 나의 종합 매칭 등급 확인 (후순위/일반/우선/최우선) |

## 시스템 자동 동작

| 조건 | 동작 |
|---|---|
| 캠페인 종료 시 | 등급 산정 점수 자동 계산 및 재산정 |
| 익월 1일 | 새 등급 적용 |
| 6개월 미매칭 도달 시 | 한 단계 등급 변경(다운) 자동 적용 |
| 매칭 확정 시 | 미매칭 기간 카운트 리셋, 등급 유지 |

## 등급 산정 기준 (점수제)

| 항목 | 가중치 | 점수 범위 |
|---|---|---|
| 보고율 | 45% | 20점(최하) ~ 80점(상) |
| 스캔율 | 45% | 20점(최하) ~ 80점(상) |
| 위드폼 오더 이용 | 10% | 0점(미이용) / 100점(이용) |

## 등급 점수 기준

| 등급 | 점수 | 산정 기간 | 혜택 |
|---|---|---|---|
| 브론즈 | 36점 미만 | 6개월 | 컵홀더 단가 50% 자부담 |
| 실버 | 36점 이상 | 6개월 | 컵홀더 무료 제공 |
| 골드 | 55점 이상 | 6개월 | 컵홀더 무료 + 멤버십 30% 할인 |
| 플래티넘 | 73점 이상 | 1년 | 컵홀더 무료 + 멤버십 30% 할인 + 개당 10원 인센티브 |

※ 플래티넘 달성을 위해 위드폼 오더 이용 필수 (등급 산정 기간 내 항상 이용 조건, 중간 비활성화 기간 존재 시 플래티넘 제외)

## 캠페인 매칭 우선순위 점수 체계

### 매칭 점수 구성

| 구분 | 항목 | 점수 |
|---|---|---|
| 등급 매칭 점수 | 플래티넘 | 4 |
| | 골드 | 3 |
| | 실버 | 2 |
| | 브론즈 | 1 |
| 멤버십 매칭 점수 | 프리미엄 | 3 |
| | 플러스 | 2 |
| | 베이직 | 1 |

### 종합 매칭 점수 매트릭스

| | 베이직(1) | 플러스(2) | 프리미엄(3) |
|---|---|---|---|
| 플래티넘(4) | 5 | 6 | 7 |
| 골드(3) | 4 | 5 | 6 |
| 실버(2) | 3 | 4 | 5 |
| 브론즈(1) | 2 | 3 | 4 |

종합 매칭 점수 = 등급 점수(1~4) + 멤버십 점수(1~3) + 포기 감점(0 ~ -n)
점수 범위: 최저 2.0점 하한 ~ 최대 7점 / 가중치 비율: 등급 4/7(약 57%) : 멤버십 3/7(약 43%)

### 매칭 우선순위 그룹

| 우선순위 그룹 | 점수 범위 | 해당 조합 (높은 점수순 매칭) |
|---|---|---|
| 최우선 매칭 | 6~7점 | 플래티넘+프리미엄(7) → 플래티넘+플러스(6) → 골드+프리미엄(6) |
| 우선 매칭 | 5점 | 플래티넘+베이직(5) → 골드+플러스(5) → 실버+프리미엄(5) |
| 일반 매칭 | 4점 | 골드+베이직(4) → 실버+플러스(4) → 브론즈+프리미엄(4) |
| 후순위 매칭 | 2~3점 | 실버+베이직(3) → 브론즈+플러스(3) → 브론즈+베이직(2) |

→ 4그룹 × 3조합 = 12개 전체 조합 균등 분배

### 캠페인 포기 감점 적용

| 항목 | 내용 |
|---|---|
| 매칭 전 포기 / 자동 포기 | -0.5점/회 |
| 참여 중 포기 (중도 포기) | -1.0점/회 (가중 감점) |
| 적용 대상 | 종합 매칭 점수에서 차감 (최저 2점 하한) |
| 누적 예시 | 매칭 전 1회(-0.5) + 참여 중 1회(-1.0) = -1.5점, 3회차는 계약 정지 |
| 초기화 조건 | 계약 정지 해제 시 감점 초기화 |
| 관리자 복원 | 합당 사유 인정 시 해당 건의 감점만큼 복원 (매칭 전: +0.5, 참여 중: +1.0) |
| 그룹 적용 | 최우선(6.0 이상), 우선(5.0~5.9), 일반(4.0~4.9), 후순위(2.0~3.9) |

### 캠페인 매칭 순서 (모든 캠페인 공통)

```
Step 1: 광고주 타겟팅 일치도순 (선행 조건)
  ├─ 타겟팅 필터: 지역, 상권, 연령층, 성별, 고객 구성, 음료 가격
  ├─ 완전 일치 파트너 풀 먼저
  ├─ 높은 일치 (5개 이상) 파트너 풀 다음
  ├─ 부분 일치 (3~4개) 파트너 풀 다음
  └─ 최소 일치 (1~2개) 파트너 풀 마지막
     ↓
Step 2: 각 풀 내에서 종합 매칭 점수순 (등급 + 멤버십 + 포기 감점)
     ↓
Step 3: 동일 점수 내에서 미매칭 기간 우선순 (미매칭 기간이 긴 파트너 우선)
     ↓
Step 4: 미매칭 기간도 동일 시 타이브레이커 (오더 이용 → 기여도 → 랜덤)
```

→ 타겟팅 일치가 항상 매칭 점수보다 우선
→ 타겟팅 완전 일치 + 후순위 매칭(2점) > 부분 일치 + 최우선 매칭(7점)

### 미매칭 기간 우선순위 (Step 3)

| 항목 | 내용 |
|---|---|
| 적용 대상 | 매칭 가능한 모든 파트너 (매칭 대기 중 파트너 기준) |
| 기간 산정 기준 | 마지막 캠페인 종료일 또는 가입일로부터 현재까지의 미매칭 기간 |
| 정렬 순서 | 미매칭 기간이 긴 파트너 우선 |

### 타이브레이커 (Step 4)

| 순서 | 기준 | 상세 |
|---|---|---|
| 1순위 | 위드폼 오더 이용 여부 | 이용 파트너 > 미이용 파트너 |
| 2순위 | 플랫폼 기여도 | 양쪽 오더 이용 시: 6개월 누적 매출 비교 / 양쪽 미이용 시: 6개월 누적 이용 금액 비교 |
| 3순위 | 랜덤 선택 | 위 모든 기준이 동일할 경우 무작위 |

### 부분 기간 매칭 (수량 충족 매칭)

캠페인 총 수량을 매칭 대기 파트너만으로 충족할 수 없는 경우, 현재 캠페인 진행 중인 파트너까지 포함하여 수량을 충족하는 매칭 방식.

| Phase | 대상 | 매칭 순서 |
|---|---|---|
| Phase 1 | 매칭 대기 파트너 | 캠페인 전체 기간 참여. Step 1~4 매칭 우선순위에 따라 배정. 각 파트너의 캠페인 기간 내 예상 소진량 계산하여 배정 |
| Phase 2 | 진행 중 파트너 (수량 부족 시) | Phase 1 합산 소진량이 총 수량 미달 시 적용. 타겟팅 일치 파트너 중 현재 캠페인 종료가 가까운 순으로 정렬. 동일 종료일 시 매칭 점수 우선순위 → 랜덤. 참여 기간: 현재 캠페인 종료 익일 ~ 새 캠페인 종료일 |
| Phase 3 | 매칭 확정 | 모든 수량 매칭 완료 후 광고주에게 캠페인 확정 통보. 각 파트너에게는 자신의 참여 기간만 안내 |

```
[부분 기간 매칭 예시]
광고주 캠페인: 3만개, 2월 1일 ~ 2월 28일

Phase 1: 매칭 대기 파트너
  파트너A (대기): 2/1~2/28, 예상 소진 8,000개
  파트너B (대기): 2/1~2/28, 예상 소진 7,000개
  파트너C (대기): 2/1~2/28, 예상 소진 6,000개
  파트너D (대기): 2/1~2/28, 예상 소진 4,000개
  → 대기 합계: 25,000개 (부족: 5,000개)

Phase 2: 진행 중 파트너 (종료 가까운 순)
  파트너E (진행 중, 현재 캠페인 2/7 종료):
    → 2/8~2/28 참여 (21일), 예상 소진 3,000개
  파트너F (진행 중, 현재 캠페인 2/10 종료):
    → 2/11~2/28 참여 (18일), 예상 소진 2,500개
  → 합계: 30,500개 ≥ 30,000개 → 매칭 완료

파트너별 안내:
  파트너A~D: "2월 1일 ~ 2월 28일 캠페인"
  파트너E: "2월 8일 ~ 2월 28일 캠페인"
  파트너F: "2월 11일 ~ 2월 28일 캠페인"
광고주 안내: "캠페인 매칭 완료. 3만개, 2/1~2/28"
```

→ 부분 기간 매칭은 단일 캠페인 원칙을 위배하지 않음 (동시 진행이 아닌 미래 기간 예약)
→ 기존 연속 매칭(종료 1주일 전 다음 매칭)의 시스템 주도 확장

### 파트너 표시 vs 시스템 내부

| 구분 | 항목 | 내용 |
|---|---|---|
| 파트너 표시 | 파트너 등급 매칭 | 등급별 매칭 라벨 (최우선/우선/일반/후순위) |
| 파트너 표시 | 멤버십 플랜 매칭 | 플랜별 매칭 라벨 (우선/일반/후순위) |
| 파트너 표시 | 나의 종합 매칭 등급 | 합산 점수 기반 그룹 라벨 |
| 파트너 표시 | 안내 문구 | "파트너 등급과 멤버십 플랜을 합산하여 캠페인 매칭 우선순위가 결정됩니다" |
| 시스템 내부 | 종합 매칭 점수 | 등급 점수 + 멤버십 점수 + 포기 감점 (비공개) |
| 시스템 내부 | 미매칭 기간 | 마지막 캠페인 종료 후 경과 기간 (비공개) |
| 시스템 내부 | 그룹 내 순위 | 점수순 + 미매칭 우선 + 타이브레이커 적용 결과 (비공개) |
| 시스템 내부 | 타겟팅 일치도 | 광고주 타겟과 파트너 매장 특성 일치도 (비공개) |
| 시스템 내부 | 부분 기간 매칭 | 수량 부족 시 진행 중 파트너 자동 배정 (비공개) |

## 연동 요약

| 연동 대상 | 연동 항목 | 영향 |
|---|---|---|
| 파트너 | 등급 변동 알림 | 등급 변경 예정 및 완료 알림 |
| 파트너 | 등급 배지 변경 | 앱 상단 등급 배지 갱신 |
| 파트너 | 혜택 자동 적용 | 등급별 혜택 발주/결제 시 자동 적용 |
| 파트너 | 매칭 등급 표시 | 종합 매칭 등급 라벨 갱신 |
| 관리자 | 등급 산정 로그 | 파트너별 등급 산정 이력 기록 |
| 관리자 | 캠페인 매칭 | 타겟팅 일치 → 매칭 점수순 자동 매칭 적용 |

## 시스템 흐름

```
[캠페인 종료 시]
캠페인 종료 → 데이터 집계 → 이전 6개월(1년) 기준 등급 재산정 → 파트너 알림 → 익월 등급 적용 → 혜택 자동 적용

[미매칭 등급 변경]
마지막 캠페인 종료 → 미매칭 기간 카운트 시작 → 3개월 경과(1차 알림) → 5개월 3주차(2차 알림) → 6개월 도달 → 한 단계 등급 변경(다운) 적용

[매칭 시 등급 유지]
미매칭 대기 중 → 캠페인 매칭 확정 → 미매칭 카운트 리셋 → 등급 유지 → 캠페인 종료 후 유예 기간 재시작

[등급 변경 후 재매칭]
등급 변경(다운) → 새 캠페인 매칭 → 캠페인 참여 및 성과 → 종료 시 이전 6개월(1년) 재산정 → 조건 충족 시 즉시 업그레이드

[캠페인 포기 (매칭 전)]
캠페인 포기 → 참여 기간은 산정 포함 → 포기 시점부터 미매칭 카운트 시작 → 포기 횟수 +1 차감 → 매칭 점수 -0.5 감점 → 관리자 사유 검토 → 3회 누적 시 계약 정지

[캠페인 포기 (참여 중 — 중도 포기, 관리자 승인 필요)]
파트너 참여 중지 요청(사유 직접 입력) → 관리자 검토 → (승인 시) 참여 기간은 산정 포함 → 승인 시점부터 미매칭 카운트 시작 → 남은 컵홀더 보상금 정산(파트너→위드폼) → 포기 횟수 +1 차감 → 매칭 점수 -1.0 감점 → 3회 누적 시 계약 정지 / (반려 시) 캠페인 계속 참여

[캠페인 매칭 알고리즘]
광고주 캠페인 생성 → 타겟팅 조건 설정 → 파트너 풀 타겟팅 필터링 → 일치도별 풀 분류 → 각 풀 내 종합 매칭 점수순 정렬 → 미매칭 기간 우선순 → 타이브레이커 (오더→기여도→랜덤) → Phase 1 매칭 → 수량 부족 시 Phase 2 (진행 중 파트너 부분 기간 매칭) → 매칭 확정
```

## 실 사용 기간 계산

| 조건 | 처리 방식 |
|---|---|
| 캠페인 참여 기간 | 등급 산정 기간에 카운팅 |
| 미매칭 대기 기간 | 등급 산정 기간에서 제외 |
| 휴업 기간 (사유 보고 시) | 등급 산정 기간에서 제외, 미매칭 기간에서도 제외 |

※ 등급 산정 기간은 달력상 경과 기간이 아닌 실제 캠페인 참여 기간 기준

## 등급 유지 및 등급 변경 규칙

| 규칙 | 조건 | 결과 |
|---|---|---|
| 등급 유지 | 6개월 이내 새 캠페인 매칭 완료 | 현재 등급 유지, 유예 기간 재시작 |

## 캠페인 포기 처리

| 상황 | 처리 |
|---|---|
| 포기 전까지 참여 기간 | 등급 산정에 포함 |
| 포기 시점 이후 | 미매칭 기간 카운트 시작 |
| 매칭 전 포기 / 자동 포기 감점 | -0.5점/회 |

| 참여 중 포기 절차 | 파트너 요청 → 관리자 승인 시 확정 (즉시 포기 아님) |
| 참여 중 포기 시 컵홀더 보상 | 승인 확정 시 남은 컵홀더 비용을 파트너가 위드폼에 정산 |
| 참여 중 포기 사유 | 직접 입력만 가능 (매칭 전 포기의 선택지와 다름) |
| 포기 횟수 | 총 3회 누적 시 계약 정지, 관리자 합당 사유 인정 시 복원 |
| 계약 정지 해제 시 | 포기 횟수 + 매칭 감점 초기화 (이력은 기록) |
