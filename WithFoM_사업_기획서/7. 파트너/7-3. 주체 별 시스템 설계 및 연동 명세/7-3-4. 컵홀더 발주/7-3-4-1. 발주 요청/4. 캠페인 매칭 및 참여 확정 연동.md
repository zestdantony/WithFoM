**1.4.** 캠페인 매칭 및 참여 확정 연동

## 파트너 동작

| 동작 | 내용 |
|---|---|
| 매칭 알림 확인 | 앱 푸시/앱 내 알림으로 캠페인 매칭 안내 확인 |
| 참여 확정/포기 선택 | 캠페인 참여(보드)에서 확인 또는 포기 선택 (캠페인 참여 필수) |
| (브론즈) 결제 진행 | 브론즈 등급은 결제 완료 시 참여 확정 |
| 포기 사유 제출 | 포기 사유 선택 및 필요 시 추가 정보 입력 후 제출 |
| 배송 현황 확인 | 캠페인 참여 섹션에서 컵홀더 배송·입고 진행 상태 확인 |
| 배송완료 수동 확인 | 배송중 단계에서 [배송완료] 버튼 클릭으로 입고 처리 (택배사 미처리 시 대비) |
| 배송 문제 문의 | [배송 문의 →] 클릭 → 7-2-11-2 발주 주문 상세에서 협력사에게 문의 |

## 관리자 동작

| 동작 | 내용 |
|---|---|
| 캠페인 매칭 완료 | 광고주 캠페인과 파트너 매칭을 3영업일 이내 완료 목표 |
| 매칭 전 포기 사유 검토 | 파트너 포기 사유 사후 검토, 합당 시 포기 횟수 복원 |
| 참여 중 포기 요청 승인/반려 | 파트너의 중도 포기 요청 검토 후 승인 또는 반려 (승인 시 포기 확정) |
| 배송 미입고 긴급 확인 | 송장 입력 후 5영업일 경과 미입고 건 긴급 알림 수신 후 확인 조치 |
| 캠페인 일정 수정 | 배송 문제로 실제 입고일이 달라질 경우 파트너 문의 후 입고일/배포 시작일 수정 |

## 협력사 동작

| 동작 | 내용 |
|---|---|
| 송장번호 입력 | 컵홀더 출고 시 택배 송장번호 입력 → 파트너 배송 현황 자동 갱신 |
| 배송 문제 문의 응대 | 파트너의 배송 문제 문의에 대한 확인 및 답변 |
| 입고 상태 복구 | 물류 사고 확인 시 입고완료를 배송준비/배송중으로 복구 → 관리자 자동 알림 |

## 시스템 자동 동작

| 조건 | 동작 |
|---|---|
| 광고주-파트너 매칭 완료 | 푸시 알림 및 앱 내 알림 발송, 캠페인 현황 메뉴 뱃지 표시 |
| 응답 기한 도래 | 미응답 시 자동 포기 처리 + 포기 횟수 1회 차감 |
| 응답 기한 2시간 전 | (필수) 리마인드 알림 발송 |
| 브론즈 결제 미완료 | 결제 기한 내 미결제 시 자동 포기 처리 |
| 포기 3회 누적 | 계약 정지(일시 정지) 처리 |
| 협력사 송장번호 입력 | 파트너 배송 현황 "배송중"으로 자동 갱신 + 도착 예정일 표시 |
| 택배사 API 배송완료 감지 | 자동 배송완료 + 입고 처리 → 캠페인 배포 자동 시작 |
| 입고 완료 (수동/자동) | 캠페인 배포 시작 + 일일 소진 보고 활성화 + 재고 관리 연동 + 알림 시스템 활성화 |
| 송장 입력 후 5영업일 경과 미입고 | 관리자에게 긴급 알림 (자동 처리하지 않음, 관리자 확인 후 수동 조치) |
| 파트너→협력사 배송 문의 | 관리자에게 자동 알림 (일반) |
| 협력사 입고 상태 복구 | 관리자에게 자동 알림 (중요) + 파트너 배송 현황 자동 갱신 |

## 응답/결제 정책

| 항목 | 내용 |
|---|---|
| 응답 기한 | 매칭 알림 수신 후 익일 15시까지 |
| 리마인드 | 기한 2시간 전 알림 (필수) |
| 브론즈 결제 | 컵홀더 단가 50% 부담, 결제 완료 시 매칭 확정 |
| 미응답/미결제 | 자동 포기 처리, 포기 횟수 차감, 해당 캠페인 재매칭 불가 |
| 포기 정책 | 총 3회 포기 시 계약 정지, 관리자 합당 사유 인정 시 횟수 복원 |
| 매칭 감점 | 매칭 전/자동 포기: -0.5점, 참여 중 포기: -1.0점, 계약 정지 해제 시 초기화. 종합 매칭 점수 = 등급 + 멤버십 + 포기 감점 |
| 참여 중 포기 절차 | 파트너 요청 → 관리자 승인 시 포기 확정 (즉시 포기 아님) |
| 참여 중 포기 보상 | 승인 확정 시 남은 컵홀더 비용을 파트너가 위드폼에 정산 (남은 수량 × 협력사 기준가) |
| 참여 중 포기 사유 | 직접 입력만 가능 (매칭 전 포기의 선택지와 다름) |
| 매칭 완료 목표 | 관리자 3영업일 이내 매칭 완료 |
| 매칭 우선순위 | 타겟팅 일치도 → 종합 매칭 점수 → 미매칭 기간 우선 → 타이브레이커(오더→기여도→랜덤) |
| 부분 기간 매칭 | 대기 파트너로 수량 부족 시 진행 중 파트너를 종료 가까운 순으로 매칭 (동일 종료일 시 매칭 점수 → 랜덤). 각 파트너에게는 참여 가능 기간만 안내 |

## 배송·입고 정책

| 항목 | 내용 |
|---|---|
| 배송 현황 표시 | 참여 확정 후 캠페인 참여 섹션에 배송 단계별 상태 버튼 표시 |
| 배송 단계 | 주문접수 → 제작중 → 배송준비 → 배송중 → 입고 |
| 배송완료 경로 A | 파트너 수동 [배송완료] 클릭 (배송중 단계에서) |
| 배송완료 경로 B | 택배사 API 배송완료 자동 감지 |
| 미입고 안전장치 | 송장번호 입력 후 영업일 5일 경과, A/B 모두 미발생 시 관리자 긴급 알림 (자동 처리 아님) |
| 입고 완료 연쇄 동작 | 배포 시작 + 소진 보고 활성화 + 재고 관리 연동 + 알림 시스템 활성화 |
| 배송 문의 경로 | 캠페인 참여 섹션 [배송 문의 →] → 7-2-11-2 발주 주문 상세 → 협력사 문의 |
| 협력사 상태 복구 | 물류 사고 시 협력사가 입고 상태를 이전 단계로 복구 가능 → 관리자 알림 |
| 입고일 변경 | 배송 문제로 실제 입고일 변경 시 관리자만 수정 가능 (파트너 문의 후) |

## 상태 변경

| 상태 | 조건 | 비고 |
|---|---|---|
| 참여 확정됨 | 실버 이상: 확인 즉시 / 브론즈: 결제 완료 | 캠페인 현황/배송·입고 관련 알림 활성화 |
| 포기됨 | 포기 제출 또는 자동 포기 | 포기 사유 기록, 재매칭 불가, 포기 횟수 차감 |
| 참여 중지 요청중 | 파트너 중도 포기 요청 제출 | 관리자 승인 전까지 캠페인 계속 참여 |
| 중도 포기 | 관리자 승인 확정 | 감점 + 보상금 정산, 재참여 불가 |
| 계약 정지 | 포기 3회 누적 | 기존 이용제한 "일시 정지"와 동일 |
| 배송중 | 협력사 송장번호 입력 | 파트너 배송 현황 자동 갱신, 배송완료 버튼 활성화 |
| 입고 | 파트너 수동 배송완료 또는 택배사 API 자동 | 배포 시작 + 소진 보고 + 재고 관리 활성화 |

## 연동 요약

| 연동 대상 | 연동 항목 | 영향 |
|---|---|---|
| 파트너 | 매칭 알림 | "캠페인이 매칭 되었습니다. 지정된 일정에 맞춰 배포를 시작해 주세요." |
| 파트너 | 기한 안내 | 응답/결제 기한 및 미응답/미결제 시 자동 포기 안내 |
| 파트너 | 결과 알림 | 참여 확정/포기 처리 결과 알림 |
| 파트너 | 포기 횟수/감점 알림 | 포기 시 남은 횟수 및 매칭 감점 안내 (매칭 전 -0.5 / 참여 중 -1.0) |
| 파트너 | 참여 중지 요청 결과 | 관리자 승인/반려 결과 알림 |
| 파트너 | 참여 중 포기 보상 안내 | 승인 시 남은 컵홀더 보상금 계산 내역 및 정산 안내 |
| 파트너 | 배송 현황 실시간 갱신 | 협력사 송장 입력/택배사 배송완료 시 자동 갱신 |
| 파트너 | 입고 완료 알림 | 배포 시작 + 소진 보고 활성화 안내 |
| 협력사 | 송장번호 연동 | 송장번호 입력 시 파트너 배송 현황 자동 갱신 |
| 협력사 | 배송 문의 수신 | 파트너의 배송 문제 문의 수신 및 응대 |
| 협력사 | 입고 상태 복구 | 물류 사고 시 입고 상태 복구 → 관리자 알림 + 파트너 현황 갱신 |
| 관리자 | 매칭/응답 로그 | 매칭, 응답, 자동 포기, 포기 사유 기록 |
| 관리자 | 브론즈 결제 상태 | 브론즈 결제 완료 여부 기반 확정 처리 |
| 관리자 | 매칭 전 포기 사유 검토 | 합당 사유 시 포기 횟수 복원 처리 |
| 관리자 | 참여 중 포기 요청 승인/반려 | 요청 검토 후 승인(포기 확정) 또는 반려(캠페인 계속) |
| 관리자 | 매칭 완료 기한 | 3영업일 이내 매칭 완료 목표 관리 |
| 관리자 | 배송 문의 알림 | 파트너→협력사 문의 시 자동 알림 (일반) |
| 관리자 | 입고 상태 복구 알림 | 협력사 상태 복구 시 자동 알림 (중요) |
| 관리자 | 미입고 긴급 알림 | 송장 입력 후 5영업일 경과 미입고 시 긴급 알림 |

## 시스템 흐름

```
광고주 캠페인 확정 → 타겟팅 필터링 → Phase 1: 대기 파트너 매칭 (점수순→미매칭 우선→타이브레이커) → 수량 부족 시 Phase 2: 진행 중 파트너 부분 기간 매칭 (종료 가까운 순→점수→랜덤) → 매칭 완료(관리자 3일 목표) → 알림 발송/뱃지 표시 → (파트너) 확인/포기 선택 → (브론즈) 결제 완료 시 확정 → 상태 업데이트 → (미응답/미결제) 자동 포기 처리(-0.5점) → (참여 확정 시) 배송 현황 표시 시작 → (협력사) 송장번호 입력 → 배송중 상태 + 배송완료 버튼 활성화 → (파트너 수동 클릭 또는 택배사 API) 배송완료 → 입고 처리 → 배포 시작 + 소진 보고 활성화 → (참여 중 포기 시) 파트너 중지 요청 → 관리자 승인/반려 → (승인 시) 남은 컵홀더 보상 정산 + 감점 -1.0점 → (매칭 전 포기 시) 관리자 사후 사유 검토
```

• 참조: 7-1-4. 캠페인 매칭 확인 및 참여 확정(캠페인 발주), 7-2-5-2. 캠페인 현황
