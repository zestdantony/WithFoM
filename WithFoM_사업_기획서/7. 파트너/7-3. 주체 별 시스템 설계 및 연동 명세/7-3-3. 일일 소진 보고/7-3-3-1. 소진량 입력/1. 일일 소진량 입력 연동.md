**1.1.** 일일 소진량 입력 연동

## 파트너 동작

| 동작 | 내용 |
|---|---|
| 소진량 입력 | 오늘 사용한 컵홀더 개수 입력 |
| 제출 버튼 클릭 | 소진 보고 데이터 전송 |

## 연동 요약

| 연동 대상 | 연동 항목 | 영향 |
|---|---|---|
| 관리자 | 보고 데이터 저장 | 일별 소진량 DB 저장 |
| 관리자 | 등급 산정 반영 | 보고율 45% 가중치로 등급 점수 반영 |
| 관리자 | 정산 근거 | 월말 정산 시 소진량 기반 정산 |
| 관리자 | FDS 검증 | QR 스캔량과 교차 검증 |
| 관리자 | 광고주 리포트 반영 | 캠페인 소진 현황 리포트에 반영 |
| 협력사 | 재고 현황 업데이트 | 물류사에 현재 재고 상태 공유 (발주 참고용) |

## 시스템 흐름

```
파트너 소진량 입력 → 관리자 DB 저장 → 등급 산정 모듈 반영 → FDS 교차 검증 → 광고주 리포트 갱신 → 협력사 재고 현황 공유
```

## 이상치 감지 조건

| 조건 | 처리 |
|---|---|
| 평균 대비 1.5배 이상 | 경고 메시지 노출 (확인 후 제출 가능) |
| 전일 대비 ±150% 이상 급변 반복 | FDS 이상 패턴 분류 |
| 급감/급증 패턴 | FDS 2차 검증 |
| QR 스캔량과 불일치 | 교차 검증 수행 |

## 경고 팝업 처리

| 동작 | 처리 |
|---|---|
| 취소 선택 | 입력창으로 돌아가 수정 가능 |
| 확인 선택 | 입력값 그대로 제출 처리 |

## FDS 이상 탐지 목적

| 목적 | 내용 |
|---|---|
| 정산 무결성 보호 | 반복적인 이상치 입력 패턴 감지 |
| 데이터 신뢰도 보장 | QR 스캔량과 소진량 교차 검증 |
