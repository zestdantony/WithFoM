　　　**1.1.** 서비스 타입 변경 연동

## 파트너 동작

| 동작 | 내용 |
|---|---|
| 서비스 타입 설정 메뉴 | 리워드형/오더형/복합형 선택 |
| 저장 버튼 클릭 | 서비스 타입 변경 |

## 서비스 타입 정의

| 타입 | 리워드 | 오더 | 설명 |
|---|---|---|---|
| 리워드형 | ✅ | ❌ | QR 스캔 후 게임/리워드만, 앱 주문 불가 |
| 오더형 | ❌ | ✅ | 앱 주문·결제만, 게임/리워드 없음 |
| 복합형(권장) | ✅ | ✅ | 리워드 + 오더 모두 사용 |

## 연동 요약

| 연동 대상 | 연동 항목 | 영향 |
|---|---|---|
| 파트너 | 메뉴 구성 변경 | 타입에 따라 사이드바 메뉴 노출/비노출 |
| 파트너 | 대시보드 변경 | 결제/리워드 보드 노출/비노출 |
| 소비자 | 기능 제한 | 타입에 따라 주문/리워드 기능 노출 여부 변경 |
| 관리자 | 타입 변경 로그 | 파트너별 서비스 타입 변경 이력 기록 |

## 시스템 흐름

```
파트너 서비스 타입 변경 → 서버 저장 → 파트너 앱 메뉴 재구성 → 소비자 앱 기능 반영 → 관리자 로그 기록
```
