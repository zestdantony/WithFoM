　　　**3.3.** 승인 반려 연동  


　　　　**3.3.1.** 관리자 동작  


| 동작 | 내용 |
|---|---|
| 서류 검토 | 계약서, 사업자등록증 검토 |
| 반려 사유 입력 | 반려 사유 선택 및 상세 내용 입력 |
| 반려 버튼 클릭 | 파트너 승인 반려 처리 |


　　　　**3.3.2.** 연동 요약  


| 연동 대상 | 연동 항목 | 영향 |
|---|---|---|
| 파트너 | 상태 변경 | "보완 요청" 상태로 변경 |
| 파트너 | 알림 발송 | 반려 사유 포함 알림 발송 |
| 파트너 | 재업로드 활성화 | 서류 재업로드 기능 활성화 |


　　　　**3.3.3.** 시스템 흐름  


```
관리자 반려 처리 → 파트너 상태 "보완 요청" 변경 → 반려 사유 저장 → 파트너 알림 발송 → 재업로드 기능 활성화
```


　　　　**3.3.4.** 알림  


| 대상 | 알림 내용 |
|---|---|
| 파트너 | "승인 요청이 반려되었습니다. 사유: [반려 사유]. 서류를 수정 후 다시 제출해주세요." |


　　　　**3.3.5.** 반려 사유 예시  


| 사유 |
|---|
| 서류 불일치 (계약서와 사업자등록증 정보 불일치) |
| 서류 불선명 (사진이 흐릿하거나 일부가 잘림) |
| 서명 누락 (계약서 서명/날인 누락) |
| 유효하지 않은 사업자등록증 |
