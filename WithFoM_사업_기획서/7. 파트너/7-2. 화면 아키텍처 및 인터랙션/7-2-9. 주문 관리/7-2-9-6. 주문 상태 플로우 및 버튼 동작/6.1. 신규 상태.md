**6.1.** 신규 상태  


**6.1.1.** 버튼 및 상태 변경  

**6.1.1.1.** 버튼  
1: [승인]  
2: [거절]  

**6.1.1.2.** 버튼 동작  
1: [승인] 클릭 시  
　A. 상태가 "준비중"으로 변경  
　B. 소비자 주문 취소 불가 상태로 전환  
　C. 소비자에게 "주문이 접수되었습니다" 푸시 알림 발송  
2: [거절] 클릭 시  
　A. 거절 사유 선택 팝업 표시  
　B. 상태가 "취소"로 변경  
　C. 결제 취소 로직(전액 환불) 자동 실행  
　D. 소비자에게 "매장 사정으로 주문이 취소되었습니다" 푸시 알림 발송  


**6.1.2.** 자동 승인 타이머  

**6.1.2.1.** 동작 방식  
1: 신규 주문 접수 시 60초 타이머 자동 시작  
2: 타이머 종료 시 자동 승인 처리  
　A. 상태가 "준비중"으로 자동 변경  
　B. 소비자 주문 취소 불가 상태로 전환  
　C. 소비자에게 "주문이 접수되었습니다" 푸시 알림 발송  
　D. 파트너에게 "자동 승인되었습니다" 토스트 메시지  


**6.1.3.** 소비자 주문 취소 가능 구간  

**6.1.3.1.** 취소 가능 조건  
1: 주문 상태가 "신규"인 동안  
　A. 파트너 승인 전  
　B. 60초 자동 승인 타이머 종료 전  
2: 소비자 앱에서 취소 버튼 활성화  
　A. 주문 내역 > 해당 주문 > [주문 취소] 버튼  
　B. 취소 사유 선택 (선택 사항)  
　C. 즉시 전액 환불 처리  

**6.1.3.2.** 취소 시 파트너 화면 처리  
1: 실시간 알림  
　A. WebSocket 이벤트: order.cancelled_by_customer  
　B. 해당 주문 카드 즉시 제거  
　C. 토스트 메시지: "고객이 주문을 취소했습니다"  
2: 취소 내역 확인  
　A. 주문 관리 > 필터 > 취소 선택  
　B. 취소 사유 및 취소 시간 확인 가능  


**6.1.4.** 파트너 거절 사유 목록  

**6.1.4.1.** 거절 사유 (필수 선택)  
1: 메뉴 품절 (솔드아웃)  
2: 재료 소진  
3: 일일 수량 한정 초과  
4: 영업 시간 종료  
5: 브레이크 타임  
6: 주문 폭주로 대기 시간 초과  
7: 장비 고장/점검 중  
8: 배달/픽업 불가 상황  
9: 특정 옵션 불가  
10: 기타 (직접 입력)  

**6.1.4.2.** 거절 사유에서 제외된 항목  
1: 영업 준비 중 (오픈 전)  
　A. 오픈 전에는 주문 자체가 불가하도록 시스템 설정  
　B. 영업 시간 외 주문 차단으로 해당 사유 발생 불가  
2: 임시 휴무  
　A. 휴무일에는 주문 자체가 불가하도록 시스템 설정  
　B. 매장 휴무 설정 시 주문 차단으로 해당 사유 발생 불가  


**6.1.5.** 승인 후 파트너 취소 처리  

**6.1.5.1.** 목적  
1: 승인 후 불가피한 취소 상황에 대한 처리 프로세스  
2: 소비자에게 신속한 알림 및 자동 환불 제공  

**6.1.5.2.** 프로세스  
1: 1단계: 파트너가 [주문 취소] 버튼 클릭  
2: 2단계: 취소 사유 선택 (필수)  
3: 3단계: 취소 확인 시 즉시 처리  
　A. 소비자에게 "매장 사정으로 주문이 취소되었습니다" 문자 발송  
　B. 결제 금액 즉시 자동 환불 처리  
　C. 주문 상태: "준비중" → "파트너 취소"로 변경  

**6.1.5.3.** 파트너 책임 취소 기록  
1: 승인 후 파트너 사유 취소는 "파트너 책임 취소"로 분류  
2: 월간 파트너 책임 취소율 집계  
3: 서비스 품질 지표에 반영 (등급 산정 참고 자료)
