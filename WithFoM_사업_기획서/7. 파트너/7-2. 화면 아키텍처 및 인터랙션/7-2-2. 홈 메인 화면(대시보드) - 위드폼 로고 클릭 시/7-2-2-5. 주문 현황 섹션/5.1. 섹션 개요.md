**5.1.** 섹션 개요  


**5.1.1.** 섹션 정의  

**5.1.1.1.** 기본 정보  
1: 섹션 위치  
　A. 파트너 포털 홈 화면(대시보드)의 다섯 번째 섹션으로 배치  
　B. 결제 내역 섹션(7-2-2-1), 리워드 사용 내역 섹션(7-2-2-2), 일일 소진 보고 섹션(7-2-2-3), 컵홀더 현황 섹션(7-2-2-4) 다음에 위치  
　C. 서비스 타입에 따라 앞선 섹션이 숨김 처리될 경우 순서가 앞당겨질 수 있음  
2: 섹션 명칭  
　A. 섹션 제목: "주문 현황"  
　B. 제목 우측에 오늘 기준 주요 지표 요약 뱃지 표시 가능 (예: "오늘 45건")  


**5.1.2.** 섹션 목적  

**5.1.2.1.** 핵심 기능  
1: 오늘의 주문, 카테고리별 판매, 매출 현황을 한눈에 요약 제공  
2: 파트너가 대시보드에서 빠르게 운영 현황을 파악할 수 있도록 지원  
3: 각 카드 클릭 시 상세 페이지로 이동하여 세부 정보 확인 가능  


**5.1.3.** 섹션 레이아웃  

**5.1.3.1.** 화면 구성  
1: 섹션 헤더  
　A. 좌측: 섹션 제목 "주문 현황"  
　B. 우측: "전체 보기" 링크 (클릭 시 주문 관리 페이지로 이동)  
2: 섹션 본문  
　A. 3개의 요약 카드로 구성  
　B. 가로 배치 (데스크톱) 또는 세로 스택 (모바일)  
　C. 각 카드는 클릭 가능한 형태  

**5.1.3.2.** 3개 요약 카드 구성  
1: 주문 현황 카드  
　A. 매장 이용 주문 건수  
　B. 테이크 아웃 주문 건수  
　C. 토탈 주문 건수  
2: 카테고리별 판매 현황 카드  
　A. 메뉴 대분류 카테고리별 (커피, 논커피, 디저트 등) 판매 갯수  
　B. 각 카테고리별 매장 이용 / 테이크 아웃 구분 표시  
　C. 토탈 판매 갯수  
3: 매출 현황 카드  
　A. 주문 금액 (리워드 적용 전)  
　B. 리워드 적용 금액  
　C. 최종 결제 금액  


**5.1.4.** 데이터 갱신 주기  

**5.1.4.1.** 실시간 갱신  
1: 갱신 방식  
　A. 위드폼 오더 이용 시: 실시간 WebSocket 연동으로 즉시 갱신  
　B. 위드폼 오더 미사용 시: 수동 입력 데이터 기반 (일일 소진 보고 연동)  
2: 갱신 시점  
　A. 신규 주문 발생 시 즉시 반영  
　B. 주문 상태 변경 시 즉시 반영 (승인, 취소, 완료)  
　C. 리워드 사용/취소 시 매출 현황 즉시 반영  


• 참조: 7-2-9. 주문 관리, 7-2-10. 결제 관리
