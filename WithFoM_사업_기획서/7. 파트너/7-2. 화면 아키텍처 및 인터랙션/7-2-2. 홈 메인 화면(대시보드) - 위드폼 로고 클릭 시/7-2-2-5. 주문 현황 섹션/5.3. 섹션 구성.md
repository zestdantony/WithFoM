**5.3.** 섹션 구성  


**5.3.1.** 섹션 헤더 영역  

**5.3.1.1.** 좌측 헤더  
1: 섹션 제목  
　A. "주문 현황" 텍스트  
　B. 폰트: 18px, 볼드  
2: 요약 뱃지 (선택적)  
　A. 제목 우측에 오늘 주문 건수 뱃지 표시  
　B. 예: "오늘 45건"  
　C. 색상: 브랜드 컬러 배경  

**5.3.1.2.** 우측 헤더  
1: 전체 보기 링크  
　A. "전체 보기" 텍스트 또는 ">" 아이콘  
　B. 클릭 시 주문 관리 페이지로 이동  


**5.3.2.** 주문 현황 카드 (1번째 카드)  

**5.3.2.1.** 카드 레이아웃  
1: 카드 제목  
　A. "주문" 또는 "주문 현황"  
　B. 폰트: 14px, 볼드  
2: 주문 유형별 건수  
　A. 매장 이용 주문 건수  
　　- 라벨: "매장"  
　　- 값: 숫자 + "건" (예: "28건")  
　B. 테이크 아웃 주문 건수  
　　- 라벨: "테이크 아웃"  
　　- 값: 숫자 + "건" (예: "17건")  
3: 토탈 주문 건수  
　A. 라벨: "합계"  
　B. 값: 숫자 + "건" (예: "45건")  
　C. 폰트: 28px, 볼드 (강조)  
4: 전일 대비  
　A. 어제 동일 시간 대비 증감률  
　B. 증가: 초록색 ↑, 감소: 빨간색 ↓  
　C. 예: "↑ 12% (어제 40건)"  

**5.3.2.2.** 표시 구조  
1: 상단 영역  
　A. 토탈 주문 건수 (대형 숫자)  
　B. 전일 대비 증감률  
2: 하단 영역  
　A. 매장 이용 / 테이크 아웃 건수를 가로로 나란히 배치  
　B. 각각 아이콘 + 라벨 + 건수 형태  


**5.3.3.** 카테고리별 판매 현황 카드 (2번째 카드)  

**5.3.3.1.** 카드 레이아웃  
1: 카드 제목  
　A. "판매 현황" 또는 "카테고리별 판매"  
　B. 폰트: 14px, 볼드  
2: 메뉴 대분류 카테고리별 판매 갯수  
　A. 커피 카테고리  
　　- 매장: 숫자 + "잔" (예: "32잔")  
　　- 테이크 아웃: 숫자 + "잔" (예: "18잔")  
　　- 소계: 숫자 + "잔" (예: "50잔")  
　B. 논커피 카테고리  
　　- 매장: 숫자 + "잔" (예: "8잔")  
　　- 테이크 아웃: 숫자 + "잔" (예: "5잔")  
　　- 소계: 숫자 + "잔" (예: "13잔")  
　C. 디저트 카테고리  
　　- 매장: 숫자 + "개" (예: "6개")  
　　- 테이크 아웃: 숫자 + "개" (예: "3개")  
　　- 소계: 숫자 + "개" (예: "9개")  
　D. 기타 카테고리 (매장별로 추가 가능)  
3: 토탈 판매 갯수  
　A. 라벨: "총 판매"  
　B. 값: 전체 카테고리 합계 (예: "72개")  
　C. 폰트: 28px, 볼드 (강조)  

**5.3.3.2.** 표시 구조  
1: 상단 영역  
　A. 토탈 판매 갯수 (대형 숫자)  
2: 중앙 영역  
　A. 카테고리별 리스트 형태  
　B. 각 카테고리 행: 카테고리명 | 매장 | T/O | 소계  
　C. 테이블 또는 리스트 뷰 형태  
3: 카테고리 축약 표시  
　A. 카드 내에서는 상위 3개 카테고리만 표시  
　B. "더보기" 클릭 시 전체 카테고리 확인 가능  


**5.3.4.** 매출 현황 카드 (3번째 카드)  

**5.3.4.1.** 카드 레이아웃  
1: 카드 제목  
　A. "매출" 또는 "매출 현황"  
　B. 폰트: 14px, 볼드  
2: 주요 지표  
　A. 오늘 최종 결제 금액 (대형 숫자)  
　B. 예: "₩1,250,000"  
　C. 폰트: 28px, 볼드  
3: 상세 내역  
　A. 주문 금액 (리워드 적용 전)  
　B. 리워드 적용 금액 (할인된 금액)  
　C. 최종 결제 금액  
　D. 예:  
　　- 주문 금액: ₩1,450,000  
　　- 리워드 할인: -₩200,000  
　　- 최종 결제: ₩1,250,000  
4: 전일 대비  
　A. 어제 동일 시간 대비 증감률  
　B. 예: "↑ 8% (어제 ₩1,150,000)"  

**5.3.4.2.** 상세 내역 표시 방식  
1: 기본 표시  
　A. 최종 결제 금액만 대형으로 표시  
　B. 상세 내역은 작은 글씨로 하단에 표시  
2: 확장 표시 (선택적)  
　A. 카드 호버 시 상세 내역 패널 확장  
　B. 또는 클릭 시 상세 페이지에서 확인
