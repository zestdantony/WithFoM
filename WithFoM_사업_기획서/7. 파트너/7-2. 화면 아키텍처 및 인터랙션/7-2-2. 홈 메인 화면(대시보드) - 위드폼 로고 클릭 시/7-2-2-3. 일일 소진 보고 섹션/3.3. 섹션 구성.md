**3.3.** 섹션 구성  


**3.3.1.** 섹션 헤더 영역  

**3.3.1.1.** 좌측 헤더  
1: 섹션 제목  
　A. "일일 소진 보고" 텍스트  
　B. 폰트: 18px, 볼드  
2: 보고 상태 뱃지  
　A. 제목 우측에 상태 뱃지 표시  
　B. 보고 필요: 빨간색 뱃지 "보고 필요"  
　C. 보고 완료: 초록색 뱃지 "보고 완료"  

**3.3.1.2.** 우측 헤더  
1: 상세 보기 링크  
　A. "상세 보기" 텍스트 또는 ">" 아이콘  
　B. 클릭 시 캠페인 소진 보고 페이지로 이동  


**3.3.2.** 날짜 표시 영역  

**3.3.2.1.** 당일 날짜 표시  
1: 기본 표시  
　A. 당일 날짜와 요일 표시 (예: "2026년 1월 28일 (수)")  
　B. 위치: 섹션 본문 상단  
　C. 폰트: 16px, 볼드  
2: 스타일  
　A. 배경: 연한 회색 라운드 박스  
　B. 아이콘: 캘린더 아이콘 좌측 배치  

**3.3.2.2.** 미보고 시 전일 날짜 추가 표시 (UI/UX 설계)  
1: 표시 조건  
　A. 전일 소진량 미보고 상태에서 자정이 지난 경우  
　B. 익일 영업 시작 후 2시간까지만 표시  
　C. 2시간 경과 후 전일 날짜 미표기 + 입력 불가 처리  
2: 듀얼 날짜 카드 UI (좌우 배치)  
　A. 좌측 카드: 전일 날짜 (빨간색 테두리, "미보고" 강조)  
　　- 예: "1월 27일 (화) - 미보고"  
　　- 입력 필드 및 제출 버튼 활성화  
　B. 우측 카드: 당일 날짜 (일반 테두리)  
　　- 예: "1월 28일 (수) - 오늘"  
　　- 전일 보고 완료 전까지 비활성화 (회색 처리)  
　C. 시각적 구분: 좌측 전일 카드에 붉은 점멸 효과 또는 주의 아이콘  
3: 보고 흐름 안내  
　A. 안내 문구: "어제 소진 보고를 먼저 완료해 주세요"  
　B. 전일 보고 완료 시 자동으로 오늘 카드 활성화  
4: 수정 마감 임박 알림  
　A. 수정 가능 시간 1시간 전부터 남은 시간 표시  
　B. 예: "1시간 23분 후 입력 마감"  
　C. 색상: 주황색 → 빨간색 (30분 이하)  

**3.3.2.3.** 24시간 영업장 처리  
1: 기준 시간 설정  
　A. 24시간 영업장은 다음 날 오전 11:00를 소진량 보고 마감 기준으로 적용  
　B. 영업일 변경 시점: 자정 (00:00) 기준  
　C. 전일 소진량 보고 마감: 오전 11:00  
2: 전일 소진량 보고 가능 여부  
　A. 오전 11:00 이전: 전일 소진량 보고 가능 (전일 카드 활성화)  
　B. 오전 11:00 이후: 전일 보고 마감, 전일 카드 비활성화  


**3.3.3.** 전일 소진량 영역  

**3.3.3.1.** 표시 정보  
1: 전일 소진량  
　A. 라벨: "어제 소진량"  
　B. 값: 숫자 (예: "127개")  
　C. 미보고 시: "미보고" 텍스트 (빨간색)  
2: 수정 버튼  
　A. 전일 소진량 옆에 [수정] 버튼  
　B. 수정 가능 시간 내에만 활성화  
　C. 수정 가능 시간: 익일 영업 시작 후 2시간까지  
3: 수정 안내 문구  
　A. 수정 버튼 하단에 작은 글씨로 안내  
　B. "실수 입력 시 익일 오픈 후 2시간까지 수정 가능"  


**3.3.4.** 오늘 소진량 입력 영역  

**3.3.4.1.** 입력 필드  
1: 입력창  
　A. 라벨: "오늘 소진량"  
　B. 플레이스홀더: "소진 수량을 입력하세요"  
　C. 입력 형식: 숫자만 입력 가능  
　D. 숫자 키패드 자동 표시 (모바일)  
2: 입력 크기  
　A. 입력창 너비: 섹션 너비의 60%  
　B. 높이: 48px  
　C. 폰트 크기: 18px  

**3.3.4.2.** 제출 버튼  
1: 버튼 배치  
　A. 입력창 우측에 배치  
　B. 버튼 텍스트: "제출"  
2: 버튼 스타일  
　A. 배경색: 브랜드 컬러  
　B. 텍스트: 흰색, 볼드  
　C. 크기: 입력창과 동일 높이 (48px)  


**3.3.5.** 보고 완료 후 표시  

**3.3.5.1.** 완료 상태 UI  
1: 입력창 변경  
　A. 입력창 비활성화 (읽기 전용)  
　B. 제출된 소진량 값 표시  
　C. 배경색: 연한 초록색  
2: 버튼 변경  
　A. 제출 버튼 → "수정" 버튼으로 변경 (수정 가능 시간 내)  
　B. 수정 가능 시간 경과 시 버튼 숨김 처리  
3: 완료 시간 표시  
　A. 하단에 "00:00에 보고 완료" 텍스트 표시  

**3.3.5.2.** 완료 상태 유지 시간  
1: 일반 매장  
　A. 당일 보고 완료 상태는 다음 날 영업 시작 시간 전까지 유지  
　B. 다음 날 영업 시작 시 새로운 당일 보고 상태로 초기화  
2: 24시간 영업장  
　A. 당일 보고 완료 상태는 다음 날 오전 11:00까지 유지  
　B. 오전 11:00 이후 새로운 당일 보고 상태로 초기화  


**3.3.6.** 캠페인 매칭 대기 시 표시  

**3.3.6.1.** 비활성화 상태 UI  
1: 섹션 전체 비활성화  
　A. 섹션 배경: 연한 회색  
　B. 모든 입력 필드 및 버튼 비활성화  
2: 안내 문구  
　A. 중앙에 "캠페인 매칭 대기 중입니다" 표시  
　B. 하단에 다음 캠페인 매칭 예정일 표시 (있는 경우)  
　C. 예: "다음 캠페인 예정: 2026년 2월 15일"  
3: 아이콘  
　A. 시계 또는 대기 아이콘 표시  


**3.3.7.** 휴무일 표시  

**3.3.7.1.** 휴무일 상태 UI  
1: 자동 처리 안내  
　A. 입력 필드 비활성화  
　B. "휴무일 (자동 0개 처리됨)" 안내 표시  
　C. 배경색: 연한 회색  
2: 상태 뱃지  
　A. 뱃지 색상: 회색  
　B. 텍스트: "휴무일"
