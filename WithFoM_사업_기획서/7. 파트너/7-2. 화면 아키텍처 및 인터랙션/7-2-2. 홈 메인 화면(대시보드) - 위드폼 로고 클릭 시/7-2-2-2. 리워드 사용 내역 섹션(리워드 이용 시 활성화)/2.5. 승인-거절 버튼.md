**2.5.** 승인-거절 버튼  


**2.5.1.** 버튼 노출 조건  

**2.5.1.1.** 위드폼 오더 사용 여부에 따른 버튼 노출  
1: 위드폼 오더 결제 연동 시  
　A. 리워드 섹션에서 승인/거절 버튼 미노출  
　B. 결제 내역 섹션(7-2-2-1)에서 주문 승인/거절 시 리워드도 자동 처리  
　C. 리워드 카드에 "결제 연동" 라벨만 표시  
2: 위드폼 오더 미사용 (리워드 단독 사용) 시  
　A. 리워드 카드에 [승인] / [거절] 버튼 노출  
　B. 파트너가 직접 리워드 승인/거절 처리  


**2.5.2.** 버튼 구성  

**2.5.2.1.** 기본 버튼 구조  
1: 버튼 배치  
　A. 카드 하단 영역에 가로로 배치  
　B. 좌측: [승인] 버튼  
　C. 우측: [거절] 버튼  
2: 버튼 크기  
　A. 각 버튼 너비: 카드 너비의 45%  
　B. 버튼 간 간격: 10%  
　C. 버튼 높이: 44px (터치 영역 확보)  


**2.5.3.** 승인 버튼  

**2.5.3.1.** 버튼 스타일  
1: 기본 상태  
　A. 배경색: 브랜드 컬러 (초록색 계열)  
　B. 텍스트: "승인" (흰색, 볼드)  
　C. 아이콘: ✓ 체크 아이콘 (선택적)  
2: 눌림 상태  
　A. 배경색: 진한 초록색  
　B. 살짝 축소 효과 (scale: 0.98)  
3: 처리 중 상태  
　A. 버튼 비활성화  
　B. 로딩 스피너 표시  
　C. 텍스트: "처리 중..."  

**2.5.3.2.** 버튼 동작  
1: 클릭 시 동작  
　A. 해당 리워드 사용 승인 처리  
　B. 소비자에게 승인 완료 푸시 알림 발송  
　C. 카드 상태 "승인"으로 변경  
2: 승인 완료 후  
　A. 버튼 영역이 "사용 완료" 텍스트로 대체  
　B. 카드 테두리 초록색으로 변경  
　C. 토스트 메시지: "리워드 사용이 승인되었습니다"  
3: 승인 후 결제 처리  
　A. 파트너는 리워드 적용 금액을 확인하여 결제 진행  
　B. 결제 전 금액 입력 시 최종 결제 금액 자동 계산  


**2.5.4.** 거절 버튼  

**2.5.4.1.** 버튼 스타일  
1: 기본 상태  
　A. 배경색: 회색 (투명 또는 연한 회색)  
　B. 텍스트: "거절" (진한 회색)  
　C. 테두리: 1px 회색 보더  
2: 눌림 상태  
　A. 배경색: 연한 빨간색  
　B. 텍스트 색상: 빨간색  

**2.5.4.2.** 버튼 동작  
1: 클릭 시 동작  
　A. 거절 사유 선택 팝업 표시  
　B. 사유 선택 후 거절 처리  
2: 거절 사유 목록  
　A. 쿠폰/리워드가 잘못 사용되었을 때  
　B. 매장 사정으로 리워드를 사용하지 못할 때  
　C. 사용 조건 미충족 (최소 주문 금액 등)  
　D. 해당 메뉴 품절  
　E. 기타 (직접 입력)  
3: 거절 완료 후  
　A. 카드 리스트에서 해당 카드 제거 (페이드아웃)  
　B. 소비자에게 거절 알림 발송 (사유 포함)  
　C. 토스트 메시지: "리워드 사용이 거절되었습니다"  
　D. 거절된 리워드는 소비자에게 자동 복구  


**2.5.5.** 자동 승인  

**2.5.5.1.** 자동 승인 로직

2: 자동 승인 표시

3: 자동 승인 후

**2.5.6.** 리워드 복구 정책
