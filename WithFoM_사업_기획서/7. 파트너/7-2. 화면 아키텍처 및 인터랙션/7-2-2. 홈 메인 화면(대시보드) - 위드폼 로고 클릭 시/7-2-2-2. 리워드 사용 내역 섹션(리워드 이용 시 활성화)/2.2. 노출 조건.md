**2.2.** 노출 조건  


**2.2.1.** 섹션 노출 조건  

**2.2.1.1.** 리워드 기능 활성화  
1: 필수 조건  
　A. 리워드 사용 내역 섹션은 "리워드 기능 ON" 상태일 때만 노출  
　B. 리워드 기능 OFF 상태에서는 섹션 자체가 숨김 처리  
2: 서비스 타입 연동  
　A. "리워드 On/Off"는 서비스 타입(리워드형/오더형/복합형) 설정과 연동  
　B. 리워드형 또는 복합형 선택 시 리워드 기능 자동 활성화  
　C. 오더형만 선택한 경우 리워드 기능 비활성화 (수동 활성화 가능)  


**2.2.2.** 서비스 타입별 노출  

**2.2.2.1.** 타입별 섹션 노출 조합  

| 서비스 타입 | 결제 내역 섹션 | 리워드 사용 내역 섹션 |
|---|---|---|
| 리워드형 | 숨김 | 첫 번째 섹션으로 노출 |
| 오더형 | 첫 번째 섹션 | 숨김 (리워드 기능 OFF 시) |
| 복합형 (권장) | 첫 번째 섹션 | 두 번째 섹션 |


**2.2.3.** 카드 노출 조건  

**2.2.3.1.** 카드 생성 조건  
1: 소비자 리워드 사용 요청 시  
　A. 소비자가 매장에서 "사용하기" 버튼 클릭  
　B. 해당 매장의 대시보드에 승인 요청 카드 즉시 생성  
2: 스탬프 적립 요청 시  
　A. 소비자가 스탬프 적립 가능 조건 충족  
　B. 파트너 대시보드에 "스탬프 적립 요청" 카드 생성  
3: 위드폼 오더 결제 시 리워드 사용  
　A. 결제 내역 카드와 동시에 리워드 사용 카드 생성  


**2.2.4.** 섹션 우선순위  

**2.2.4.1.** 대시보드 섹션 순서  
1: 복합형 파트너  
　A. 1순위: 결제 내역 섹션 (7-2-2-1)  
　B. 2순위: 리워드 사용 내역 섹션 (7-2-2-2)  
　C. 3순위: 일일 소진 보고 섹션 (7-2-2-3)  
2: 리워드형 파트너  
　A. 1순위: 리워드 사용 내역 섹션 (7-2-2-2)  
　B. 2순위: 일일 소진 보고 섹션 (7-2-2-3)
