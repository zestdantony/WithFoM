**2.6.** 액션 버튼 및 화면 전환  


**2.6.1.** 섹션 제목 클릭 시  

**2.6.1.1.** 이동 경로  
1: 클릭 대상  
　A. 섹션 제목 "리워드 사용 내역" 텍스트  
　B. 제목 우측의 실시간 대기 건수 뱃지 (예: "(2)")  
2: 이동 목적지  
　A. 사이드바 > 리워드•스탬프 카테고리 > 내역 페이지로 이동  
　B. 내역 페이지에서 전체 리워드 사용 목록 확인 가능  
3: 이동 시 필터 적용  
　A. 상태 필터: "승인 대기" 상태로 자동 필터링  
　B. 기간 필터: "오늘" 자동 선택  


**2.6.2.** 전체 보기 링크 클릭 시  

**2.6.2.1.** 이동 경로  
1: 클릭 대상  
　A. 섹션 헤더 우측의 "전체 보기" 텍스트 링크  
　B. 또는 ">" 화살표 아이콘  
2: 이동 목적지  
　A. 사이드바 > 리워드•스탬프 카테고리 > 내역 페이지로 이동  
　B. 필터 없이 전체 리워드 사용 목록 표시  


**2.6.3.** 개별 카드 클릭 시  

**2.6.3.1.** 카드 본문 영역 클릭  
1: 클릭 대상  
　A. 카드 내 버튼 영역과 금액 입력 영역을 제외한 모든 영역  
　B. 리워드 정보, 사용자 정보 영역  
2: 이동 목적지  
　A. 사이드바 > 리워드•스탬프 카테고리 > 내역 페이지로 이동  
　B. 해당 리워드 사용 상세 패널이 자동으로 열린 상태로 이동  
3: 리워드 상세 패널 내용  
　A. 리워드 전체 정보 (유형, 이름, 혜택 내용)  
　B. 사용 조건 상세 (유효기간, 최소 주문 금액 등)  
　C. 사용자 정보 (닉네임, 회원 등급, 사용 이력)  
　D. 상태별 버튼 표시  

**2.6.3.2.** 상태별 패널 버튼 (리워드 단독 사용 시)  
1: 승인 대기 상태 카드 클릭 시  
　A. [승인] 버튼 / [거절] 버튼 표시  
　B. 거절 시 사유 선택 팝업  
2: 승인 완료 상태 카드 클릭 시  
　A. "사용 완료" 상태 텍스트 표시  
　B. [취소] 버튼 표시 (예외 상황용)  
　C. 취소 시 사유 입력 필요, 리워드 소비자 복구  


**2.6.4.** 결제 연동 카드 클릭 시  

**2.6.4.1.** 연동 주문 이동  
1: 클릭 대상  
　A. 결제 연동 카드의 주문 번호 영역  
2: 이동 목적지  
　A. 사이드바 > 주문 관리 카테고리 > 주문 관리 페이지  
　B. 해당 주문 상세 패널 자동 열림  
3: 버튼 처리  
　A. 리워드 섹션에서 승인/거절 버튼 미노출  
　B. "결제 섹션에서 처리" 안내 텍스트 표시  
　C. 결제 섹션에서 주문 거절 시 리워드도 자동 거절 및 복구  


**2.6.5.** 금액 입력 필드 동작  

**2.6.5.1.** 결제 전 금액 입력  
1: 입력 필드 위치  
　A. 리워드 정보 영역 아래, 버튼 영역 위  
　B. 모든 리워드 카드에서 사용 가능  
2: 입력 동작  
　A. 필드 클릭/탭 시 숫자 키패드 표시  
　B. 숫자 입력 시 실시간으로 최종 결제 금액 계산  
　C. 리워드 사용 전/후 모두 입력 가능  
3: 계산 결과 표시  
　A. 입력 필드 아래에 "최종 결제 금액" 표시  
　B. 리워드 할인 금액 차감 후 결제 금액 표시  
　C. 음수 발생 시 0원 표시 + "무료" 안내  


**2.6.6.** 빈 상태 UI  

**2.6.6.1.** 요청이 없는 경우  
1: 빈 상태 표시  
　A. 카드 영역에 빈 상태 컴포넌트 표시  
　B. 아이콘: 선물 상자 또는 쿠폰 형태의 회색 아이콘  
　C. 메인 문구: "리워드 사용 요청이 없습니다"  
　D. 보조 문구: "고객이 리워드를 사용하면 여기에 표시됩니다"  
2: 실시간 갱신  
　A. WebSocket으로 신규 요청 수신 시 빈 상태 컴포넌트 제거  
　B. 카드 리스트로 즉시 전환  

**2.6.6.2.** 일일 초기화 시  
1: 초기화 시점  
　A. 매일 해당 매장의 영업 시간 시작 시 자동 초기화  
　B. 초기화 기준: 매장 설정에 등록된 영업 시작 시간 (필수 설정 항목)  
2: 초기화 동작  
　A. 전일 승인/취소 완료된 리워드 사용 카드 제거  
　B. 리워드 사용 내역 섹션: 빈 카드 리스트로 표시  
　C. 안내 문구: "리워드 사용 요청이 없습니다" 노출  
3: 데이터 보존  
　A. 초기화된 리워드 사용 내역은 내역 페이지에서 계속 조회 가능  
　B. 취소 이력도 별도 저장되어 조회 가능  
　C. 정산 데이터에는 영향 없음  


**2.6.7.** 팝업/모달 UI 전환  

**2.6.7.1.** 거절 확인 팝업 (리워드 단독 사용 시)  
1: 표시 조건  
　A. 승인 대기 상태 카드에서 [거절] 버튼 클릭 시  
2: 팝업 구성  
　A. 팝업 제목: "리워드 사용을 거절하시겠습니까?"  
　B. 팝업 내용: "거절 시 고객에게 알림이 발송되며, 리워드가 복구됩니다."  
　C. 거절 사유 선택 영역 (필수, 5개 항목)  
　D. 버튼: [취소] / [거절 확인]  
3: 거절 사유 목록  
　A. 쿠폰/리워드가 잘못 사용되었을 때  
　B. 매장 사정으로 리워드를 사용하지 못할 때  
　C. 사용 조건 미충족 (최소 주문 금액 등)  
　D. 해당 메뉴 품절  
　E. 기타 (직접 입력)  
4: 애니메이션  
　A. 팝업 등장: 페이드인 + 스케일업 (200ms)  
　B. 배경 딤 처리 (opacity: 0.5)  

**2.6.7.2.** 취소 확인 팝업 (승인 완료 후)  
1: 표시 조건  
　A. 승인 완료 상태에서 [취소] 버튼 클릭 시  
2: 팝업 구성  
　A. 팝업 제목: "리워드 사용을 취소하시겠습니까?"  
　B. 팝업 내용: "취소 시 고객에게 해당 리워드가 복구됩니다."  
　C. 취소 사유 입력 영역  
　D. 버튼: [취소] / [취소 확인]  
3: 팝업 닫기  
　A. [취소] 버튼 클릭  
　B. 팝업 외부 영역 클릭  
　C. ESC 키 입력
