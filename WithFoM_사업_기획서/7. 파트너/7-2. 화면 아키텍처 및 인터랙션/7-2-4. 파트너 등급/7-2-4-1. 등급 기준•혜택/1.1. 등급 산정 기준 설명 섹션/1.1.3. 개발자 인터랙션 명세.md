**1.1.3.** 개발자 인터랙션 명세  


**1.1.3.1.** 데이터 조회 API  

1: 등급 기준 조회  
　A. 엔드포인트: GET /api/v1/partner/grade/criteria  
　B. 요청 파라미터: partnerId (JWT에서 자동 추출)  

2: 응답 구조  

```json
{
  "criteria": {
    "reportRate": {
      "bronze": { "min": 0, "max": 84, "actual": 70 },
      "silver": { "min": 85, "max": 99, "actual": 70 },
      "gold": { "min": 100, "max": 100, "actual": 90 },
      "platinum": { "min": 100, "max": 100, "actual": 90 }
    },
    "scanRate": {
      "bronze": { "min": 0, "max": 79 },
```

**1.1.3.2.** 컴포넌트 상태 관리  

1: 확장/축소 상태 (expandedSections)  
　A. reportRate: boolean  
　B. scanRate: boolean  
　C. withfomOrder: boolean  

2: 툴팁 상태 (tooltipState)  
　A. visible: boolean  
　B. position: { x, y }  


**1.1.3.3.** 애니메이션 정의  

1: 확장/축소 애니메이션  
　A. duration: 250ms  
　B. easing: ease-in-out  
　C. property: height, opacity  

2: 툴팁 애니메이션  
　A. 등장: fadeIn 200ms  
　B. 사라짐: fadeOut 150ms  


**1.1.3.4.** 접근성 요구사항  

1: ARIA 레이블  
　A. 섹션: role="region", aria-labelledby="grade-criteria-title"  
　B. 확장 영역: aria-expanded="true/false"  
　C. 툴팁: role="tooltip"  

2: 키보드 내비게이션  
　A. Tab: 각 지표 영역 간 이동  
　B. Enter/Space: 확장/축소 토글  
　C. Escape: 툴팁 닫기  


**1.1.3.5.** 에러 처리  

1: 에러 코드

2: 사용자 메시지  
　A. "등급 기준 정보를 불러오지 못했습니다"
