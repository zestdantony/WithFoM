**1.6.** 스탬프 수동 적립  


**1.6.1.** 개요  

**1.6.1.1.** 수동 적립 목적  
　1: 위드폼 오더를 통하지 않고 결제한 소비자에게 스탬프 적립 제공  
　2: 시스템에서 결제 금액 확인 불가 시 파트너가 수동으로 적립 처리  
　3: 위드폼 오더 사용을 유도하기 위해 자동 적립 대비 다소 번거로운 프로세스 적용  

**1.6.1.2.** 적용 대상  
　1: 리워드 기능을 사용하는 모든 파트너  
　2: 위드폼 오더 미사용 시 수동 적립만 가능  
　3: 복합형(오더+리워드) 파트너도 오더 외 결제 시 수동 적립 사용  


**1.6.2.** 수동 적립 패널 UI  

**1.6.2.1.** 패널 오픈 방식  
　1: 스탬프 현황 섹션에서 [수동 적립] 버튼 클릭  
　2: 우측 슬라이드 패널 또는 모달 형태로 오픈  
　3: 패널 닫기: X 버튼, 외부 클릭, ESC 키  

**1.6.2.2.** 패널 레이아웃  
　1: 상단: 패널 제목 "스탬프 수동 적립"  
　2: Step 1: 소비자 검색  
　3: Step 2: 결제 금액 입력  
　4: Step 3: 적립 확인 및 승인  
　5: 하단: [취소] / [적립] 버튼  


**1.6.3.** Step 1: 소비자 검색  

**1.6.3.1.** 검색 방식  
　1: 검색 입력창  
　　A. 플레이스홀더: "닉네임 또는 전화번호 뒤 4자리"  
　　B. 숫자 4자리 입력 시 전화번호 검색으로 자동 전환  
　2: QR 코드 스캔 버튼  
　　A. 소비자 앱 > 마이페이지 > "스탬프 적립 QR" 제시  
　　B. 파트너가 QR 스캔 시 자동 검색  

**1.6.3.2.** 검색 결과 표시  
　1: 소비자 정보 카드  
　　A. 닉네임 (마스킹된 이름)  
　　B. 전화번호 뒷자리 (예: ****1234)  
　　C. 현재 스탬프 보유 개수 / 혜택 기준 개수 (예: 7/10개)  
　　D. 단골 스토어 지정 여부 (✅ 지정됨 / ❌ 미지정)  
　2: 검색 결과 없음  
　　A. "검색 결과가 없습니다" 안내  
　　B. "단골 스토어 지정 후 스탬프 적립이 가능합니다" 안내  

**1.6.3.3.** 단골 스토어 미지정 시  
　1: 스탬프 적립 불가 안내  
　2: 소비자에게 단골 스토어 지정 방법 안내 유도  
　3: [적립] 버튼 비활성화  


**1.6.4.** Step 2: 결제 금액 입력  

**1.6.4.1.** 결제 금액 입력  
　1: 입력 필드  
　　A. 라벨: "결제 금액"  
　　B. 플레이스홀더: "실결제 금액 입력"  
　　C. 입력 형식: 숫자 (원 단위)  
　　D. 숫자 키패드 자동 표시  
　2: 입력 시 실시간 계산  
　　A. 스탬프 적립 기준 금액과 비교  
　　B. 적립 가능 스탬프 개수 자동 계산  
　　C. 예시: "15,000원 → 스탬프 5개 적립 가능"  

**1.6.4.2.** 적립 기준 미달 시  
　1: 경고 메시지 표시  
　2: "최소 3,000원 이상 결제 시 스탬프 적립이 가능합니다"  
　3: [적립] 버튼 비활성화  


**1.6.5.** Step 3: 적립 확인 및 승인  

**1.6.5.1.** 적립 결과 미리보기  
　1: 적립 전후 스탬프 현황  
　　A. 현재 보유: N개  
　　B. 적립 예정: N개  
　　C. 적립 후 총: N개  
　2: 혜택 도달 안내 (해당 시)  
　　A. 단일 혜택 설정 파트너: "🎁 10개 혜택 도달! 쿠폰 자동 발급 예정"  
　　B. 다중 혜택 설정 파트너: 혜택 자동 발급 없음 (소비자가 앱에서 혜택 선택)  

**1.6.5.2.** 적립 버튼 동작  
　1: [적립] 버튼 클릭 시  
　　A. 스탬프 적립 처리  
　　B. 소비자에게 적립 완료 푸시 알림 발송  
　　C. 파트너에게 토스트 메시지: "스탬프 N개가 적립되었습니다"  
　　D. 패널 자동 닫힘 (설정에 따라)  
　2: 혜택 쿠폰 발급 (단일 혜택 설정 시)  
　　A. 혜택 기준 도달 시 쿠폰 자동 발급  
　　B. 사용된 스탬프만 차감 (잔여 스탬프 유지)  
　　C. 소비자에게 쿠폰 발급 알림  


**1.6.6.** 수동 적립 내역 관리  

**1.6.6.1.** 적립 로그 저장  
　1: 저장 항목  
　　A. 적립 일시  
　　B. 적립 직원 계정 (staff ID)  
　　C. 소비자 정보 (닉네임, 전화번호 뒷자리)  
　　D. 결제 금액  
　　E. 적립 스탬프 개수  
　　F. 적립 유형: "수동"  
　2: 조회 위치  
　　A. 7-2-8-2. 내역 페이지에서 "적립 유형" 필터로 조회  
　　B. 수동 적립 / 자동 적립 구분 표시  

**1.6.6.2.** 대표 계정 건별 보고  
　1: 보고 방식  
　　A. 수동 적립 발생 시 대표 계정에 푸시 알림 (선택적)  
　　B. 대표가 알림 수신 여부 설정 가능 (ON/OFF)  
　　C. 설정 위치: 계정 설정 > 알림 설정 > 수동 적립 알림  
　2: 알림 내용  
　　A. "수동 스탬프 적립: 김직원 → 커피좋아님(1234) 3개, 9,000원"  
　　B. 알림 클릭 시 내역 상세 페이지로 이동  

**1.6.6.3.** 이상 패턴 경고 알림  
　1: 자동 감지 조건  
　　A. 일일 수동 적립 비율이 자동 적립의 30% 초과 시  
　　B. 동일 소비자에게 같은 직원이 1일 5회 이상 적립 시  
　　C. 결제 금액 대비 과다 적립 패턴 감지  
　2: 경고 알림  
　　A. 대표 계정에 경고 푸시 알림 발송  
　　B. 의심 건 상세 내역 확인 링크 포함  


**1.6.7.** 적립 취소 및 정정  

**1.6.7.1.** 취소 기능  
　1: 취소 가능 조건  
　　A. 적립 후 일정 시간 내 (예: 10분)  
　　B. 또는 당일 내 취소 가능  
　2: 취소 방법  
　　A. 7-2-8-2. 내역 페이지에서 해당 적립 건 선택  
　　B. [취소] 버튼 클릭  
　　C. 취소 사유 입력 (필수)  
　3: 취소 처리  
　　A. 적립된 스탬프 차감  
　　B. 소비자에게 취소 알림 발송  
　　C. 취소 이력 별도 저장  


**1.6.8.** 리포트 (멤버십별 제공)  

**1.6.8.1.** 베이직  
　1: 수동 적립 기본 로그 저장  
　2: 내역 페이지에서 조회 가능  

**1.6.8.2.** 플러스  
　1: 일일 요약 리포트 (매일 영업 종료 후 자동 발송)  
　　A. 오늘 총 수동 적립 건수  
　　B. 자동 적립 대비 비율  
　　C. 적립자별 현황 (직원별 건수)  
　2: 리포트 확인 위치: 대시섹션 알림 또는 이메일  

**1.6.8.3.** 프리미엄  
　1: 플러스 기능 포함  
　2: 주간/월간 수동 적립 통계  
　　A. 기간별 수동 적립 추이 차트  
　　B. 직원별 수동 적립 비교  
　　C. 이상 패턴 발생 이력  
　3: 수동 적립 분석 인사이트  
　　A. "수동 적립 비율이 높습니다. 위드폼 오더 도입을 권장합니다"
