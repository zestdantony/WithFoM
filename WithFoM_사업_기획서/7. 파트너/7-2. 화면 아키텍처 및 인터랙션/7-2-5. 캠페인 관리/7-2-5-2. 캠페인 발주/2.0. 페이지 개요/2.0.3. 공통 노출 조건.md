**2.0.3.** 공통 노출 조건  


**2.0.3.1.** 기본 노출 조건  

1: 노출 대상  
　A. 캠페인 매칭 중이거나 참여 중인 파트너에게 노출  
　B. 매칭 대기 상태인 경우 "캠페인 매칭 대기 중입니다" 안내 표시  
　C. 캠페인 이력이 없는 신규 파트너는 빈 상태 안내 표시  

2: 신규 파트너 빈 상태 안내 상세  
　A. 안내 제목: "아직 매칭된 캠페인이 없습니다"  
　B. 안내 본문: '위드폼에서 매장 조건에 맞는 캠페인을 자동으로 매칭해 드립니다.  
매칭 알림을 받으시면 참여를 확정해 주세요.'  
　C. 안내 아이콘: 빈 상자 일러스트 (캠페인 없음 상태 시각화)  
　D. 배경: 연한 회색 (#F5F5F5), 텍스트: 진한 회색 (#666666)  

3: 섹션별 노출 조건  
　A. 캠페인 참여 섹션: 매칭 대기/매칭 중/참여 중 관계없이 항상 표시  
　　a. 매칭 전: 빈 상태 안내 또는 "캠페인 매칭 대기 중입니다" 표시  
　　b. 매칭 시: 매칭 정보 + 확인/포기 버튼 표시  
　　c. 참여 중: 참여중 상태 뱃지 + 참여 중지 버튼 표시  
　B. 캠페인 현황 섹션: 캠페인 매칭 확정 후부터 표시  
　C. 배포 시작 일정 및 절차 섹션: 캠페인 매칭 확정 후부터 표시  
　D. 배포 의무 및 책임 섹션: 캠페인 매칭 확정 후부터 표시  
　E. 추가 발주 섹션: 캠페인 매칭 확정 후부터 표시  
　F. 자동 발주 임계치 설정 섹션: 캠페인 매칭 확정 후부터 표시  


**2.0.3.2.** 데이터 갱신 조건  

1: 실시간 갱신  
　A. 캠페인 매칭 알림 수신 시 즉시 반영  
　B. 입고 완료 시 즉시 반영  
　C. 배포 시작/종료 상태 변경 시 즉시 반영  

2: 주기적 갱신  
　A. 재고율: 소진 보고 입력 시마다 업데이트  
　B. 하루 평균 소진량: 추가 발주 시점에 재계산하여 업데이트  
　　a. 계산 기준: 입고일 ~ 추가 발주일 사이의 소진량 ÷ 경과 일수  
　　b. 상세 로직: 2.2.1.6. 하루 평균 소진량 표시 참조  
　C. 입고율: 입고 완료 처리 시 업데이트  


**2.0.3.3.** 반응형 레이아웃  

1: 디바이스별 레이아웃  
　A. 데스크톱(1024px 이상): 전체 레이아웃, 좌측 사이드바 + 우측 콘텐츠  
　B. 태블릿(768px~1023px): 사이드바 축소, 콘텐츠 영역 확장  
　C. 모바일(767px 이하): 단일 컬럼, 하단 네비게이션
