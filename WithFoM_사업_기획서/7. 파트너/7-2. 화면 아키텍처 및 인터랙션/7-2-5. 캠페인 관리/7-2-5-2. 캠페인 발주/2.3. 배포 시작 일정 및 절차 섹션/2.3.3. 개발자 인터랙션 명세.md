**2.3.3.** 개발자 인터랙션 명세  


**2.3.3.1.** 배포 일정 조회 API  

1: 엔드포인트  
　A. `GET /api/v1/partner/campaign/distribution/schedule`  
　B. 인증: Bearer Token (파트너 세션)  

2: 응답 JSON  

```json
{
  "schedule": {
    "campaignId": "CMP-2026-0042",
    "round": 3,
    "distributionStartDate": "2026-03-01",
    "firstReceiveDate": "2026-02-28",
    "receiveStatus": "RECEIVED",
    "distributionStatus": "READY",
    "campaignPeriod": {
      "startDate": "2026-03-01",
      "endDate": "2026-03-31"
    }
  }
}
```

3: 응답 필드  
　A. `distributionStatus`: PENDING(대기), READY(시작 가능), STARTED(배포 중), COMPLETED(완료)  
　B. `receiveStatus`: PENDING(입고 대기), SCHEDULED(입고 예정), RECEIVED(입고 완료)  


**2.3.3.2.** 배포 시작 API  

1: 엔드포인트  
　A. `POST /api/v1/partner/campaign/distribution/start`  
　B. 인증: Bearer Token (파트너 세션)  

2: 요청 Body  

```json
{
  "campaignId": "CMP-2026-0042"
}
```

3: 응답 JSON  

```json
{
  "status": "STARTED",
  "startedAt": "2026-03-01T09:00:00+09:00",
  "dailyReportEnabled": true,
  "message": "배포가 시작되었습니다"
}
```


**2.3.3.3.** 컴포넌트 상태 관리  

1: 상태 정의  

```js
const distributionState = {
  scheduleData: null,          // 배포 일정 정보
  distributionStatus: 'PENDING', // PENDING | READY | STARTED | COMPLETED
  receiveStatus: 'PENDING',     // PENDING | SCHEDULED | RECEIVED
  isStartButtonEnabled: false,  // 배포 시작 버튼 활성화 여부
  isLoading: false,
  error: null
};

const startDialogState = {
  isOpen: false,
  message: '컵홀더 배포를 시작하시겠습니까?\n배포 시작 후 일일 소진 보고를 시작해 주세요.',
  isProcessing: false
};

// 배포 시작 버튼 활성화 조건
const canStartDistribution = computed(() => {
  return distributionState.receiveStatus === 'RECEIVED'
    && distributionState.distributionStatus === 'READY'
    && isToday(distributionState.scheduleData.distributionStartDate);
});
```


**2.3.3.4.** 접근성 요구사항  

1: 배포 시작 다이얼로그  
　A. `role="alertdialog"`, `aria-modal="true"` 적용  
　B. 확인/취소 버튼에 `aria-label` 지정  
　C. 다이얼로그 열림 시 확인 버튼으로 포커스 이동  

2: 배포 상태 표시  
　A. 상태 변경 시 `aria-live="polite"` 영역으로 상태 메시지 전달  
　B. 배포 시작 버튼 비활성화 시 `aria-disabled="true"` 및 비활성 사유 tooltip 제공  


**2.3.3.5.** 에러 처리  

1: 에러 코드  
　A. `DST_001`: 배포 일정 조회 실패 → "배포 일정 정보를 불러올 수 없습니다. 잠시 후 다시 시도해 주세요." 토스트 표시  
　B. `DST_002`: 배포 시작 처리 실패 → "배포 시작 처리에 실패했습니다. 잠시 후 다시 시도해 주세요." 토스트 표시, 재시도 버튼 제공  
　C. `DST_003`: 입고 미완료 상태에서 배포 시작 시도 → "입고가 완료되지 않았습니다. 입고 완료 후 배포를 시작해 주세요." 안내 표시
