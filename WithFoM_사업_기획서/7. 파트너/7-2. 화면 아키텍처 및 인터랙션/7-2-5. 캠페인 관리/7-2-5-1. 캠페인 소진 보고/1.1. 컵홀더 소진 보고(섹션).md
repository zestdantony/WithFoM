**1.1.** 컵홀더 소진 보고(섹션)  


**1.1.1.** 섹션 제목 및 상태 표시  

**1.1.1.1.** 상태 및 색상  

　1: 상태 표시  
　　A. 섹션 제목 오른쪽 옆에 작은 카드로 완료/미완료 표시  
　　B. "오늘 보고 완료" 또는 "오늘 보고 미완료" 텍스트  

　2: 섹션 색상  
　　A. 미완료: 빨간색  
　　B. 완료: 그린 컬러  
　　C. 색상 변경은 소진 보고 제출 시 자동 반영  


**1.1.2.** 전일 소진량 표시  

**1.1.2.1.** 소진량 표시 및 수정  

　1: 전일 소진량 및 수정 버튼  
　　A. 전일 소진량 표시  
　　B. 전일 소진량 옆에 수정 버튼 생성  

　2: 수정 버튼 안내 문구  
　　A. 수정 버튼 위에 '실수 입력/누락 시 수정 가능(익일 오픈 후 2시간까지)' 문구 표시  
　　B. 수정 가능 시간까지만 활성화  

　3: 수정 가능 시간  
　　A. "익일 오픈 후 2시간"이 기준  
　　B. 기준은 매장별 설정값 (영업 시작 시간 기준)  
　　C. 24시간 매장 예외: 다음 날 오전 11:00까지 수정 가능  

　4: 수정 불가능 시간  
　　A. 수정 불가능 시간엔 팝업으로 이유 안내  
　　B. 팝업 멘트: '소진량 수정 시간 종료  
소진량 수정은 익일 오픈 후 2시간까지만 수정 가능하며,  
이후에는 정산 정확도를 위해 변경할 수 없습니다.'  

　5: 수정 결과 연동  
　　A. 수정 결과는 캘린더나 대시섹션 등에 모두 연동  
　　B. 수정 이력 기록 (수정 전/후 값 모두 저장)  


**1.1.3.** 오늘 소진량 입력창  

**1.1.3.1.** 입력 및 제출  

　1: 입력창  
　　A. 입력창 안에 '오늘 소진량을 입력 하세요' 표시  
　　B. 소진 수량(갯수) 입력  

　2: 제출 버튼  
　　A. 입력창 우측에 제출 버튼  

　3: 경고 메시지  
　　A. 제출 후 입력값이 평균 대비 1.5배 이상 차이 나면 경고 메세지 팝업  
　　B. 팝업 멘트: '입력값이 평균값을 벗어났습니다.  
맞는지 확인해 주세요.'  
　　C. 팝업창에 취소/확인 버튼 생성  
　　D. 취소 선택: 입력창으로 돌아가 수정 가능  
　　E. 확인 선택: 입력값 그대로 제출 처리  

　4: 입력값 제출 연동  
　　A. 입력값 제출 시 관련 모든 시스템에 연동  


**1.1.4.** 미보고 알림  

**1.1.4.1.** 알림 발송 조건  

　1: 소진 보고 미완료 시 알림  
　　A. 영업 종료 후 1시간 경과 시 1차 알림 발송  
　　B. 익일 오전 10시 미보고 시 2차 알림 발송  
　　C. 연속 3일 미보고 시 경고 알림 발송  

　2: 알림 메시지  
　　A. 메시지: '어제 소진 보고가 완료되지 않았습니다.  
소진 보고를 완료해 주세요.'  
　　B. 소진 보고 바로가기 버튼 제공  

　3: 연속 미보고 경고  
　　A. 연속 미보고 시 경고 메시지: '소진 보고가 연속으로 누락되고 있습니다.  
정확한 정산을 위해 소진 보고를 꼭 완료해 주세요.'  
　　B. 파트너 등급에 영향 안내  


**1.1.5.** 이상 탐지 (FDS 연동)  

**1.1.5.1.** 이상 탐지 기능  

　1: 이상 탐지 목적  
　　A. 반복적인 이상치 입력 패턴을 감지하여 정산 무결성과 데이터 신뢰도를 보호  

　2: 탐지 기준  
　　A. 전일 대비 ±150% 이상 급변이 반복되는 경우 이상 패턴으로 분류될 수 있음  
　　B. QR 스캔량과 소진량 교차 검증  


**1.1.6.** 클릭 시 이동  

**1.1.6.1.** 이동 경로  

　1: 섹션 클릭 시  
　　A. 사이드바의 일일 소진 보고 페이지로 이동  
　　B. 상세 소진 보고 화면으로 이동  


• 참조: 7-1-5-3. 영업 종료 시점 일일 소진량 입력, 7-1-5-4. 미보고 알림 수신
