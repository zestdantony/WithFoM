**2.4.3.** 개발자 인터랙션 명세  


**2.4.3.1.** 임계치 설정 API  

　1: API 엔드포인트  
　　A. PUT /api/v1/partner/campaign/threshold  
　　B. 인증: Bearer Token (파트너 세션)  

　2: 요청 파라미터  
　　A. thresholdPercent: 20~40  

　3: 응답 JSON  
　　A. thresholdPercent: number  
　　B. status: 'success' | 'fail'  


**2.4.3.2.** 임계치 현황 조회 API  

　1: API 엔드포인트  
　　A. GET /api/v1/partner/campaign/threshold  
　　B. 인증: Bearer Token (파트너 세션)  


**2.4.3.3.** 컴포넌트 상태 관리  

　1: 상태 변수  
　　A. sliderValue: number  
　　B. storeCapacity: number  
　　C. defaultOrderQuantity: number  


**2.4.3.4.** 슬라이더 구현  

　1: ThresholdSlider 설정  
　　A. min: 20, max: 40, step: 5, defaultValue: 30  

```javascript
const ThresholdSlider = {
  min: 20,
  max: 40,
  step: 5,
  defaultValue: 30,
  getLabel: (value) => `재고가 ${value}% 이하로 떨어지면 발주 알림을 받게 됩니다`
};
```


**2.4.3.5.** 접근성 요구사항  

　1: ARIA 속성  
　　A. role="slider"  
　　B. aria-valuemin: 20  
　　C. aria-valuemax: 40  
　　D. aria-valuenow: 현재 설정값  
　　E. aria-label: "자동 발주 임계치 설정"
