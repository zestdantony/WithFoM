**2.1.1.** 구성 및 표시 정보  


**2.1.1.1.** 캠페인 매칭 규칙  

1: 단일 캠페인 원칙  

　1: 동시 참여 제한  
　　A. 파트너 계정당 동시에 하나의 캠페인만 참여 가능  
　　B. 복수 캠페인 동시 참여 불가  
　　C. 현재 캠페인 종료 전까지 새로운 캠페인 매칭 불가  

　2: 캠페인 연속 매칭  
　　A. 현재 캠페인 종료 1주일 전부터 다음 캠페인 매칭 가능  
　　B. 다음 캠페인 확정 시 현재 캠페인 종료 익일부터 자동 전환  
　　C. 캠페인 간 공백 없이 연속 진행 가능  
　　D. 시스템의 부분 기간 매칭에 의해 진행 중에도 다음 캠페인이 예약될 수 있음 (참여는 현재 캠페인 종료 후 시작)  

2: 매칭 대기 상태  

　1: 매칭 대기 중 표시  
　　A. 현재 캠페인 종료 후 ~ 다음 캠페인 매칭 전까지  
　　B. "캠페인 매칭 대기 중입니다" 안내 표시  
　　C. 다음 캠페인 예정일이 있으면 함께 표시  

　2: 매칭 확정 시  
　　A. 매칭 대기 상태 종료  
　　B. 일반 캠페인 진행 상태로 전환  


**2.1.1.2.** 매칭 시 표시 항목  

1: 매칭 알림  
　A. 매칭 시, '캠페인이 매칭 되었습니다.  
지정된 일정에 맞춰 컵홀더 배포를 시작해 주세요.' 메세지 표시  

2: 캠페인 정보  
　A. 파트너의 N회차 캠페인 넘버 표시  
　B. 매칭 된 컵홀더 갯수 표시  
　C. 캠페인 기간 표시  
　D. 컵홀더 유형 표시  

3: 확인/포기 버튼 표시  
　A. 확인/포기 버튼 생성  

4: 응답 기한 및 자동 처리  
　A. 응답 기한: 매칭 알림 수신 후 익일 15시까지  
　　a. 응답 기한 카운트다운 타이머 표시 (남은 시간 실시간 표시)  
　B. 리마인드 알림: 기한 2시간 전 자동 발송 (필수)  
　　a. 알림 메시지: '캠페인 매칭 응답 기한이 2시간 남았습니다.  
참여를 확정하시려면 지금 확인해 주세요.  
미응답 시 자동 포기 처리됩니다.'  
　　b. 발송 채널: 푸시 알림 + 앱 내 알림 (필수)  
　C. 미응답 시 자동 포기 처리  
　　a. 응답 기한(익일 15시) 경과 시 시스템이 자동으로 포기 처리  
　　b. 자동 포기 사유: "TIMEOUT" (응답 기한 초과)으로 기록  
　　c. 자동 포기 알림: '캠페인 매칭 응답 기한이 초과되어  
자동으로 포기 처리되었습니다.  
다음 캠페인 매칭을 기다려 주세요.'  
　　d. 해당 캠페인 재매칭 불가  
　　e. 자동 포기도 포기 횟수에 포함  

5: 캠페인 포기 정책
　A. 캠페인 참여는 필수이며, 포기 시 포기 횟수가 차감됨
　B. 포기 가능 횟수: 총 3회 (매칭 전 포기 + 참여 중 포기 + 자동 포기 모두 포함)
　C. 포기 횟수별 조치
　　a. 1~2회: 포기 처리 + 남은 포기 횟수 알림 발송
　　　aa. 알림 메시지: '캠페인 포기가 처리되었습니다. 남은 포기 횟수: N회'
　　b. 3회(최종): 포기 처리 + 계약 정지 (기존 이용제한 체계의 "일시 정지"와 동일)
　　　aa. 정지 알림: '캠페인 포기가 3회에 도달하여 계약이 정지되었습니다.
상세 내용은 고객센터로 문의해 주세요.'
　D. 관리자 사유 검토
　　a. 포기 사유는 관리자가 사후 검토
　　b. 합당한 사유로 판단 시 포기 횟수 복원 + 파트너에게 복원 알림
　　c. 자동 포기(TIMEOUT)는 즉시 1회 차감, 파트너 사후 문의 시 관리자 검토 가능
　E. 포기 시 매칭 감점
　　a. 매칭 전 포기 / 자동 포기(TIMEOUT): 매칭 점수 -0.5점 감점
　　b. 참여 중 포기(중도 포기): 매칭 점수 -1.0점 감점
　　c. 감점은 다음 캠페인 매칭 우선순위에 반영
　F. 초기화 조건
　　a. 계약 정지 해제 시 포기 횟수 + 매칭 감점 모두 초기화
　　b. 포기 이력(사유, 일시, 관리자 검토 결과)은 영구 기록
　G. 포기 횟수 표시: 캠페인 참여 섹션 하단에 "포기 N/3회" 표시


**2.1.1.3.** 캠페인 참여 중 중도 포기 정책

1: 중도 포기 절차 (관리자 승인 필요)
　A. 참여 확정 후 캠페인 진행 중에도 중도 포기(참여 중지) 요청 가능
　B. "참여 중지" 버튼 클릭 시 사유 직접 입력 필수 (선택지 없음, 텍스트 직접 입력만 가능)
　C. 사유 입력 후 제출 시 관리자에게 중도 포기 요청이 전달됨 (즉시 포기 아님)
　D. 관리자가 요청을 검토하여 승인 또는 반려
　　a. 승인: 중도 포기 확정 → 후속 처리(감점, 보상 등) 진행
　　b. 반려: 요청 취소 → 파트너는 캠페인 계속 참여

2: 요청 중 상태
　A. 요청 제출 후 ~ 관리자 승인/반려 전까지 "참여 중지 요청중" 상태
　B. 요청 중에도 파트너는 캠페인에 계속 참여해야 함 (컵홀더 배포 및 소진 보고 유지)
　C. 요청 중에는 추가 참여 중지 요청 불가
　D. 요청 상태 표시: 캠페인 참여 섹션에 "참여 중지 요청중" 뱃지 표시

3: 승인 후 영향
　A. 포기 전까지 참여 기간은 등급 산정에 포함
　B. 승인 시점부터 미매칭 기간 카운트 시작
　C. 해당 캠페인 재참여 불가
　D. 포기 횟수 1회 차감 + 매칭 점수 -1.0점 감점 (매칭 전 포기 -0.5점보다 가중)
　E. 관리자가 승인 시 합당한 사유로 판단한 경우 포기 횟수 미차감 처리 가능

4: 남은 컵홀더 보상
　A. 승인 확정 시 남은 컵홀더에 대한 비용을 파트너가 위드폼에 지불
　B. 남은 컵홀더 수량 = 입고된 수량 - 파트너가 보고한 소진량
　C. 개당 가격 = 승인 시점 위드폼 협력사 기준가
　D. 보상 금액 = 남은 컵홀더 수량 × 개당 기준가
　E. 정산 시점: 관리자 승인 확정 즉시 정산

5: 중도 포기 사유
　A. 직접 입력만 가능 (매칭 전 포기의 선택지 카테고리와 다름)
　B. 텍스트 입력 필수 (미입력 시 요청 버튼 비활성화)


**2.1.1.4.** 컵홀더 배송·입고 진행 상태  

1: 표시 조건  
　A. 캠페인 참여 확정 후부터 노출 (매칭 시점에는 미노출)  
　B. 위치: 참여중 상태 뱃지 + 캠페인 기본 정보 아래, 참여 중지 버튼 위  
　C. 입고 완료 후에도 "입고" 상태로 유지 표시  

2: 배송 단계별 표시  
　A. 주문접수: 참여 확정 시 자동 — [주문접수] 회색 상태 버튼 (읽기 전용)  
　B. 제작중: 관리자/협력사 상태 변경 시 — [제작중] 파란색 상태 버튼 (읽기 전용)  
　C. 배송준비: 협력사 출고 준비 시 — [배송준비] 파란색 상태 버튼 (읽기 전용)  
　D. 배송중: 협력사가 송장번호 입력 시 — [배송중] 주황색 상태 버튼 (읽기 전용) + [배송완료] 녹색 클릭 가능 버튼 동시 표시  
　　a. 배송중 상태에서 도착 예정일 표시 (택배사 API 연동)  
　　b. 송장번호 및 택배사명 표시  
　E. 입고: 배송완료 처리 후 — [입고] 녹색 완료 상태 버튼 (읽기 전용)  
　　a. 입고일 표시  
　　b. "캠페인 배포가 시작되었습니다!" 안내 메시지  

3: 배송완료 → 입고 처리 (2가지 경로)  
　A. 경로A — 파트너 수동: 배송중 단계에서 파트너가 [배송완료] 버튼 클릭  
　　a. 배달원이 배송 후 택배사에서 완료 처리를 하지 않은 경우에도 파트너가 직접 확인 가능  
　　b. 클릭 시 확인 팝업: "컵홀더가 정상 수령되었습니까?" → [취소] / [확인]  
　　c. 확인 시 즉시 입고 처리 → [입고] 표시로 전환 → 캠페인 배포 자동 시작  
　B. 경로B — 택배사 API 자동: 택배사 시스템이 배송완료 감지  
　　a. 시스템이 자동으로 배송완료 + 입고 처리  
　　b. 파트너 알림: "컵홀더 배송이 완료되어 입고 처리되었습니다"  
　　c. 화면 자동 갱신 → [입고] 표시로 전환  
　C. 안전장치: A, B 모두 미발생 시  
　　a. 협력사 송장번호 입력 후 영업일 기준 5일 경과 시  
　　b. 관리자에게 긴급 알림: "주문 #{번호} 송장 입력 후 5영업일 경과, 입고 미처리 상태입니다. 확인이 필요합니다."  
　　c. 자동 처리하지 않음 (관리자 확인 후 수동 조치)  
　D. 어느 경로든 먼저 도달한 것이 트리거되고, 나머지 경로는 무시  

4: 입고 완료 시 연쇄 동작  
　A. 캠페인 배포 자동 시작  
　B. 일일 소진 보고 활성화  
　C. 재고 관리 시스템 연동 활성화  
　D. 알림 시스템 활성화 (소진 보고 미완료, 재고 부족 등)  
　E. 파트너 알림: "입고가 완료되어 캠페인 배포가 시작되었습니다. 오늘부터 일일 소진 보고를 완료해 주세요."  

5: 배송 문의 버튼  
　A. 위치: 배송 현황 영역 하단 우측  
　B. 형태: 작은 텍스트 버튼 — [배송 문의 →] (회색 아이콘 + 밑줄 텍스트)  
　C. 노출 조건: "배송중" 또는 "입고" 단계에서 노출  
　D. 동작: 클릭 시 7-2-11-2. 발주 > 주문 내역 > 해당 주문 상세 > 배송 추적 페이지로 이동  
　E. 이동 후 해당 페이지에서 협력사에게 문의 가능  

6: 물류 사고 대응  
　A. 실제 미수령인데 자동 입고됨: [배송 문의 →] → 7-2-11-2 이동 → 협력사 문의 → 협력사 확인 후 상태 복구  
　B. 배송 문제로 실제 입고일이 달라질 경우: 파트너가 관리자에게 문의 → 관리자가 캠페인 입고일/배포 시작일 수정 (관리자만 수정 가능)  

7: 관리자 알림  
　A. 파트너가 협력사에게 배송 문의 시: 관리자에게 "파트너 {매장명}이 주문 #{번호} 배송 문제를 문의했습니다" 알림 (일반)  
　B. 협력사가 입고 상태 복구 시: 관리자에게 "협력사 {사명}이 주문 #{번호} 입고 상태를 {이전}→{복구}로 변경했습니다" 알림 (중요)  
　C. 송장 입력 후 5영업일 경과 미입고 시: 관리자에게 긴급 알림 (긴급)
