**3.2.3.** 개발자 인터랙션 명세  


**3.2.3.1.** 잔여 재고 조회 API  

1: 엔드포인트  
　A. GET /api/v1/partner/campaign/remaining-stock  

2: 응답 필드  
　A. campaignId: 캠페인 ID  
　B. remainingQuantity: 잔여 수량  
　C. processMethod: 처리 방법 (carryover / return / dispose)  
　D. processDeadline: 처리 기한  
　E. processStatus: 처리 상태 (pending / completed)  


**3.2.3.2.** 처리 방법 선택 API  

1: 엔드포인트  
　A. PUT /api/v1/partner/campaign/remaining-stock/process  

2: 요청 파라미터  
　A. campaignId: 캠페인 ID  
　B. processMethod: 처리 방법 (carryover_store / carryover_return / dispose)  


**3.2.3.3.** 반품 발송 확인 API  

1: 엔드포인트  
　A. POST /api/v1/partner/campaign/remaining-stock/return  

2: 요청 파라미터  
　A. campaignId: 캠페인 ID  
　B. trackingNumber: 운송장 번호  


**3.2.3.4.** 폐기 확인 API  

1: 엔드포인트  
　A. POST /api/v1/partner/campaign/remaining-stock/dispose  

2: 요청 파라미터  
　A. campaignId: 캠페인 ID  


**3.2.3.5.** 컴포넌트 상태 관리  

1: 상태 변수  
　A. processMethod: 선택된 처리 방법  
　B. trackingNumber: 반품 운송장 번호  
　C. processStatus: 처리 진행 상태 (pending / in_progress / completed)  


**3.2.3.6.** 접근성 및 에러 처리  

1: 접근성  
　A. 처리 방법 선택 라디오 그룹: role="radiogroup", aria-label 설정  
　B. 버튼: aria-label에 처리 방법 명시  

2: 에러 코드  
　A. STK_001: 잔여 재고 조회 실패  
　B. STK_002: 처리 방법 선택 실패  
　C. STK_003: 반품 발송 확인 실패  
　D. STK_004: 폐기 확인 실패  


• 참조: 7-1-6-2. 잔여 재고 처리 확인
