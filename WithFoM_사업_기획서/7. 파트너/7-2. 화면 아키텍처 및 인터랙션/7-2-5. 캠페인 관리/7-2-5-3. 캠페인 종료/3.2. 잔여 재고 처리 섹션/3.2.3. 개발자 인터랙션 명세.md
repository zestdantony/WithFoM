**3.2.3.** 개발자 인터랙션 명세  


**3.2.3.1.** 잔여 재고 조회 API  

1: 엔드포인트  
　A. GET /api/v1/partner/campaign/remaining-stock  
　B. 인증: Bearer Token (파트너 세션)  

2: 응답 JSON  

```json
{
  "campaignId": "CMP-2026-001",
  "remainingQuantity": 120,
  "distributionPolicy": "continue",
  "distributionEndDeadline": null,
  "processMethod": null,
  "processDeadline": null,
  "processStatus": "distributing"
}
```

3: 응답 필드 설명  
　A. distributionPolicy: continue (배포 지속) / deadline_end (배포 종료 기한 도래)  
　B. distributionEndDeadline: 배포 종료 기한 (기한 없는 캠페인은 null)  
　C. processMethod: 처리 방법 — dispose (폐기) / return (반품) / null (배포 지속 중)  
　D. processStatus: distributing (배포 지속 중) / pending (처리 대기) / completed (처리 완료)  


**3.2.3.2.** 처리 방법 선택 API (배포 종료 기한 캠페인 전용)  

1: 엔드포인트  
　A. PUT /api/v1/partner/campaign/remaining-stock/process  
　B. 인증: Bearer Token (파트너 세션)  

2: 요청 파라미터  
　A. campaignId: 캠페인 ID  
　B. processMethod: dispose (폐기) / return (반품)  


**3.2.3.3.** 반품 발송 확인 API  

1: 엔드포인트  
　A. POST /api/v1/partner/campaign/remaining-stock/return  
　B. 인증: Bearer Token (파트너 세션)  

2: 요청 파라미터  
　A. campaignId: 캠페인 ID  
　B. trackingNumber: 운송장 번호  


**3.2.3.4.** 폐기 확인 API  

1: 엔드포인트  
　A. POST /api/v1/partner/campaign/remaining-stock/dispose  
　B. 인증: Bearer Token (파트너 세션)  

2: 요청 파라미터  
　A. campaignId: 캠페인 ID  


**3.2.3.5.** 컴포넌트 상태 관리  

1: 상태 변수  
　A. distributionPolicy: 배포 정책 (continue / deadline_end)  
　B. processMethod: 선택된 처리 방법 (dispose / return / null)  
　C. trackingNumber: 반품 운송장 번호  
　D. processStatus: 처리 진행 상태 (distributing / pending / completed)  

2: 렌더링 분기  
　A. distributionPolicy === 'continue' 시: 배포 지속 안내 메시지 표시, 처리 버튼 미노출  
　B. distributionPolicy === 'deadline_end' 시: 처리 방법 선택 UI 표시 (폐기 기본 / 반품)  


**3.2.3.6.** 접근성 및 에러 처리  

1: 접근성  
　A. 처리 방법 선택 라디오 그룹: role="radiogroup", aria-label 설정  
　B. 버튼: aria-label에 처리 방법 명시  

2: 에러 코드  
　A. STK_001: 잔여 재고 조회 실패  
　B. STK_002: 처리 방법 선택 실패  
　C. STK_003: 반품 발송 확인 실패  
　D. STK_004: 폐기 확인 실패  


• 참조: 7-1-6-2. 잔여 재고 처리 확인
