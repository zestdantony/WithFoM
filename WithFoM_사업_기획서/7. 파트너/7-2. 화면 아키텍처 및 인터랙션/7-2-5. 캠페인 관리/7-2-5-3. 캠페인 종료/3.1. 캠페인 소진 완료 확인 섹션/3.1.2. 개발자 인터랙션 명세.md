**3.1.2.** 개발자 인터랙션 명세  


**3.1.2.1.** 캠페인 종료 상태 조회 API  

1: 엔드포인트  
　A. GET /api/v1/partner/campaign/completion-status  
　B. 인증: Bearer Token (파트너 세션)  

2: 응답 JSON  

```json
{
  "campaignId": "CMP-2026-001",
  "status": "completed",
  "terminationType": "full_consumption",
  "assignedQuantity": 500,
  "consumedQuantity": 500,
  "remainingQuantity": 0,
  "campaignPeriod": {
    "startDate": "2026-01-01",
    "endDate": "2026-03-31",
    "actualEndDate": "2026-03-15"
  },
  "retroactiveInputDeadline": "2026-03-22",
  "nextCampaign": {
    "matched": true,
    "campaignId": "CMP-2026-002",
    "startDate": "2026-03-16",
    "autoTransition": true
  },
  "alerts": [
    {
      "type": "completion",
      "message": "축하합니다! 캠페인 컵홀더가 모두 소진되었습니다.",
      "sentAt": "2026-03-15T18:00:00Z"
    }
  ]
}
```

3: 응답 필드 설명  
　A. terminationType: full_consumption (조기 소진) / period_expired (기간 만료) / terminated (중도 종료)  
　B. nextCampaign: 다음 캠페인 매칭 정보 (미매칭 시 null)  
　C. nextCampaign.autoTransition: 현재 캠페인 종료 익일 자동 전환 여부  


**3.1.2.2.** 소급 입력 API  

1: 엔드포인트  
　A. POST /api/v1/partner/campaign/retroactive-report  
　B. 인증: Bearer Token (파트너 세션)  
　C. 종료 후 7일 이내 가능  


**3.1.2.3.** 컴포넌트 구현  

1: 정보 표시 전용  
　A. 상태별 렌더링 분기: completed / expired / terminated  

2: 다음 캠페인 표시  
　A. nextCampaign !== null 시: "다음 캠페인이 매칭되었습니다. {시작일}부터 자동 전환됩니다" 안내 표시  
　B. nextCampaign === null 시: "현재 캠페인 종료 1주일 전부터 다음 캠페인 매칭이 가능합니다" 안내 표시  


**3.1.2.4.** 접근성 요구사항  

1: ARIA 속성  
　A. role="region"  
　B. aria-label 설정  
　C. status 아이콘 alt text 제공  


**3.1.2.5.** 에러 처리  

1: 에러 코드  
　A. END_001: 상태 조회 실패  
　B. END_002: 소급 입력 실패
