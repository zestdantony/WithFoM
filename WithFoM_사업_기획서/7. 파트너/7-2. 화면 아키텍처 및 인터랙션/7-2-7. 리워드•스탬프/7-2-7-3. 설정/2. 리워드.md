# 2: 리워드  

　A. 리워드 참여 여부 선택  
　　a. 참여 여부 On/Off 버튼 생성  
　　b. 참여 여부는 서비스 타입(리워드/오더)의 표시와 연동  
　　c. 리워드 활성화 시 게임도 자동 활성화 (연동)  
　B. 리워드 생성(최대 6개)  
　　a. 기본 입력박스 1개 노출  
　　b. 오른쪽 상단 "리워드 추가" 버튼  
　　c. 누를 때마다 1개씩 추가  
　　d. 총 6개까지 생성 가능  
　　e. 리워드 생성 박스 공통 구성  
　　　aa. 왼쪽 상단: N번째 생성 숫자  
　　　bb. 오른쪽 상단: 수정/삭제 버튼  
　　　cc. 입력 항목  
　　　　• 유형(쿠폰 or 포인트)  
　　　　• 리워드명(예: 10% 할인 쿠폰, 1000원 할인 쿠폰, 포인트 500점, 포인트 3000점 등)  
　　　　• 리워드 가격 or 가격대비 적용 퍼센트  
　　　　• 총 발행 수량  
　　　　• 유효기간  
　　f. 리워드 조건 유형  
　　　aa. 최소/최대 결제금액 조건  
　　　bb. 메뉴 조건 (카테고리/단일 메뉴)  
　　　cc. 채널 조건 (매장/테이크아웃/오더)  
　　　dd. 시간/요일 조건  
　　g. 리워드 당첨 확률 보드  
　　　aa. 상단 표시  
　　　　• 생성된 리워드 개수  
　　　　• 총 확률 합계  
　　　bb. 리스트 표시  
　　　　• 리워드별 당첨 확률 입력  
　　　　• 리워드별 총 추정 예산  
　　　　• 리워드별 최소 총 추정 매출  
　　　cc. 리스트 우측 하단 표시  
　　　　• 토탈 예산 (추정치)  
　　　　• 토탈 최소 매출 (추정치)  
　　h. 사용 제한 설정  
　　　aa. 쿠폰 제한  
　　　　• 1일 사용 횟수 제한  
　　　　• 1회 결제당 사용 개수 제한  
　　　bb. 포인트 제한  
　　　　• 1일 사용 가능 총액 제한  
　　　　• 1회 결제당 결제 금액 대비 비율 제한  
　　i. 박스 하단: 확인 버튼  
　　　aa. 누를 시 생성 완료  
　　　bb. 완료 된 상태로 그대로 유지  
　　　cc. 설정 완료 된 리워드가 6개일 경우, 리워드 생성 박스 생성이 불가능 하고 버튼을 비활성화  
　　　dd. 버튼을 눌러도 더 이상 생성이 불가 합니다 팝업 메세지 출력
