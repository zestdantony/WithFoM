**1.7.** 입고 처리  


**1.7.1.** 입고 개요  

**1.7.1.1.** 입고 유형  
1: 자동 입고 (발주 연동)  
　A. 발주 상품 배송 완료 시 자동 재고 증가  
　B. "확인 요망" 표시로 파트너 검증 유도  
2: 수동 입고  
　A. 외부 구매 물품 입고  
　B. 재고 실사 후 조정  
　C. 기타 입고 건  


**1.7.2.** 자동 입고 (발주 연동)  

**1.7.2.1.** 자동 입고 프로세스  
1: 트리거 조건  
　A. 발주 시스템에서 배송 완료 처리 시  
　B. 배송 추적 정보 연동 (택배사 API)  
2: 자동 처리  
　A. 배송 완료 시점에 재고 자동 증가  
　B. 발주 수량만큼 해당 품목 재고 추가  
　C. 재고 변동 이력에 "입고 (발주연동)" 기록  
3: 확인 요망 표시  
　A. 자동 입고된 품목에 "확인 요망" 뱃지 표시  
　B. 파트너가 실물 확인 후 승인 필요  

**1.7.2.2.** 확인 프로세스  
1: 확인 요망 알림  
　A. 품목 리스트에 주황색 "입고 확인" 뱃지 표시  
　B. 대시보드에 "입고 확인 필요" 알림 카드 표시  
　C. 푸시 알림 발송 (설정 시)  
2: 확인 팝업 구성  
　A. 발주 정보: 발주일, 발주 수량, 배송 완료일  
　B. 입고 정보: 입고 수량, 입고 후 재고량  
　C. 버튼:  
　　- [확인 완료]: 입고 수량 정상 확인  
　　- [수량 조정]: 실제 입고량과 차이가 있을 경우  
　　- [이의 신청]: 배송 누락, 파손 등 문제 발생 시  
3: 확인 완료 동작  
　A. "입고 확인" 뱃지 제거  
　B. 입고 확인 이력에 기록  
　C. 확인자, 확인 일시 저장  

**1.7.2.3.** 수량 불일치 처리  
1: 수량 조정  
　A. 실제 입고량 입력  
　B. 차이 사유 선택: 배송 누락 / 파손 / 수량 오류 / 기타  
　C. 재고는 실제 입고량으로 조정  
2: 이의 신청  
　A. 심각한 문제 발생 시 (대량 누락, 전체 파손 등)  
　B. 문제 상황 설명 및 사진 첨부  
　C. 관리자에게 자동 전달  


**1.7.3.** 수동 입고  

**1.7.3.1.** 입고 등록  
1: 접근 경로  
　A. 품목 상세 > [수량 조정] > 입고 선택  
　B. 재고 관리 > [입고 등록] 버튼  
2: 입력 항목  
　A. 품목 선택 (단일 또는 복수)  
　B. 입고 수량  
　C. 입고 유형: 외부구매 / 재고실사 / 반품입고 / 기타  
　D. 입고일 (기본값: 오늘)  
　E. 메모/사유 (선택)  
3: 저장 동작  
　A. 재고 즉시 증가  
　B. 재고 변동 이력에 "입고 (수동)" 기록  


**1.7.4.** 입고 이력 관리  

**1.7.4.1.** 이력 조회  
1: 접근 경로  
　A. 재고 관리 > 입고 이력 탭  
　B. 품목 상세 > 재고 변동 이력  
2: 필터 기능  
　A. 기간별 필터  
　B. 입고 유형별 필터 (발주연동/수동/실사 등)  
　C. 품목별 필터  
3: 표시 정보  
　A. 입고 일시  
　B. 품목명  
　C. 입고 수량  
　D. 입고 유형  
　E. 발주 번호 (발주 연동 시)  
　F. 확인 상태 (확인 완료/미확인)  
　G. 처리자
