**1.1.** 게임 생성 섹션  


**1.1.1.** 게임 미생성 상태  


**1.1.1.1.** 화면 구성  

1: 중앙 안내 영역  
　A. 아이콘 + "아직 게임이 없습니다" 메시지  
　B. "게임을 만들어 고객에게 리워드를 제공해 보세요" 보조 문구  

2: [게임 만들기] CTA 버튼  
　A. 클릭 시 게임 생성·수정 화면(1.2.)으로 이동  
　B. 파트너 게임 탭에서만 표시  


**1.1.2.** 게임 생성 완료 상태  


**1.1.2.1.** 현재 게임 카드  

1: 게임 정보 표시  
　A. 게임 유형 아이콘 + 유형명(룰렛/뽑기/스크래치/사다리)  
　B. 생성일  
　C. 활성/비활성 상태 배지  
　D. 적용 중인 리워드 수(예: "리워드 4개 적용 중")  

2: 유형 변경 잔여 횟수 표시  
　A. "이번 달 유형 변경: N/M회 사용" 형식  
　B. 베이직: 0/1, 플러스: 0/2, 프리미엄: 무제한  
　C. 매월 1일 00:00 자동 초기화  


**1.1.2.2.** 액션 버튼 영역  

1: [유형 변경] 버튼  
　A. 잔여 변경 횟수가 1회 이상 남아있을 때 활성화, 0회일 때 비활성화  
　B. 비활성화 시 툴팁: "이번 달 유형 변경 횟수를 모두 사용했습니다"  
　C. 클릭 시 확인 모달 → 게임 생성·수정 화면(1.2.) Step 1로 이동  
　D. 확인 모달 내용: "게임 유형을 변경하면 현재 게임이 삭제됩니다. 리워드 구성은 프리셋에 저장하면 다음에 빠르게 적용할 수 있습니다."  

2: [설정 수정] 버튼  
　A. 클릭 시 1.2. 화면 Step 2(디자인·상세 설정)부터 진입  

3: [삭제] 버튼  
　A. 삭제 확인 모달: "게임을 삭제하면 현재 적용된 리워드 구성이 초기화됩니다. 삭제하시겠습니까?"  
　B. 삭제 시 게임 데이터 초기화, 1.1.1. 미생성 상태로 전환  
　C. 저장된 프리셋 및 라이브러리 리워드는 유지  


• 참조: 1.0. 페이지 개요 > 1.0.1.2. 게임 유형 구분, 1.2. 게임 생성·수정 화면
