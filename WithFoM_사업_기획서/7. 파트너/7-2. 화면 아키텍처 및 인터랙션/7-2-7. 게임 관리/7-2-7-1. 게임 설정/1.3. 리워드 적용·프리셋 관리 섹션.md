**1.3.** 리워드 적용·프리셋 관리 섹션  


**1.3.1.** 섹션 개요  


**1.3.1.1.** 섹션 역할  

1: 현재 파트너 게임에 적용된 리워드 구성을 실시간 편집  
2: 적용 중인 리워드의 등수·확률 직접 변경  
3: 저장된 리워드 라이브러리(7-2-8-3 > 3.1.7.)에서 리워드 추가/제거  
4: 프리셋 저장·불러오기·삭제  
5: 예상 예산·매출 실시간 계산 표시  
6: 리워드 총량 보충  


**1.3.2.** 리워드 편집 영역  


**1.3.2.1.** 현재 적용 리워드 카드 목록  

1: 게임에 적용된 리워드를 등수 순(1등→6등)으로 카드 나열  
2: 각 카드 표시 항목  
　A. 등수(드래그 핸들로 위/아래 이동 → 등수 변경)  
　B. 리워드명  
　C. 유형(쿠폰/포인트) 아이콘  
　D. 당첨 확률(%) — 인라인 수정 가능  
　E. 총 발행 수량 — 읽기 전용(개별 수정 불가, 리워드 보충으로만 조정)  
　F. 잔여 수량 표시  
　G. 유효기간(상대/절대) 표시  
　H. 게임 적용 시 안내 문구 표시  
　I. [제거] 버튼 → 게임에서 리워드 적용 해제  
3: 드래그 앤 드롭으로 카드 순서(등수) 변경  
4: 리워드 미적용 시 빈 상태 안내: "리워드 설정에서 게임에 리워드를 적용해 주세요"  
　A. [리워드 설정으로 이동] 링크 버튼 → 7-2-8-3 페이지로 네비게이션  
5: 수량 상태 시각 표시(개별 카드)  
　A. 잔여 10% 이하: 주황색 경고 배지 "잔여 N개"  
　B. 소진(0개): 빨간색 "소진" 배지 + 비활성(회색) 처리  
　C. 정상: 기본 표시  


**1.3.2.2.** 저장 라이브러리 연동 패널  

1: 우측 사이드 패널에 저장된 리워드 목록 표시  
2: 리워드 클릭 → 새 독립 복사본 생성 → 적용 리워드 카드 목록 하단에 추가  
3: 추가 시 확률·수량은 빈 값 → 파트너 직접 입력  
4: 적용 중인 리워드 합계 6개 도달 시 추가 불가 안내  

**1.3.2.3.** 리워드 제거 규칙  

1: [제거] 클릭 시 확인 모달: "게임에서 해당 리워드를 제거하시겠습니까?"  
2: 제거 시 해당 복사본 영구 삭제(라이브러리 원본은 유지)  
3: 제거 후 확률 합계 재검증 필요 → 100% 미달 시 경고  
4: 제거 후 최소 수량 조건 미달 시 제거 차단  
　A. 꽝 ON: 리워드가 1개인 상태에서 제거 시도 시 차단 + 안내: "최소 1개의 리워드가 필요합니다"  
　B. 꽝 OFF: 리워드가 2개인 상태에서 제거 시도 시 차단 + 안내: "최소 2개의 리워드가 필요합니다"  


**1.3.3.** 꽝 설정 영역  


**1.3.3.1.** 꽝 토글  

1: 꽝 포함 ON/OFF 토글  
　A. ON → 리워드 최대 5개 + 꽝 1개  
　B. OFF → 리워드 최대 6개  
2: 꽝 확률(%) 인라인 수정 가능  
3: 꽝 메시지 수정 가능(기본: "다음 기회에!", 최대 20자)  
4: 꽝 OFF 전환 조건  
　A. 리워드 2개 이상일 때만 OFF 전환 가능  
　B. 리워드 1개인 상태에서 OFF 전환 시도 시 차단 + 안내: "꽝을 해제하려면 리워드를 2개 이상 적용해 주세요"  


**1.3.4.** 확률·예산 요약 영역  


**1.3.4.1.** 확률 검증  

1: 실시간 확률 합계 표시(리워드 + 꽝)  
2: 100% 미달 시 경고 + [게임에 반영] 버튼 비활성화  
3: 100% 충족 시 녹색 체크 표시  

**1.3.4.2.** 예산·매출 추정  

1: 리워드별 추정 예산 계산(리워드 가치 × 확률 × 예상 참여 수)  
2: 리워드별 최소 추정 매출 계산  
3: 토탈 예산·토탈 최소 매출 합계 표시  
4: 예상 참여 수 입력란(기본값: 직전 30일 평균, 수동 조정 가능)  
5: 안내 문구 표시: "본 수치는 설정된 확률과 예상 참여 수를 기반으로 한 추정치이며, 실제 운영 결과와 다를 수 있습니다. 참고용으로만 활용해 주세요."  


**1.3.5.** 리워드 보충 영역  


**1.3.5.1.** 총량 보충 기능  

1: [리워드 보충] 버튼  
2: 보충할 총량 숫자 입력  
3: 분배 미리보기: "A 리워드 +N개, B 리워드 +N개, C 리워드 +N개" (현재 확률 비례 분배)  
4: [보충 확정] 버튼 → 서버 반영  
　A. 소진된 리워드 재활성화(당첨 후보 복원)  
　B. 게임 일시정지 상태였을 경우 자동 재개  
　C. 예산·매출 추정치 자동 재계산  
5: 보충 원칙  
　A. 개별 리워드 수량 직접 수정 불가  
　B. 입력한 총량을 현재 확률 비율에 따라 각 리워드에 자동 분배  
　C. 확률·등수·꽝 설정은 보충으로 변경되지 않음  


**1.3.6.** 게임 반영 버튼  


**1.3.6.1.** 버튼 기능  

1: 활성화 조건  
　A. 리워드+꽝 합계 최소 2개 이상 적용  
　B. 확률 합계 = 100%  
2: 클릭 시 현재 편집 상태를 라이브 게임에 즉시 반영  
3: 반영 완료 토스트: "게임 리워드 구성이 업데이트되었습니다"  
4: 반영 전/후 비교 모달 미표시(즉시 반영)  


**1.3.7.** 프리셋 관리 영역  


**1.3.7.1.** 프리셋 저장  

1: [현재 구성을 프리셋으로 저장] 버튼  
2: 저장 시 입력 항목  
　A. 프리셋명(기본: "프리셋 YYYY-MM-DD", 수정 가능)  
3: 프리셋 저장 내용  
　A. 각 리워드의 sourceRewardId(저장 라이브러리 원본 참조)  
　B. 등수, 확률(%), 수량  
　C. 꽝 ON/OFF 상태 + 꽝 확률 + 꽝 메시지  
4: 절대 유효기간 리워드 필터  
　A. 절대 유효기간(YYYY-MM-DD까지) 리워드가 포함된 경우  
　B. 해당 리워드는 프리셋 저장에서 자동 제외 + 안내: "절대 유효기간 리워드(N개)는 프리셋에 포함되지 않습니다"  
　C. 제외 후 나머지 리워드만 프리셋에 저장  
5: 저장 한도  
　A. 베이직: 1개, 플러스: 2개, 프리미엄: 4개  
　B. 한도 초과 시: "프리셋 저장 한도에 도달했습니다. 기존 프리셋을 삭제하거나 플랜을 업그레이드해 주세요"  

**1.3.7.2.** 프리셋 불러오기  

1: 저장된 프리셋 목록 표시(프리셋명 + 저장일 + 리워드 수)  
2: [적용] 클릭 시  
　A. sourceRewardId 기반으로 라이브러리에서 리워드 확인  
　B. 유효한 리워드 → 새 복사본 생성 → 게임에 적용  
　C. 저장된 확률·수량·등수·꽝 설정 복원  
3: 불완전 프리셋(라이브러리에서 삭제된 리워드 포함) 처리  
　A. 경고 아이콘 + "일부 리워드가 삭제되어 불완전합니다 (N/M개 유효)"  
　B. [적용] 시 유효한 리워드만 적용 + 확률 합계 미달 안내  
　C. 파트너가 부족분 직접 보충(라이브러리에서 추가 또는 리워드 설정에서 새로 생성)  

**1.3.7.3.** 프리셋 삭제  

1: 각 프리셋 우측 [삭제] 버튼  
2: 삭제 확인 모달 → 삭제  
3: 삭제 후 해당 슬롯에 새 프리셋 저장 가능  


**1.3.8.** 멤버십 다운그레이드 시  


**1.3.8.1.** 프리셋 잠금 처리  

1: 모든 프리셋 잠김 상태 전환  
2: 새 플랜 한도에 맞게 유지할 프리셋 선택 안내  
3: 선택 완료 전까지 프리셋 저장·적용 기능 제한  
4: 선택 완료 후 미선택 프리셋 영구 삭제  


• 참조: 1.0. 페이지 개요 > 1.0.4. 리워드-게임 연동 규칙, 1.2. 게임 생성·수정 화면 > 1.2.2.2. 게임 메커니즘 상세, 7-2-8-3 > 3.1.7. 리워드 저장 라이브러리, 7-2-8-3 > 3.1.3. 꽝 및 확률 검증, 7-2-6-1 > 1.2. 멤버십 상세 설명
