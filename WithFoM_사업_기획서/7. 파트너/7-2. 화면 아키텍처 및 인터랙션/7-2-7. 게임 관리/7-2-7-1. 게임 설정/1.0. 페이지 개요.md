**1.0.** 페이지 개요  


**1.0.1.** 페이지 목적  


**1.0.1.1.** 페이지 정의  

1: 페이지 명칭  
　A. 게임 설정 페이지  
　B. 게임 관리 카테고리 내 첫 번째 하위 메뉴  

2: 페이지 역할  
　A. 파트너가 자신의 매장에서 운영할 단일 게임 생성·유형 변경·관리  
　B. 광고주 게임 7등 슬롯 리워드 프리셋 관리 및 참여 ON/OFF 제어  
　C. 게임 유형(룰렛/뽑기/스크래치/사다리) 선택 및 리워드 적용 확인  
　D. 파트너 게임/광고주 게임 탭 전환으로 독립 관리  

3: 탭 UI 구조  
　A. 상단에 "파트너 게임 / 광고주 게임" 세그먼트 선택 버튼 배치  
　B. 7-2-8-1 현황의 쿠폰/포인트/스탬프 선택 버튼과 동일한 UI 패턴  
　C. 파트너 게임 탭: 1.1.~1.3. 섹션 표시  
　D. 광고주 게임 탭: 1.4. 섹션 표시  


**1.0.1.2.** 게임 유형 구분  

1: 광고주 캠페인 게임  
　A. 광고주가 10단계 캠페인 위자드(6-3-1)에서 생성  
　B. 1~6등 광고주 리워드(6개) + 7등 파트너 리워드(최대 6개 풀, 확률 기반 결정)  
　C. 소비자가 컵홀더 QR 스캔 시 무제한 무료 참여  
　D. 캠페인 게임은 파트너 참여 여부와 무관하게 QR 통해 항상 운영  
　E. 100% 당첨(꽝 없음)  
　F. 파트너 7등 참여 OFF 시: 광고주 예비 7등 리워드 또는 꽝으로 대체  

2: 파트너 자체 게임  
　A. 파트너가 본 페이지에서 직접 생성·관리  
　B. 파트너 리워드만 사용(광고주 리워드 미포함)  
　C. 소비자가 폼 코인 1개 소모하여 참여, 또는 파트너 무료 게임 제공 시 무료 참여  
　D. 매장당 1개 게임 운영, 유형 변경 가능(플랜별 월 변경 횟수 제한)  
　E. 당첨 확률·꽝 설정 파트너 자유  

3: 유형별 고정 슬롯 수  
　A. 룰렛: 6칸  
　B. 뽑기(가챠): 12캡슐  
　C. 스크래치: 16칸(4×4)  
　D. 사다리: 6줄  

4: 유형 변경 횟수  
　A. 베이직: 월 1회  
　B. 플러스: 월 2회  
　C. 프리미엄: 무제한  

5: 리워드 활용 원칙  
　A. 7-2-8-3 리워드 설정에서 생성·적용한 리워드는 파트너 자체 게임에 독립 복사본으로 적용  
　B. 저장된 리워드 템플릿은 광고주 게임 7등 슬롯 배정에도 재사용 가능  
　C. 파트너 자체 게임과 광고주 게임 7등의 리워드 구성은 독립적으로 관리  


**1.0.1.3.** 소비자 참여 조건 비교  

1: 광고주 캠페인 게임  
　A. 진입 경로: 컵홀더 QR 스캔  
　B. 참여 비용: 무료(무제한)  
　C. 당첨 구성: 1~6등 광고주 리워드 + 7등 파트너 리워드  
　D. 꽝: 없음(100% 당첨)  

2: 파트너 자체 게임 — 유료 이용  
　A. 진입 경로: 웹/앱 내 파트너 스토어에서 진입  
　B. 참여 비용: 폼 코인 1개 소모  
　C. 당첨 구성: 파트너 리워드만  
　D. 꽝: 파트너 설정에 따라 포함/미포함  

3: 파트너 자체 게임 — 무료 이용  
　A. 조건: 파트너가 코인 적립하여 무료 제공 설정 시(7-2-7-2 > 2.4.)  
　B. 대상: 파트너 설정 대상 고객(신규/단골/VIP/휴면/전체)  
　C. 제한: 1인당 하루 1회, 초과 시 본인 폼 코인 사용  

4: 파트너 자체 게임 — 조건부 무료 기회  
　A. 단골 스토어 첫 등록 시 1회 무료  
　B. 친구 초대 성공 시 1회 무료(초대자·피초대자 모두)  
　C. 스탬프 1개 적립 시 1회 무료  


**1.0.2.** 핵심 기능  


**1.0.2.1.** 게임 생성·현황  

1: 매장 단일 게임 생성·유형 변경·현재 상태 관리  


**1.0.2.2.** 게임 생성·수정  

1: 파트너 자체 게임 전용, 유형 선택 → 상세 설정 → 저장  


**1.0.2.3.** 리워드 적용·프리셋 관리  

1: 현재 게임 리워드 구성 편집 + 프리셋 저장/불러오기 + 예산·매출 표시  


**1.0.2.4.** 광고주 게임 7등 관리  

1: 7등 리워드 참여 ON/OFF + 7등 리워드 풀 구성 + 프리셋 관리  


**1.0.3.** 진입 경로  


**1.0.3.1.** 사이드바에서 진입  

1: 사이드바 네비게이션  
　A. 사이드바 > 게임 관리 > 게임 설정  
　B. 파트너 게임 탭 기본 활성화, 게임 생성 섹션(1.1.) 표시  

2: 온보딩 플로우  
　A. 멤버십 구독 완료 후 자동 진입(선택 사항)  
　B. 건너뛰기 가능, 나중에 사이드바에서 설정  

3: 리워드 설정(7-2-8-3)에서 진입  
　A. [게임 적용] 완료 모달의 [게임 설정으로 이동] 버튼 클릭 시  
　B. 파트너 게임 탭 > 1.3. 리워드 적용·프리셋 관리 섹션 포커스  


**1.0.4.** 리워드-게임 연동 규칙  


**1.0.4.1.** 활성화 연동  

1: 리워드 활성화 시 게임 자동 활성화  
　A. 리워드 활성화(7-2-8-3 > 3.1.1.1.) 시 게임 기능 전체 활성화  
　B. 리워드 비활성화 시 게임 기능 전체 비활성화  
　C. 개별 분리 불가(항상 동기화)  

2: 게임 활성화 필수 조건  
　A. 리워드가 활성화되어 있어야 함  
　B. 게임 유형이 선택되어 있어야 함  
　C. 리워드가 최소 1개 이상 게임에 적용되어 있어야 함  
　D. 조건 미충족 시 활성화 토글 비활성화 + 안내 메시지  


**1.0.4.2.** 복사본 원칙  

1: 게임 적용 리워드의 독립성  
　A. 게임에 적용된 리워드는 저장 라이브러리(7-2-8-3 > 3.1.7.)와 별개의 독립 복사본  
　B. 라이브러리 원본 삭제·수정 → 게임 복사본 영향 없음  
　C. 게임 복사본은 적용 해제(1.3.에서 제거) 전까지 삭제 불가  

2: 프리셋과의 관계  
　A. 프리셋은 저장 라이브러리 리워드에 대한 참조 + 확률·수량·등수 저장  
　B. 라이브러리 리워드 삭제 시 프리셋에서 해당 항목 자동 제거  
　C. 프리셋 적용 시 라이브러리에서 새 복사본 생성 → 게임에 적용  
　D. 프리셋 저장 수: 베이직 1개, 플러스 2개, 프리미엄 4개 (파트너·광고주 각각)  


• 참조: 7-2-3-4. 게임 관리 > 4.1. 게임 설정, 7-2-8-3. 설정 > 3.1. 리워드, 7-2-8-1. 현황 > 1.2. 탭 선택 UI, 6-3-1. 10단계 캠페인 생성 위자드, 3-7-1 > 5. 코인 시스템
