**3.9.** 개발자 인터랙션 명세  


**3.9.1.** API 명세  


**3.9.1.1.** 게임 운영 현황 조회  

1: 엔드포인트  
　A. `GET /api/v1/partner/games/statistics`  
　B. 인증: Bearer Token(파트너 세션)  

2: 요청 쿼리 파라미터  

| 파라미터 | 타입 | 필수 | 설명 |
|----------|------|------|------|
| preset | string | N | today \| week \| month \| custom. 기본값: today |
| startDate | string | N | YYYY-MM-DD(preset=custom 시 필수) |
| endDate | string | N | YYYY-MM-DD(preset=custom 시 필수) |

3: 응답 JSON  

```json
{
  "partnerGame": {
    "totalPlays": 0,
    "freePlays": 0,
    "paidPlays": 0,
    "coinsConsumed": 0,
    "coinsRemaining": 0,
    "rewardsIssued": 0,
    "rewardsUsed": 0
  },
  "advertiserGame": {
    "totalParticipations": 0,
    "partnerRewardWins": 0
  }
}
```

4: 에러  
　A. 401: 미인증  
　B. GS_001: 통계 조회 실패  


**3.9.1.2.** 상품별 현황 조회  

1: 엔드포인트  
　A. `GET /api/v1/partner/games/statistics/prizes`  
　B. 인증: Bearer Token(파트너 세션)  

2: 요청 쿼리 파라미터  

| 파라미터 | 타입 | 필수 | 설명 |
|----------|------|------|------|
| gameId | string | N | 특정 게임 필터(미지정 시 전체) |
| preset | string | N | 기간 프리셋 |
| startDate | string | N | YYYY-MM-DD |
| endDate | string | N | YYYY-MM-DD |

3: 응답 JSON  

```json
{
  "partnerPrizes": [
    {
      "prizeName": "string",
      "prizeType": "reward | custom | coin | blank",
      "winCount": 0,
      "winRate": 0.0,
      "configuredRate": 0.0,
      "dailyLimit": 0,
      "dailyRemaining": 0
    }
  ],
  "advertiserPrizes": [
    {
      "campaignName": "string",
      "rewardName": "string",
      "totalWins": 0,
      "usedCount": 0
    }
  ]
}
```

4: 에러  
　A. GS_002: 상품별 현황 조회 실패  


**3.9.2.** 컴포넌트 구현 참고  


**3.9.2.1.** 듀얼 현황 표시  

1: 파트너 자체 게임 통계와 광고주 캠페인 게임 통계를 섹션 분리 표시  
2: 광고주 캠페인 없을 시 해당 영역 숨김 또는 "연동된 캠페인 없음" 표시  


**3.9.2.2.** 매장 관리 연동  

1: 경영 분석 지표(참여율 추이, 고객 유형별, 재방문 효과)는 매장 관리 API(7-2-6-3 > 3.9.)에서 제공  
2: 본 페이지에서는 [상세 분석 보기] 링크로 매장 관리 페이지 유도  
3: 게임 활성화 매장에만 게임 관련 분석 지표가 매장 관리에 표시됨  


• 참조: 3.0.~3.2., 3.8., 7-2-6-3 > 3.9. 개발자 인터랙션 명세
