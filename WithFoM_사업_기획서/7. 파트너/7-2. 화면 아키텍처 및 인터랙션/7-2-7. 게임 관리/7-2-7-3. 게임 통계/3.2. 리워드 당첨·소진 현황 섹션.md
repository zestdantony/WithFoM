**3.2.** 리워드 당첨·소진 현황 섹션  


**3.2.1.** 리워드 당첨 분석  


**3.2.1.1.** 당첨 비율 차트  

1: 도넛 차트 구성  
　A. 현재 게임에 적용된 리워드별 당첨 비율 시각화  
　B. 각 리워드 등수(1등~6등)별 색상 구분  
　C. 꽝(설정 시): 별도 색상으로 표시  
　D. 차트 중앙: 선택 기간 내 총 게임 이용 횟수 표시  
　E. 각 영역 클릭/호버 시 해당 리워드의 당첨 횟수·비율 툴팁 표시  


**3.2.1.2.** 당첨 상세 테이블  

1: 테이블 컬럼 구성  
　A. 리워드명: 적용 중인 리워드의 이름  
　B. 등수: 1등~6등 (꽝은 "꽝"으로 표시)  
　C. 당첨 횟수: 선택 기간 내 해당 리워드 당첨 횟수  
　D. 설정 확률: 게임 설정에서 파트너가 설정한 당첨 확률(%)  
　E. 실제 비율: 선택 기간 내 실제 당첨 비율(%)  
　F. 차이: 설정 확률 대비 실제 비율의 차이 (△: 높음, ▽: 낮음, ≈: 유사)  

2: 수량 소진 리워드 표시  
　A. 소진된 리워드는 행에 회색 처리 + "소진" 배지 표시  
　B. 소진 시점 이후 해당 리워드는 당첨 풀에서 제외되어 나머지 리워드에 확률 비례 재분배  


**3.2.2.** 리워드 소진 현황  


**3.2.2.1.** 소진 프로그레스 바  

1: 리워드별 프로그레스 바 구성  
　A. 리워드명 + 등수 라벨  
　B. 프로그레스 바: 소진량 / 총 발행 수량 비율 시각화  
　C. 잔여 수량 텍스트: "N개 남음 (M%)"  
　D. 색상 규칙  
　　① 잔여 수량 > 10%: 기본 색상 (파랑)  
　　② 잔여 수량 ≤ 10%: 경고 색상 (주황)  
　　③ 잔여 수량 = 0: 위험 색상 (빨강) + "소진" 배지  
　　④ 운영 7일 미만 (데이터 부족): 비활성 상태 + "데이터 수집 중" 텍스트  

2: 꽝 항목  
　A. 꽝은 수량 제한 없으므로 프로그레스 바 미표시  
　B. "무제한" 라벨로 표시  


**3.2.2.2.** 예상 소진 시점  

1: 소진 시점 계산 방식  
　A. 최근 7일간 평균 일일 소모량 기준 추정  
　B. 잔여 수량 ÷ 평균 일일 소모량 = 예상 잔여 일수  
　C. 데이터 부족 시 (운영 7일 미만): "데이터 수집 중" 표시  

2: 표시 형식  
　A. 리워드별: "약 N일 후 소진 예상 (M월 D일)"  
　B. 전체 리워드 최단 소진 예상 항목 강조 표시  
　C. 전체 리워드 총량 잔여 ≤ 10% 시 경고 배너: "리워드 총량이 10% 이하입니다. 보충을 권장합니다."  


**3.2.2.3.** 보충 이력  

1: 보충 이력 표시  
　A. 최근 보충 내역 최대 5건 표시  
　B. 표시 항목: 보충 일시, 보충 수량, 보충 후 총량  
　C. 보충 이력 없을 시: "보충 이력이 없습니다" 표시  


**3.2.3.** 광고주 7등 리워드 현황  


**3.2.3.1.** 7등 파트너 리워드 현황 (광고주 게임 ON 시에만 표시, 읽기 전용)  

1: 표시 항목  
　A. 활성 캠페인명  
　B. 7등 리워드명  
　C. 누적 당첨 건수 (선택 기간 내)  
　D. 당첨 후 실제 사용 건수  
　E. 사용률: 사용 건수 / 당첨 건수 × 100 (%)  


• 참조: 3.0. 페이지 개요, 7-2-7-1. 게임 설정 > 1.3. 리워드 적용·프리셋 관리, 7-2-8-1. 현황
