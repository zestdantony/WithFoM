**1.6.** 액션 버튼 및 화면 전환  


**1.6.1.** 섹션 제목 클릭 시  

**1.6.1.1.** 이동 경로  
1: 클릭 대상  
　A. 섹션 제목 "결제 내역" 텍스트  
　B. 제목 우측의 실시간 주문 건수 뱃지 (예: "(3)")  
2: 이동 목적지  
　A. 사이드바 > 주문 관리 카테고리 > 주문 관리 페이지로 이동  
　B. 주문 관리 페이지에서 전체 주문 목록 확인 가능  
3: 이동 시 필터 적용  
　A. 상태 필터: "신규" 상태로 자동 필터링  
　B. 기간 필터: "오늘" 자동 선택  


**1.6.2.** 전체 보기 링크 클릭 시  

**1.6.2.1.** 이동 경로  
1: 클릭 대상  
　A. 섹션 헤더 우측의 "전체 보기" 텍스트 링크  
　B. 또는 ">" 화살표 아이콘  
2: 이동 목적지  
　A. 사이드바 > 주문 관리 카테고리 > 주문 관리 페이지로 이동  
　B. 필터 없이 전체 주문 목록 표시  


**1.6.3.** 개별 카드 클릭 시  

**1.6.3.1.** 카드 본문 영역 클릭  
1: 클릭 대상  
　A. 카드 내 버튼 영역을 제외한 모든 영역  
　B. 주문 정보, 고객 정보, 시간 정보 영역  
2: 이동 목적지  
　A. 사이드바 > 주문 관리 카테고리 > 주문 관리 페이지로 이동  
　B. 해당 주문 상세 패널이 자동으로 열린 상태로 이동  
3: 주문 상세 패널 내용  
　A. 주문 전체 정보 (메뉴, 옵션, 수량, 금액)  
　B. 결제 정보 (결제 방법, 리워드 적용 내역)  
　C. 고객 정보 (닉네임, 연락처, 주문 이력)  
　D. 상태별 버튼 표시  

**1.6.3.2.** 상태별 패널 버튼  
1: 신규 상태 카드 클릭 시  
　A. [승인] 버튼 / [거절] 버튼 표시  
　B. 거절 시 사유 선택 팝업  
2: 준비중 상태 카드 클릭 시  
　A. [준비 완료] 버튼 표시  
　B. [주문 취소] 버튼 표시 (예외 상황용)  
　C. 취소 시 사유 선택 → 자동 환불 처리  
3: 준비완료 상태 카드 클릭 시  
　A. "픽업 대기 중" 상태 텍스트 표시  
　B. [주문 취소] 버튼 표시 (예외 상황용)  
　C. 미픽업 시간 경과 시 [리마인더 발송] / [폐기 예고 발송] / [미픽업 폐기] 버튼 표시  


**1.6.4.** 빈 상태 UI  

**1.6.4.1.** 주문이 없는 경우  
1: 빈 상태 표시  
　A. 카드 영역에 빈 상태 컴포넌트 표시  
　B. 아이콘: 커피컵 또는 주문서 형태의 회색 아이콘  
　C. 메인 문구: "대기 중인 주문이 없습니다"  
　D. 보조 문구: "주문이 들어오면 여기에 표시됩니다"  
2: 실시간 갱신  
　A. WebSocket으로 신규 주문 수신 시 빈 상태 컴포넌트 제거  
　B. 카드 리스트로 즉시 전환  

**1.6.4.2.** 일일 초기화 시  
1: 초기화 시점  
　A. 매일 해당 매장의 영업 시간 시작 시 자동 초기화  
　B. 초기화 기준: 매장 설정에 등록된 영업 시작 시간 (필수 설정 항목)  
2: 초기화 동작  
　A. 전일 처리 완료된 주문 카드 (준비완료/픽업완료 상태) 제거  
　B. 전일 취소 처리된 주문 카드 기록 제거  
　C. 결제 내역 섹션: 빈 카드 리스트로 표시  
　D. 안내 문구: "대기 중인 주문이 없습니다" 노출  
3: 데이터 보존  
　A. 초기화된 주문 내역은 주문 관리 페이지에서 계속 조회 가능  
　B. 결제 정산 및 리포트 데이터에는 영향 없음  


**1.6.5.** 팝업/모달 UI 전환  

**1.6.5.1.** 거절 확인 팝업  
1: 표시 조건  
　A. 신규 상태 카드에서 [거절] 버튼 클릭 시  
2: 팝업 구성  
　A. 팝업 제목: "주문을 거절하시겠습니까?"  
　B. 팝업 내용: "거절 시 고객에게 알림이 발송되며, 결제가 취소됩니다."  
　C. 거절 사유 선택 영역 (필수, 10개 항목)  
　D. 버튼: [취소] / [거절 확인]  
3: 애니메이션  
　A. 팝업 등장: 페이드인 + 스케일업 (200ms)  
　B. 배경 딤 처리 (opacity: 0.5)  

**1.6.5.2.** 취소 확인 팝업  
1: 표시 조건  
　A. 준비중/준비완료 상태에서 [주문 취소] 버튼 클릭 시  
2: 팝업 구성  
　A. 팝업 제목: "주문을 취소하시겠습니까?"  
　B. 팝업 내용: "취소 시 고객에게 알림이 발송되며, 결제가 환불됩니다."  
　C. 취소 사유 선택 영역 (필수, 10개 항목)  
　D. 버튼: [취소] / [취소 확인]  
3: 팝업 닫기  
　A. [취소] 버튼 클릭  
　B. 팝업 외부 영역 클릭  
　C. ESC 키 입력

**1.6.6.1.** 표시 위치  
1: 화면 하단 중앙  
　A. 하단에서 16px 여백  
　B. 가로 중앙 정렬

**1.6.6.2.** 토스트 종류

　A. "주문이 승인되었습니다"  
　B. "자동 승인되었습니다"  
　C. "준비 완료 처리되었습니다"  
　D. "취소 완료되었습니다"

　A. "주문번호 #0042 - 고객이 주문을 취소했습니다"

　A. "네트워크 오류가 발생했습니다. 다시 시도해 주세요."

　B. 에러 토스트는 5초 유지  
2: 중복 토스트  
　A. 동일 메시지는 중복 표시하지 않음  
　B. 새 토스트는 기존 토스트 위에 스택  


**1.6.7.** 화면 전환 애니메이션  

**1.6.7.1.** 페이지 이동  
1: 대시섹션 → 주문 관리 페이지  
　A. 좌측에서 우측으로 슬라이드 전환 (300ms)  
　B. 이전 페이지는 우측으로 페이드아웃  
2: 주문 관리 페이지 → 대시섹션  
　A. 우측에서 좌측으로 슬라이드 전환 (300ms)  

**1.6.7.2.** 주문 상세 패널  
1: 패널 열기  
　A. 하단에서 상단으로 슬라이드 업 (250ms)  
　B. 배경 딤 처리 (opacity: 0.5)  
2: 패널 닫기  
　A. 상단에서 하단으로 슬라이드 다운 (200ms)  
　B. 배경 딤 페이드아웃  

**1.6.7.3.** 카드 상태 변경 애니메이션  
1: 신규 → 준비중  
　A. 테두리 색상 변경 (브랜드 컬러 → 주황색, 300ms)  
　B. "NEW" 뱃지 → "준비중" 뱃지 크로스페이드 (200ms)  
　C. 버튼 영역 변경 (승인/거절 → 준비 완료)  
2: 준비중 → 준비완료  
　A. 테두리 색상 변경 (주황색 → 초록색, 300ms)  
　B. "준비중" 뱃지 → "완료" 뱃지 크로스페이드 (200ms)  
3: 취소 시  
　A. 카드 붉은색으로 깜빡임 (0.5초)  
　B. 페이드아웃 후 제거 (300ms)  
4: 소비자 취소 시  
　A. 카드 붉은색으로 깜빡임 (0.5초)  
　B. "고객 취소" 라벨 오버레이 표시  
　C. 페이드아웃 후 제거 (300ms)
