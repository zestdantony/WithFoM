**1.3.** 카드 구성  


**1.3.1.** 카드 노출 규칙  

**1.3.1.1.** 노출 개수  
1: 최대 5개 카드까지만 노출  
　A. 위드폼 오더로 들어온 최근 5개 주문을 각각의 카드로 표시  
　B. 6개 이상 주문 시 가장 오래된 주문 카드는 숨김 처리  
　C. 전체 주문은 주문 관리 페이지에서 확인 가능  


**1.3.2.** 카드 정렬 순서  

**1.3.2.1.** 정렬 기준  
1: 주문 시간 기준 최신순 정렬  
　A. 가장 최근 주문이 좌측 첫 번째 카드로 노출  
　B. 주문 시간이 오래된 순서대로 우측으로 배치  
2: 상태별 우선순위  
　A. 승인 대기(신규) 상태의 주문이 우선 노출  
　B. 준비중/준비완료 상태의 주문도 시간순으로 이어서 노출  
3: 상시 노출 원칙  
　A. 최근 5개 주문 카드는 상태와 관계없이 항상 결제 내역 섹션에 노출  
　B. 승인 완료(준비중), 준비완료 상태도 카드 형태로 계속 표시  
　C. 취소된 주문만 리스트에서 제거  
　D. 6번째 이상 오래된 주문은 자동으로 숨김 처리 (주문 관리 페이지에서 확인 가능)  


**1.3.3.** 카드 레이아웃  

**1.3.3.1.** 카드 구조  
1: 카드 크기  
　A. 가로: 280px (고정)  
　B. 세로: 자동 (콘텐츠에 따라 가변)  
　C. 카드 간 간격: 16px  
2: 카드 구성 영역  
　A. 상단: 배경 영역 (매장 로고 또는 매장명)  
　B. 중단: 주문 정보 영역  
　C. 하단: 버튼 영역 (승인/거절)  
3: 카드 배경  
　A. 매장 로고 이미지가 등록된 경우: 로고 이미지를 배경으로 사용 (투명도 적용)  
　B. 로고 미등록 시: 매장명을 큰 텍스트로 배경에 표시  


**1.3.4.** 카드 상태 표시  

**1.3.4.1.** 시각적 구분  
1: 신규 주문 카드 (승인 대기)  
　A. 카드 테두리: 브랜드 컬러 (강조)  
　B. 좌측 상단에 "NEW" 뱃지 표시  
　C. 카운트다운 타이머 활성화  
　D. 승인/거절 버튼 노출  
2: 준비중 카드 (승인 완료)  
　A. 카드 테두리: 주황색 (진행 중)  
　B. 좌측 상단에 "준비중" 뱃지 표시  
　C. [준비 완료] 버튼 노출  
　D. 승인/거절 버튼 숨김  
3: 준비완료 카드  
　A. 카드 테두리: 초록색 (완료)  
　B. 좌측 상단에 "완료" 뱃지 표시  
　C. 버튼 영역에 "픽업 대기 중" 상태 텍스트 표시  
4: 취소된 카드  
　A. 리스트에서 즉시 제거  
　B. 취소 완료 시 토스트 메시지로 알림  


**1.3.5.** 카드 초기화 규칙  

**1.3.5.1.** 일일 초기화  
1: 초기화 시점  
　A. 매일 해당 매장의 영업 시간 시작 시 자동 초기화  
　B. 초기화 기준: 매장 설정에 등록된 영업 시작 시간 (필수 설정 항목)  
2: 초기화 대상  
　A. 전일 처리 완료된 주문 카드 (준비완료/픽업완료 상태)  
　B. 전일 취소 처리된 주문 카드 기록  
3: 초기화 후 상태  
　A. 결제 내역 섹션: 빈 카드 리스트로 표시  
　B. 안내 문구: "대기 중인 주문이 없습니다" 노출  
　C. 신규 주문 발생 시 즉시 카드 생성  

**1.3.5.2.** 데이터 보존  
1: 주문 내역 보존  
　A. 초기화된 주문 내역은 주문 관리 페이지에서 계속 조회 가능  
　B. 결제 정산 및 리포트 데이터에는 영향 없음  


• 참조: 7-2-9-4. 주문 리스트 섹션
