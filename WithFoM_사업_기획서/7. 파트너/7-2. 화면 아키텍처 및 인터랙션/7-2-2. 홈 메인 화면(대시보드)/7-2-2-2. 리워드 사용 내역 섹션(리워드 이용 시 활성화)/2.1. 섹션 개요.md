**2.1.** 섹션 개요  


**2.1.1.** 섹션 정의  

**2.1.1.1.** 기본 정보  
1: 섹션 위치  
　A. 파트너 포털 홈 화면(대시섹션)의 두 번째 섹션으로 배치  
　B. 결제 내역 섹션(7-2-2-1) 바로 아래에 위치  
　C. 오더 기능을 사용하지 않는 파트너의 경우, 첫 번째 섹션으로 노출  
2: 섹션 명칭  
　A. 섹션 제목: "리워드 사용 내역"  
　B. 제목 우측에 실시간 승인 대기 건수 뱃지 표시 (예: "리워드 사용 내역 (2)")  


**2.1.2.** 섹션 목적  

**2.1.2.1.** 핵심 기능  
1: 소비자가 매장에서 사용한 리워드(쿠폰/포인트/스탬프 혜택)를 파트너가 즉시 확인하고 처리  
2: 위드폼 오더 미사용 시 리워드 승인/거절을 대시섹션에서 바로 처리  
3: 60초 자동 승인 타이머로 바쁜 시간에도 리워드 사용 누락 방지  
4: 리워드 적용 후 결제 금액 계산 기능 제공  


**2.1.3.** 섹션 레이아웃  

**2.1.3.1.** 화면 구성  
1: 섹션 헤더  
　A. 좌측: 섹션 제목 "리워드 사용 내역" + 실시간 승인 대기 건수 뱃지  
　B. 우측: "전체 보기" 링크 (클릭 시 리워드 사용 페이지로 이동)  
2: 섹션 본문  
　A. 가로 슬라이드 형태의 카드 리스트  
　B. 최근 5개 리워드 사용 요청까지만 카드로 노출  
　C. 카드가 없을 경우 "리워드 사용 요청이 없습니다" 안내 문구 표시  


**2.1.4.** 리워드 유형별 처리  

**2.1.4.1.** 쿠폰  
1: 쿠폰 정의  
　A. 파트너가 발행한 매장 전용 쿠폰  
　B. 특정 조건 충족 시 혜택을 제공하는 일회성 리워드  
2: 쿠폰 유형  
　A. 무료 제공 쿠폰: 특정 메뉴 무료 제공 (예: 아메리카노 무료 쿠폰)  
　B. 할인 쿠폰: 고정 금액 또는 비율 할인 (예: 1,000원 할인, 10% 할인)  
3: 사용 처리  
　A. 소비자가 쿠폰 사용 요청 → 파트너 승인 → 혜택 적용  
　B. 무료 제공: 해당 메뉴 금액 전액 차감  
　C. 할인: 결제 금액에서 할인 금액 차감  

**2.1.4.2.** 포인트  
1: 포인트 정의  
　A. 게임을 통해 획득한 포인트를 현금처럼 사용  
　B. 1포인트 = 1원으로 결제 금액에서 차감  
2: 획득 방식  
　A. 포인트는 게임 참여를 통해서만 획득 가능  
　B. 별도 적립 기능 없음 (게임 전용 리워드)  
3: 사용 처리  
　A. 소비자가 사용할 포인트 금액 입력 → 사용 요청  
　B. 파트너 승인 → 결제 금액에서 포인트 금액 차감  
　C. 예시: 5,000원 결제 시 500포인트 사용 → 4,500원 결제  

**2.1.4.3.** 스탬프 혜택  
1: 스탬프 정의  
　A. 스탬프 카드 완성(예: 10개) 시 제공되는 혜택을 리워드로 사용  
2: 혜택 유형  
　A. 스탬프 설정에서 지정한 혜택이 쿠폰으로 발급됨  
　B. 예시: 무료 음료 쿠폰, 할인 쿠폰 등  
3: 사용 처리  
　A. 스탬프 카드 완성 시 설정된 혜택이 쿠폰으로 자동 발급  
　B. 발급된 쿠폰을 리워드로 사용 (쿠폰 처리 방식 적용)  


**2.1.5.** 위드폼 오더 연동  

**2.1.5.1.** 위드폼 오더 결제 시 리워드 사용  
1: 표시 방식  
　A. 결제 내역 섹션(7-2-2-1)에 결제 내역 카드가 노출되는 동시에  
　B. 리워드 사용 내역 섹션에도 "결제 시 사용된 리워드" 카드가 함께 표시  
2: 버튼 처리  
　A. 리워드 섹션에서는 승인/거절 버튼 미노출 (결제 섹션에서 일괄 처리)  
　B. 리워드 카드에 "결제 연동" 라벨 표시  
3: 승인/거절 연동  
　A. 결제 섹션에서 주문 승인 시 → 리워드 사용도 자동 승인 처리  
　B. 결제 섹션에서 주문 거절 시 → 리워드 사용도 자동 거절 처리, 리워드 소비자 복구  


**2.1.6.** 위드폼 오더 미사용 시 리워드 단독 사용  

**2.1.6.1.** 리워드 단독 사용 처리  
1: 사용 시나리오  
　A. 소비자가 매장에서 직접 리워드 사용 요청  
　B. 위드폼 오더가 아닌 일반 결제 시 리워드 적용  
2: 버튼 처리  
　A. 리워드 카드에 [승인] / [거절] 버튼 노출  
　B. 파트너가 승인 시 리워드 혜택 확정  
　C. 파트너가 수동으로 리워드 금액 적용하여 결제 진행  
3: 결제 금액 계산  
　A. 리워드 카드에 "결제 전 금액" 입력 필드 제공  
　B. 금액 입력 시 리워드 적용 후 "최종 결제 금액" 자동 계산 표시  
　C. 파트너는 계산된 최종 금액으로 결제 진행
