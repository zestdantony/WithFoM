**6.2.** 노출 조건  


**6.2.1.** 활성화 조건  

**6.2.1.1.** 기본 노출 조건  
1: 리워드 기능 활성화 파트너에게 노출  
　A. 리워드형 또는 복합형 파트너에게 기본 노출  
　B. 쿠폰, 포인트, 스탬프 중 하나 이상 활성화 시 노출  
2: 오더형 전용 파트너  
　A. 리워드 기능 미사용 시 섹션 숨김 처리  
　B. "리워드 기능을 활성화하면 고객 재방문율을 높일 수 있습니다" 안내 (선택적)  


**6.2.2.** 서비스 타입별 표시 범위  

**6.2.2.1.** 리워드형/복합형 파트너  
1: 쿠폰 카드  
　A. 쿠폰 기능 활성화 시 표시  
　B. 쿠폰 미사용 시: "쿠폰 기능을 활성화하세요" 안내  
2: 포인트 카드  
　A. 포인트 기능 활성화 시 표시 (게임을 통해서만 발급)  
　B. 포인트 미사용 시: "게임을 통해 포인트를 발급할 수 있습니다" 안내  
3: 스탬프 카드  
　A. 스탬프 기능 활성화 시 표시  
　B. 스탬프 미사용 시: "스탬프 기능을 활성화하세요" 안내  

**6.2.2.2.** 활성화 기능별 표시  
1: 쿠폰만 활성화  
　A. 쿠폰 카드만 데이터 표시  
　B. 나머지 카드는 활성화 유도 안내  
2: 전체 활성화  
　A. 3개 카드 모두 데이터 표시  


**6.2.3.** 데이터 없는 경우  

**6.2.3.1.** 오늘 데이터 없음  
1: 쿠폰 카드  
　A. "오늘 발행 0건 / 사용 0건"  
2: 포인트 카드  
　A. "오늘 발행 0P / 사용 0P"  
3: 스탬프 카드  
　A. "오늘 적립 0개 / 혜택 발급 0건"  

**6.2.3.2.** 신규 가입 파트너  
1: 빈 상태 처리  
　A. 아직 리워드 발행 이력이 없는 신규 파트너에게는 각 카드에 안내 문구 표시  
　B. "첫 쿠폰을 발행하면 여기에 표시됩니다" (쿠폰)  
　C. "첫 포인트가 발급되면 여기에 표시됩니다" (포인트)  
　D. "첫 스탬프가 적립되면 여기에 표시됩니다" (스탬프)  


**6.2.4.** 초기화  

**6.2.4.1.** 일일 초기화 시점  
1: 초기화 기준  
　A. 매일 영업 오픈 시간 기준으로 "오늘" 데이터 초기화  
　B. 24시간 영업장은 자정(00:00) 기준으로 초기화  
2: 초기화 동작  
　A. 오늘 발행/사용 건수 → 0으로 리셋  
　B. 오늘 적립/혜택 발급 → 0으로 리셋  
3: 데이터 보존  
　A. 초기화 전 데이터는 "어제" 데이터로 저장  
　B. 상세 페이지에서 기간별 조회 가능  

**6.2.4.2.** 월별 초기화 시점  
1: 토탈 데이터 초기화  
　A. 매월 1일 영업 오픈 시간에 토탈 데이터 초기화  
　B. 토탈 값은 해당 월의 합산 데이터만 표시 (한 달치)  
2: 초기화 동작  
　A. 토탈 발행/사용 건수 → 0으로 리셋  
　B. 토탈 적립/혜택 발급 → 0으로 리셋  
3: 데이터 보존  
　A. 초기화 전 데이터는 "전월" 데이터로 저장  
　B. 상세 페이지에서 월별/기간별 조회 가능
