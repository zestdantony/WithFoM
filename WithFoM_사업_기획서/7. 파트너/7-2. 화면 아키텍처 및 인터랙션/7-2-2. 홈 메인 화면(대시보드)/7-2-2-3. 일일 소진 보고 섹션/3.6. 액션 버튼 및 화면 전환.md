**3.6.** 액션 버튼 및 화면 전환  


**3.6.1.** 섹션 제목 클릭 시  

**3.6.1.1.** 이동 경로  
1: 클릭 대상  
　A. 섹션 제목 "일일 소진 보고" 텍스트  
　B. 제목 우측의 보고 상태 뱃지  
2: 이동 목적지  
　A. 사이드바 > 캠페인 관리 카테고리 > 캠페인 일일 소진 보고 페이지로 이동  
　B. 캠페인 일일 소진 보고 상세 페이지에서 전체 이력 확인 가능  
3: 이동 시 필터 적용  
　A. 기간 필터: "이번 캠페인" 자동 선택  
　B. 캠페인 필터: 대시섹션에서 선택된 캠페인으로 필터링  


**3.6.2.** 상세 보기 링크 클릭 시  

**3.6.2.1.** 이동 경로  
1: 클릭 대상  
　A. 섹션 헤더 우측의 "상세 보기" 텍스트 링크  
　B. 또는 ">" 화살표 아이콘  
2: 이동 목적지  
　A. 사이드바 > 캠페인 관리 카테고리 > 캠페인 일일 소진 보고 페이지로 이동  
　B. 필터 없이 전체 소진 보고 이력 표시  


**3.6.3.** 제출 버튼 클릭 시  

**3.6.3.1.** 제출 동작  
1: 버튼 상태 변화  
　A. 클릭 시: 버튼 비활성화 + 로딩 스피너 표시  
　B. 완료 시: 성공 토스트 표시 + 섹션 상태 변경  
2: 제출 성공 시  
　A. 토스트 메시지: "소진 보고가 완료되었습니다"  
　B. 입력창 → 완료 상태 UI로 전환  
　C. 섹션 테두리 색상: 빨간색 → 초록색  
　D. 상태 뱃지: "보고 필요" → "보고 완료"  
3: 제출 실패 시  
　A. 에러 토스트: "제출에 실패했습니다. 다시 시도해 주세요."  
　B. 입력값 유지  
　C. 재시도 버튼 활성화  


**3.6.4.** 수정 버튼 클릭 시  

**3.6.4.1.** 전일 소진량 수정  
1: 클릭 시 동작  
　A. 인라인 수정 모드 활성화  
　B. 전일 소진량 값이 입력 필드로 변환  
　C. [저장] / [취소] 버튼 표시  
2: 저장 시 동작  
　A. 수정값 검증 (평균 대비 경고 적용)  
　B. 저장 완료 토스트: "소진량이 수정되었습니다"  
　C. 수정 이력 기록  
3: 취소 시 동작  
　A. 원래 값으로 복원  
　B. 수정 모드 종료  

**3.6.4.2.** 당일 보고 수정 (보고 완료 후)  
1: 수정 버튼 표시 조건  
　A. 당일 보고 완료 후 수정 가능 시간 내  
　B. 익일 영업 시작 후 2시간까지  
2: 클릭 시 동작  
　A. 보고 완료 UI → 입력 모드로 전환  
　B. 기존 입력값 표시된 상태에서 수정 가능  


**3.6.5.** 캠페인 선택 변경 시  

**3.6.5.1.** 탭/드롭다운 변경  
1: 변경 시 동작  
　A. 선택된 캠페인의 소진 정보로 즉시 갱신  
　B. 전일 소진량, 오늘 소진량 입력 상태 모두 갱신  
　C. 누적 소진량, 잔여 수량 등 통계 정보 갱신  
2: 애니메이션  
　A. 페이드 인/아웃 전환 (200ms)  


**3.6.6.** 팝업/모달 UI 전환  

**3.6.6.1.** 경고 확인 팝업  
1: 표시 조건  
　A. 이상치 감지 시 (평균 대비 1.5배 이상 차이)  
　B. 0 입력 시  
2: 팝업 구성  
　A. 팝업 제목: "입력값 확인" 또는 "소진량 확인"  
　B. 팝업 내용: 상황별 안내 메시지  
　C. 버튼: [취소] / [확인]  
3: 애니메이션  
　A. 팝업 등장: 페이드인 + 스케일업 (200ms)  
　B. 배경 딤 처리 (opacity: 0.5)  

**3.6.6.2.** 수정 불가 안내 팝업  
1: 표시 조건  
　A. 수정 가능 시간 경과 후 수정 버튼 클릭 시  
2: 팝업 구성  
　A. 팝업 제목: "수정 시간 종료"  
　B. 팝업 내용: 수정 불가 사유 안내  
　C. 버튼: [확인]  


**3.6.7.** 토스트 메시지
