**3.5.** 입력 및 경고 로직  


**3.5.1.** 입력 유효성 검사  

**3.5.1.1.** 입력 형식 검증  
1: 허용 입력  
　A. 숫자만 입력 가능 (0~9)  
　B. 음수 입력 불가  
　C. 소수점 입력 불가  
2: 입력 제한  
　A. 최대 입력 자릿수: 5자리 (99,999개)  
　B. 최소 입력값: 0  
3: 실시간 검증  
　A. 숫자 외 문자 입력 시 자동 제거  
　B. 입력 즉시 천 단위 콤마 적용  


**3.5.2.** 평균 대비 경고  

**3.5.2.1.** 이상치 감지 조건  
1: 경고 기준  
　A. 입력값이 해당 파트너 평균 대비 1.5배 이상 높은 경우  
　B. 입력값이 해당 파트너 평균 대비 0.5배 이하인 경우  
　C. 평균 산정: 최근 14일간 일일 소진량 평균  
2: 예외 상황  
　A. 첫 보고 시에는 경고 없음 (평균 데이터 부족)  
　B. 캠페인 시작 7일 이내에는 경고 완화 적용  

**3.5.2.2.** 경고 팝업  
1: 팝업 제목  
　A. "입력값 확인"  
2: 팝업 내용  
　A. "입력값이 맞는지 확인해 주세요."  
3: 팝업 버튼  
　A. [취소]: 입력창으로 돌아가 수정  
　B. [확인]: 입력값 그대로 제출  


**3.5.3.** 제출 확인 로직  

**3.5.3.1.** 0 입력 시 확인  
1: 조건  
　A. 소진량으로 "0"을 입력 후 제출 시  
2: 확인 팝업  
　A. 팝업 제목: "소진량 확인"  
　B. 팝업 내용: "오늘 소진량이 0개가 맞습니까?"  
　C. 버튼: [취소] / [확인]  

**3.5.3.2.** 일반 제출 시  
1: 제출 동작  
　A. 경고 조건에 해당하지 않으면 바로 제출  
　B. 경고 조건에 해당하면 경고 팝업 후 확인 시 제출  
2: 제출 완료  
　A. 성공 토스트: "소진 보고가 완료되었습니다"  
　B. 섹션 상태 변경: 빨간색 → 초록색  
　C. 뱃지 변경: "보고 필요" → "보고 완료"  


**3.5.4.** 전일 소진량 수정 로직  

**3.5.4.1.** 수정 가능 조건  
1: 시간 조건  
　A. 익일 영업 시작 후 2시간까지 수정 가능  
　B. 24시간 영업 매장: 다음 날 오전 11:00까지  
2: 수정 버튼 상태  
　A. 수정 가능 시간: 활성화 (클릭 가능)  
　B. 수정 불가 시간: 비활성화 (회색 처리)  

**3.5.4.2.** 수정 불가 안내  
1: 수정 불가 시 팝업  
　A. 팝업 제목: "수정 시간 종료"  
　B. 팝업 내용:  
　　"소진량 수정 시간이 종료되었습니다.  
　　수정은 익일 오픈 후 2시간까지만 가능하며,  
　　이후에는 정산 정확도를 위해 변경할 수 없습니다."  
　C. 버튼: [확인]  

**3.5.4.3.** 수정 이력 관리  
1: 수정 기록  
　A. 수정 전 값과 수정 후 값 모두 저장  
　B. 수정 시간 및 수정자 정보 기록  
2: 연동  
　A. 캘린더, 대시섹션, 리포트에 수정 결과 반영  


**3.5.5.** 미보고 알림 로직  

**3.5.5.1.** 알림 발송 조건  
1: 1차 알림  
　A. 영업 종료 후 1시간 경과 시 발송  
　B. 푸시 알림 + 앱 내 알림  
2: 2차 알림  
　A. 익일 영업 시작 1시간 후 미보고 시 발송  
　B. 푸시 알림 + 앱 내 알림  
3: 경고 알림  
　A. 연속 3일 미보고 시 발송  
　B. 파트너 등급 영향 안내 포함  

**3.5.5.2.** 알림 메시지  
1: 1차/2차 알림  
　A. "어제 소진 보고가 완료되지 않았습니다. 소진 보고를 완료해 주세요."  
　B. [바로가기] 버튼으로 보고 페이지 이동  
2: 연속 미보고 경고  
　A. "소진 보고가 연속으로 누락되고 있습니다."  
　B. "정확한 정산을 위해 소진 보고를 꼭 완료해 주세요."  
　C. "미보고 지속 시 파트너 등급에 영향을 줄 수 있습니다."  


**3.5.6.** 이상 탐지 (FDS 연동)  

**3.5.6.1.** 이상 패턴 감지  
1: 탐지 기준  
　A. 전일 대비 ±150% 이상 급변이 반복되는 경우  
2: 탐지 목적  
　A. 정산 무결성 보호  
　B. 데이터 신뢰도 확보  
3: 탐지 시 조치  
　A. 시스템 내부 플래그 설정  
　B. 운영팀 검토 대상으로 분류  


• 참조: 7-2-5-1. 캠페인 일일 소진 보고 > 1.1.3. 오늘 소진량 입력창, 7-1-5-4. 미보고 알림 수신
